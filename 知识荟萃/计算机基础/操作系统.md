### **并发控制核心概念**

#### 1. **并发 vs 并行**

- **并发（Concurrency）**
  多个任务在单核CPU上交替执行（时间片轮转），宏观看似“同时”，微观是串行。
  **核心目的**：提高资源利用率，避免CPU空闲。
  **举例**：单核CPU运行多个线程。
- **并行（Parallelism）**  平行！！
  多核CPU真正同时执行多个任务。
  **核心目的**：提升吞吐量。
  **举例**：多线程计算矩阵乘法，每个线程处理不同分块。

#### 2. **竞争条件（Race Condition）**

- **定义**：多**个进程/线程对共享资源的访问顺序不确定，导致结果不可预测。 ** **并发问题/线程安全问题**

- **经典场景**：

  

  ```c++
  // 共享变量
  int counter = 0;
  void increment() { counter++; } // 多个线程同时执行此函数
  ```

  **问题**：`counter++`在汇编层面是“读-改-写”三步操作，可能被中断导致结果错误。 不是原子操作 也不是单一语句命令

#### 3. **临界区（Critical Section）**

- **定义**：<span style="background:#FF0000;">**访问共享资源的代码段**。</span>
- **解决竞争条件的三要素**：
  - **互斥（Mutual Exclusion）**：同一时刻仅一个线程进入临界区。
  - **有限等待（Bounded Waiting）**：线程等待进入临界区的时间有限。
  - **进展（Progress）**：无线程在临界区时，其他线程可竞争进入。

自旋锁（Spinlock）是什么？
定义：一种**忙等（busy-waiting）**机制，线程在尝试获取锁时，如果获取不到，不会挂起，而是在循环中持续尝试（自旋）。

常见实现：while(!CAS(lock)) continue;

✅ 优点：
线程不会发生上下文切换，适合锁时间非常短的场景（如多核 CPU）。

❌ 缺点：
若锁持有时间较长，会浪费 CPU，性能反而下降。

非公平锁，不能保证“先等待先获取”，不满足有限等待。

✅ 三、重复尝试获取锁（重试锁）是什么？
定义：线程在某种逻辑下，尝试获取锁，如果失败，会退让/等待一段时间再试。

一般用于用户态的轻量级锁，或带有退避机制的乐观锁。

✅ 特点：
一定程度上解决了自旋锁的“忙等”问题，结合有限等待策略（如指数退避）可满足“有限等待”条件。

常用于数据库乐观锁或 CAS 重试场景。

CAS 是一种由**硬件支持的原子指令**，它会将某个变量的当前值与预期值进行比较：

> **如果相等，则更新为新值；否则不做操作。**

这一操作是**原子的**，不会被中断，因此可在并发场景下保证数据一致性。

✅**CAS 优点**

| 优点                 | 说明                                 |
| -------------------- | ------------------------------------ |
| ✅ 无锁，提高并发性能 | 不使用传统加锁，避免线程阻塞         |
| ✅ 原子性由硬件保证   | 安全高效，适用于轻量并发控制         |
| ✅ 应用于各种并发组件 | 如 `Atomic*` 类、`ConcurrentHashMap` |

**❌ CAS 缺点（面试常考）**

| 问题             | 说明                                         |
| ---------------- | -------------------------------------------- |
| ABA 问题         | 某值从 A→B→A，CAS 会认为没变，实际发生了变化 |
| 只能操作单个变量 | 多变量需要借助 `AtomicReference` 或锁机制    |

#### 1. **互斥锁（Mutex）**

- **适用场景**：保护临界区，确保单线程访问。
- **实现原理**：
  - **原子操作**：基于**CPU的CAS（Compare-And-Swap）指令实现锁状态变更。**
  - **阻塞等待**：未获取锁的线程进入休眠，由内核调度唤醒。
- **问题**：
  - **死锁**：若锁未释放或重复加锁。
  - **优先级反转**：高优先级线程等待低优先级线程持有的锁。

#### 2. **信号量（Semaphore）**

- **分类**：

  - **二进制信号量**：取值0或1，等同于互斥锁。
  - **计数信号量**：表示资源数量（如缓冲区空槽数）。

- **经典应用**：生产者-消费者问题。

  ```c++
  sem_t empty, full, mutex;
  sem_init(&empty, 0, N); // 初始N个空位
  sem_init(&full, 0, 0);  // 初始0个数据
  sem_init(&mutex, 0, 1); // 互斥锁
  
  // 生产者
  sem_wait(&empty);
  sem_wait(&mutex);
  enqueue(data);
  sem_post(&mutex);
  sem_post(&full);
  
  // 消费者
  sem_wait(&full);
  sem_wait(&mutex);
  data = dequeue();
  sem_post(&mutex);
  sem_post(&empty);
  ```

#### 3. **条件变量（Condition Variable）**

- **作用**：让线程在条件不满足时主动休眠，避免**忙等待**。

- **使用模式**：

  ```c++
  pthread_mutex_lock(&mutex);
  while (condition_not_met) { // 必须用while而非if（防止虚假唤醒）
      pthread_cond_wait(&cond, &mutex);
  }
  // 执行操作
  pthread_cond_signal(&cond); // 唤醒一个线程
  pthread_mutex_unlock(&mutex);
  ```

#### 4. **读写锁（Read-Write Lock）**

- **特点**：
  - 读锁可共享，写锁独占。
  - 适合读多写少场景（如数据库缓存）。
- **潜在问题**：
  - **写者饥饿**：持续有读者时写者无法获取锁。
  - **解决方案**：设置最大连续读者数量或优先级策略。

### **经典问题与解决方案**

#### 1. **哲学家就餐问题**

- **死锁场景**：所有哲学家同时拿起左边筷子。
- **解决方案**：
  - **资源分级**：为筷子编号，哲学家必须先拿编号小的。
  - **限制并发数**：最多允许N-1个哲学家同时拿筷子。
  - **Try-Lock**：非阻塞获取锁，失败则释放已持有的锁。

#### 2. **读者-写者问题**

- **读者优先**：
  - 写者可能饥饿。
  - 实现：用读计数器+互斥锁+条件变量。
- **写者优先**：
  - 新到达的写者阻塞后续读者。
  - 实现：增加写者等待计数器。

### **面试高频问题**

#### 1. **进程、线程、协程的区别？**

|          | 进程             | 线程             | 协程               |
| :------- | :--------------- | :--------------- | :----------------- |
| 资源分配 | 独立地址空间     | 共享进程资源     | 用户态轻量级       |
| 切换开销 | 高（需切换页表） | 中（切换内核栈） | 低（无需内核介入） |
| 通信方式 | 管道、共享内存等 | 全局变量、互斥锁 | 通道、消息队列     |

#### 2. **死锁的四个必要条件及解决方案**

- **必要条件**：
  1. 互斥访问
  2. 持有并等待
  3. 不可抢占
  4. 循环等待
- **解决方案**：
  - **预防**：破坏任一条件（如资源一次性分配）。
  - **避免**：银行家算法动态检查资源分配安全性。
  - **检测与恢复**：定期检测死锁并终止进程或回滚。

#### 3. **用户态线程 vs 内核态线程**

- **用户态线程（协程）**：
  - 由用户级线程库（如Go的GMP模型）管理。
  - **优点**：切换快（无内核介入）。
  - **缺点**：一个线程阻塞会导致整个进程阻塞。
- **内核态线程**：
  - 由操作系统直接调度。
  - **优点**：可利用多核，阻塞不影响其他线程。
  - **缺点**：切换开销大。

### **性能优化技巧**

#### 1. **减少锁竞争**

- **锁粒度细化**：将一个大锁拆分为多个小锁（如ConcurrentHashMap分段锁）。

- **无锁数据结构**：使用CAS操作实现原子更新。

  

  ```
  // CAS示例（原子递增）
  int atomic_increment(int* value) {
      int old = *value;
      while (!__sync_bool_compare_and_swap(value, old, old+1)) {
          old = *value;
      }
      return old+1;
  }
  ```

#### 2. **异步编程模型**

- **Reactor模式**：单线程事件循环 + 非阻塞I/O（如Redis、Nginx）。
- **Proactor模式**：异步I/O + 完成通知（如Windows IOCP）。
