```dart
// Flutter WebSocket 封装
class ChatWebSocket {
  WebSocketChannel? _channel;
  Timer? _heartbeatTimer;
  
  void connect(String token) {
    _channel = WebSocketChannel.connect(
      Uri.parse('ws://your-server:8090?token=$token')
    );
    
    _channel!.stream.listen((message) {
      final data = jsonDecode(message);
      _handleMessage(data['type'], data['data']);
    });
    
    _startHeartbeat();
  }
  
  void _startHeartbeat() {
    _heartbeatTimer = Timer.periodic(Duration(seconds: 20), (timer) {
      _channel?.sink.add(jsonEncode({'type': 2}));
    });
  }
}
```

### 2.1.3 业务逻辑层（90% 复用）

#### 认证流程
1. 用户输入用户名密码
2. 调用登录 API 获取 Token
3. 存储 Token 到本地
4. 使用 Token 建立 WebSocket 连接
5. 监听连接状态变化

#### 消息处理流程
1. 接收 WebSocket 消息

## 2.3 风险矩阵与应对策略

### 2.3.1 高风险项目

#### WebRTC 兼容性风险
**风险描述**：Flutter WebRTC 插件在某些 Android 设备上可能存在兼容性问题

**影响评估**：
- 影响范围：音视频通话功能
- 严重程度：高（核心功能不可用）
- 发生概率：中等（15-20% 设备）

**应对策略**：
1. **技术方案**：
   - 使用成熟的 `flutter_webrtc` 插件
   - 建立设备兼容性测试矩阵
   - 准备降级方案（纯音频通话）

2. **验收标准**：
   ```dart
   // 设备兼容性检测
   class DeviceCompatibility {
     static Future<bool> checkWebRTCSupport() async {
       try {
         final mediaDevices = await navigator.mediaDevices.enumerateDevices();
         return mediaDevices.isNotEmpty;
       } catch (e) {
         return false;
       }
     }
   }
   ```

3. **回滚方案**：
   - 如果 WebRTC 不可用，隐藏音视频通话按钮
   - 提供文字提示："您的设备暂不支持音视频通话"

#### 性能风险
**风险描述**：大量消息渲染可能导致 UI 卡顿

**影响评估**：
- 影响范围：聊天页面滚动性能
- 严重程度：中等（用户体验下降）
- 发生概率：高（消息量大时必现）

**应对策略**：
1. **技术方案**：
   - 实现消息列表虚拟滚动
   - 图片懒加载
   - 消息分页加载

2. **验收标准**：
   ```dart
   // 性能监控
   class PerformanceMonitor {
     static void measureScrollPerformance() {
       final stopwatch = Stopwatch()..start();
       // 滚动操作
       stopwatch.stop();
       assert(stopwatch.elapsedMilliseconds < 16); // 60fps
     }
   }
   ```

3. **回滚方案**：
   - 限制单次加载消息数量
   - 提供"清理历史消息"功能

### 2.3.2 中风险项目

#### 状态同步风险
**风险描述**：网络不稳定时状态同步可能出现不一致

**应对策略**：
1. **技术方案**：
   - 实现乐观更新 + 服务端确认机制
   - 添加消息发送状态指示
   - 网络恢复后自动同步

2. **验收标准**：
   - 弱网环境下消息不丢失
   - 状态指示清晰（发送中/已发送/已读）

#### UI 适配风险
**风险描述**：不同屏幕尺寸和分辨率适配问题

**应对策略**：
1. **技术方案**：
   - 使用 Flutter 响应式布局组件
   - 建立设计规范和组件库
   - 多设备测试

2. **验收标准**：
   - 支持主流 Android 设备（5-7 英寸）
   - 横竖屏切换正常
   - 不同分辨率下 UI 不变形

### 2.3.3 低风险项目
