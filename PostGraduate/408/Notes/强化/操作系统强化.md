# 操作系统 · 强化分章树（块③·OS）

> 对标 **2027 王道 408 强化**。主干见 [[408强化总纲]]，五大链路在总纲②。
> 每个强化点统一四项范式：**溯源（硬件/前置缺陷推出它）→ 落地（怎么考）→ 短板（边界坑）→ 衍生（催生谁、省略隐患）**，并挂 **[联动计组]**。
> 优先级来自真题命中信号：**S** 必拿（第2、3章）、**A** 重点（第1、4、5章）、**B** 只列要点。剔除入门定义。

# 第1章 概述与运行机制（A-，选择常驻、近年与计组联动）

## 1.1 用户态/内核态与特权指令

- **溯源**：任何指令物理上都能碰全局资源（[[408强化总纲]] 约束 5）→ 必须用 **CPU 特权级**把"会影响全局/他人"的操作锁进内核态。
- **落地**：CPU 用 **PSW** 标志位区分态；特权指令（关中断、置 PSW、IO、修改页表基址）仅内核态可执行。
- **短板**：单纯两态不够细 → 衍生 Ring 0-3、VMM 特权级。
- **[联动计组]**：态的切换靠**中断/异常/陷入**硬件机制完成（计组第5章中断隐指令）；PSW 是计组寄存器。

## 1.2 中断 / 异常 / 系统调用（概念边界，必考陷阱）

| 类型 | 别名 | 来源 | 同步性 | 例子 |
|---|---|---|---|---|
| 外中断 | 中断 | CPU **外部**（设备/时钟） | 异步 | 时钟、IO 完成、键盘 |
| 内中断·trap | 陷入 | 用户**主动**请求内核 | 同步 | **系统调用** |
| 内中断·fault | 故障（可恢复） | 执行中发现问题 | 同步 | **缺页** |
| 内中断·abort | 终止（不可恢复） | 严重错误 | 同步 | 除零、硬件故障 |

- **陷阱**：① **系统调用是 trap，属内中断**；② **缺页是 fault（异常/内中断），不是普通中断、更不是 Cache miss**；③ 外中断在**指令末尾**检测，内中断在指令执行中触发。
- **[联动计组]**：见 [[计算机组成原理强化]] 5.5/7.2，硬件如何响应、保存断点、查向量。

## 1.3 系统调用 vs 库函数

- **溯源**：用户程序不能直接执行特权操作 → 通过系统调用陷入内核。
- **落地**：过程 = **传参 → 陷入指令 → 内核态 → 内核服务例程 → 返回用户态**；分类：进程控制/通信、文件、设备、内存管理。
- **陷阱**：**库函数 ≠ 系统调用**——printf 是库函数，最终可能调 write（系统调用），但库函数本身可不陷入内核。

## 1.4 体系结构 & 引导 & 虚拟机（B，只列要点）

- **大内核**：功能全在内核态，性能高、但庞杂难维护；**微内核**：只留最基本机制，策略上移用户态 → 可靠、但**状态切换频繁、性能下降**。分层/模块化/外核。
- **引导**：ROM/BIOS → POST 自检 → 加载 MBR/活动分区 PBR → 启动管理器 → 内核 → 可加载模块。
- **VMM**：第一类（裸机，性能好，ESXi/Xen）/ 第二类（宿主 OS 之上，VirtualBox）。
- **[联动计组]**：引导起点是 CPU 从 ROM 固定入口取第一条指令（[[408强化总纲]] 链 1 起点）。

# 第2章 进程、线程与处理机调度（S，OS 第一优先级，选择+大题双爆）

## 2.1 进程与 PCB

- **溯源**：多道并发后，程序"走走停停"、需要记录每个执行实例的状态 → 用 **PCB** 描述，进程 = **PCB + 程序段 + 数据段**（进程映像）。
- **落地**：**PCB 是进程存在的唯一标志**；进程是动态的、程序是静态的；进程是资源分配和（传统）调度的基本单位。
- **[联动计组]**：进程切换保存的"上下文"= PC + 通用寄存器 + PSW + 页表基址寄存器 PTBR（[[计算机组成原理强化]] 5.1，[[408强化总纲]] 链 2）。

## 2.2 进程状态转换（选择必考）

- **五态**：创建 → 就绪 ⇄ 运行 → 终止；运行 → 阻塞（主动，等资源）→ 就绪（被动唤醒）。
- **短板/陷阱**：① **阻塞态不能直接到运行态**，必须先回就绪；② 运行→就绪是**时间片到/被抢占**（被动），运行→阻塞是**主动**请求；③ 就绪→阻塞**不存在**。
- **衍生**：内存不足 → **挂起态**（就绪挂起/阻塞挂起），由**中级调度**换出到外存。

## 2.3 进程控制与通信

- **进程控制 = 原语**：创建/撤销/阻塞/唤醒/切换都是**原子操作**，用**关中断**实现不可分割（否则 PCB 改一半被打断 → 状态不一致）。
- **进程通信**：
  - **共享存储**：划一块共享内存，需互斥（最快）。
  - **消息传递**：格式化消息，直接/间接（信箱）通信。
  - **管道 pipe**：**半双工**、固定大小缓冲、字节流、写满阻塞写者/读空阻塞读者。
- **[联动计组]**：原语关中断 = 计组中断屏蔽；共享存储靠 MMU 把同一物理页映射进多进程地址空间。

## 2.4 线程与多线程模型

- **溯源**：进程切换开销大（要换地址空间/页表/TLB），且进程内并发需求增长 → **线程**作为更轻的调度单位。
- **落地**：线程**共享**进程的地址空间/代码/数据/打开文件；**私有**栈、寄存器、PC、程序状态。
- **实现**：**用户级线程 ULT**（线程库管理，内核不可见，切换不进内核但**一个线程阻塞整个进程阻塞**、不能利用多核）/ **内核级线程 KLT**（内核调度、可多核并行、切换开销大）。
- **多线程模型**：一对一（并发好、开销大）/ 多对一（高效、一阻全阻）/ 多对多（折中）。
- **陷阱**：**只有内核级线程才是处理机分配单位**；ULT 切换不需内核态。
- **[联动计组]**：KLT 真并行依赖多核（计组多处理器/ SMT）。

## 2.5 处理机调度

### 2.5.1 三级调度

- **高级（作业）调度**：外存后备 → 内存，决定哪些作业进系统（多道程序度）。
- **中级（内存）调度**：挂起进程的换入换出（对应挂起态）。
- **低级（进程）调度**：就绪 → 运行，最频繁。

### 2.5.2 调度时机/方式/切换

- **非抢占**：只在进程主动让出（结束/阻塞）时调度。**抢占**：更高优先级到达/时间片到可夺权。
- **陷阱**：**进程切换 ≠ 进程调度**（调度选谁，切换换上下文）；**不能切换的时机**：原语执行中、中断处理中、临界区内（需锁）。

### 2.5.3 调度算法（大题必算）

| 算法 | 溯源/动机 | 短板（坑） | 指标 |
|---|---|---|---|
| **FCFS** | 最公平简单 | **利长不利短**、对短作业周转差 | 非抢占 |
| **SJF/SRTN** | 求**平均周转最短** | 需预知运行时间、**长作业饥饿** | SRTN 是抢占版 |
| **HRRN** | 兼顾等待与服务 | 需算响应比 | `响应比=(等待+服务)/服务` |
| **RR** | 分时响应 | **时间片太小→切换开销爆、太大→退化 FCFS** | 抢占、靠时钟中断 |
| **优先级** | 区分轻重缓急 | 低优先级**饥饿**→老化 aging | 静/动、抢/非抢 |
| **MLFQ** | 综合 RR+优先级+反馈 | 参数调优 | 408 常考综合 |

- **评价指标**：周转时间 = 完成 − 到达；**带权周转 = 周转 / 服务时间**（≥1，越小越好）；等待时间；响应时间。
- **[联动计组]**：RR 的时间片靠**时钟中断**强制切换（[[408强化总纲]] 链 2）；切换开销来自寄存器/页表/TLB/流水线冲刷。

## 2.6 同步与互斥（PV 大题命根）

### 2.6.1 临界区四原则

**空闲让进、忙则等待、有限等待、让权等待**。下面每种方法都按"违反了哪条"来记。

### 2.6.2 软件实现（看违反哪条原则）

| 方法 | 缺陷 |
|---|---|
| 单标志法 | 违反**空闲让进**（强制轮流，对方不用也进不去）|
| 双标志先检查 | 违反**忙则等待**（先检查后置位 → 可能**同时进**临界区）|
| 双标志后检查 | 违反**空闲让进/有限等待**（都先置位 → 互相谦让饥饿）|
| **Peterson** | 软件法最优（结合标志+谦让），但仍**忙等**、违反让权等待 |

### 2.6.3 硬件实现

- **中断屏蔽**：关中断进临界区。**短板**：**仅单核有效**（用户笔记重点：多核其他核仍可访问共享资源）；不适合用户程序。
- **TestAndSet / Swap**：硬件**原子**指令，实现忙等锁。**短板**：忙等、违反让权等待。
- **[联动计组]**：原子性依赖计组 **CAS/TAS** 硬件指令；中断屏蔽 = 计组中断允许位。

### 2.6.4 信号量（重点）

- **溯源**：硬件锁忙等浪费 CPU → 记录型信号量用 **block/wakeup** 把等待进程阻塞，实现**让权等待**。
- **落地**：`P(S)=wait`（S--，<0 则 block 自己）；`V(S)=signal`（S++，≤0 则 wakeup 一个）。P 申请、V 释放，二者及 P-P 之间互斥（原子）。
- **三类关系套路**：**互斥**用 mutex（初值 1，同进程内 P…V 包夹临界区）；**同步**用"**前 V 后 P**"（前驱操作后 V，后继操作前 P）；前驱图 = 每条边一个初值 0 的信号量。
- **陷阱**：**互斥 P 与同步 P 的顺序**——同步 P 必须在互斥 P **之前**，否则"拿着锁等资源" → **死锁**（用户笔记：交换顺序→循环等待→死锁）；产品的生产/消费应放在 P-V **之外**减小临界区。

### 2.6.5 经典问题 + 管程

- 生产者-消费者（mutex + empty + full）、多生产多消费、读者-写者（读优先/写优先/公平）、**哲学家进餐**（三方案分别破坏：限制人数=破坏请求保持、奇偶不同序拿=破坏循环等待、双筷可用才拿=破坏请求保持）、吸烟者。
- **管程**：**溯源**——PV 分散在程序各处、极易写错 → 管程把共享数据+操作**封装**，**编译器保证同一时刻仅一个进程在管程内**（自动互斥），配条件变量 wait/signal。
- **[联动计组]**：阻塞/唤醒 = 进程状态切换，最终落到寄存器上下文保存（链 2）。

## 2.7 死锁（大题：银行家）

- **四个必要条件（同时成立）**：互斥、不可剥夺、请求并保持、循环等待。
- **处理策略**：
  - **预防**：破坏四条件之一（破坏循环等待 = **资源有序分配**，最常用）。
  - **避免**：**银行家算法**——`Need = Max − Allocation`，每步找 `Need ≤ Available` 的进程加入安全序列，找得到则**安全状态**。
  - **检测+解除**：**资源分配图化简**（能完全化简 = 无死锁）；解除靠剥夺/撤销/回滚。
- **陷阱**：死锁一定有循环等待，但**有环不一定死锁**（多实例资源）；避免 ≠ 预防（避免是运行时动态判断）。
- **[联动计组]**：死锁是纯 OS 资源逻辑问题，但与 Cache 抖动、颠簸并列为 [[408强化总纲]] 故障溯源六大坑之一。

# 第3章 内存管理（S，大题核心，链路 3 OS 侧）

## 3.1 装入与链接（链路 1 OS 侧）

- **装入三方式**：绝对装入（编译期定物理址，仅单道）→ **可重定位装入（静态重定位）**（装入期改址，必须一次连续装完、运行中不能移动）→ **动态重定位**（运行期靠**重定位寄存器**实时加，**支持换入换出/虚存**）。
- **链接三方式**：静态链接 / 装入时动态链接 / 运行时动态链接（用到才链，省内存、便于共享）。
- **[联动计组]**：重定位寄存器 = 计组**基址寄存器 BR**，由 OS 在 PCB 中设置（[[计算机组成原理强化]] 4.3）。

## 3.2 内存扩充：覆盖 vs 交换

- **覆盖**：**同一进程内**，把不会同时用的模块共用一个覆盖区，**由程序员声明覆盖结构**（已淘汰）。
- **交换（对换）**：**进程之间**，把暂不运行的整个进程换到外存对换区（=中级调度，进程进**挂起态**）。
- **衍生**：二者都治标 → **虚拟内存**才是根治（3.5）。

## 3.3 连续分配与碎片

- 单一连续（内部碎片）→ 固定分区（**内部碎片**）→ **动态分区**（**外部碎片**，靠**紧凑/拼接**回收，需动态重定位支持）。
- **动态分区算法**：首次适应（**综合最好最快**，低址累积小碎片）/ 邻近适应（从上次位置找，易破坏低址大块）/ 最佳适应（留下大量**小外部碎片**）/ 最坏适应（大块快用完）。
- **陷阱**：**内部碎片**=分给进程但用不完（固定分区/分页）；**外部碎片**=空闲但太小装不下（动态分区/分段）。

## 3.4 分页（基本，链路 3 地基）

- **溯源**：连续分配有外部碎片、大程序装不下 → **离散分配**：内存切**页框**、进程切**页**，页可散落。
- **落地**：逻辑地址 = **页号 + 页内偏移**（偏移位数由页大小定）；**基本地址变换**：页号 ×页表项长 + 页表基址 → 页表项 → 页框号 || 偏移 = 物理址；**越界检查**（页号 ≥ 页表长度则中断）。
- **短板 1**：每次访问要**两次访存**（查页表 + 取数）→ 衍生 **快表 TLB**。
- **短板 2**：单级页表**占内存巨大** → 衍生 **多级页表**（页表离散存储、按需建立）。
- **[联动计组]**：页表项的页框号拼偏移由 **MMU** 硬件完成；二次访存的"慢"正是存储墙（[[408强化总纲]] 链 3）。

## 3.5 快表 TLB + 多级页表

- **快表 TLB**：地址变换高速缓存（相联存储器），命中则**一次访存**取数。**短板**：容量小；**进程切换时 TLB 整体失效需清空**（用户笔记重点）→ 衍生 ASID。
- **多级页表**：顶级页表常驻、下级按需。**短板**：**N 级 → N+1 次访存**（无快表时）。
- **陷阱（必算）**：含快表的 **有效访问时间 EAT**——`EAT = 命中率×(查TLB+访存) + (1−命中率)×(查TLB+查页表+访存)`；多级时把"查页表"展开为多次。
- **[联动计组]**：TLB↔Cache 都是"硬件加速、对程序透明"，但 **TLB miss ≠ 缺页 ≠ Cache miss**（[[408强化总纲]] 链 3 三铁律）。

## 3.6 分段与段页式

- **分段**：按逻辑模块分（代码/数据/栈），**二维地址（段号 + 段内）**，段表存基址+段长；**便于共享与保护**（按段权限）。**短板**：段长不定 → **外部碎片**。
- **分页 vs 分段**：页是物理单位、对用户透明、一维；段是逻辑单位、用户可见、二维、利于共享保护。
- **段页式**：先分段再分页 → 无外部碎片 + 便于共享。**短板**：**一次访问要 3 次访存**（段表 → 页表 → 取数），靠 TLB 缓解。

## 3.7 虚拟内存（请求分页，大题压舱）

- **溯源**：覆盖/交换治标，且程序有**局部性** → **请求分页**：只装当前需要的页，缺了再调（"虚拟"出大内存）。
- **落地**：页表项扩展——**状态位（在否内存）/访问位/修改(脏)位/外存地址**；缺页 = **缺页中断（fault）** → OS 调页 → 改页表 → **重新执行原指令**。
- **页面置换算法**：

| 算法 | 特点 | 坑 |
|---|---|---|
| OPT | 最优 | **不可实现**（需未来串）|
| FIFO | 简单 | **有 Belady 异常**（页框增反缺页增）|
| LRU | 近似 OPT、性能好 | 开销大（需栈/时间戳硬件）|
| Clock/改进 Clock | LRU 低成本近似 | 改进 Clock 看(访问位,修改位)优先淘汰(0,0) |

- **分配策略**：固定/可变分配 × 局部/全局置换（**固定分配局部置换、可变分配全局置换、可变分配局部置换**三种组合；**固定分配全局置换不存在**）。
- **故障：抖动 Thrashing** —— 页框 < 工作集，缺页率飙升。**衍生**：**工作集模型**（驻留集 ≥ 工作集）、缺页率 PFF 调节（[[408强化总纲]] 故障 1）。
- **[联动计组]**：缺页处理要读磁盘（链 4/5）；EAT 含缺页时叠加 ms 级磁盘 IO，对缺页率 p 极敏感。

# 第4章 文件系统（A，选择高频、综合可出，链路 5 OS 侧）

## 4.1 逻辑结构 vs 物理结构

- **逻辑结构**（用户视角）：无结构（字节流，如 txt）/ 有结构（记录式：顺序/索引/索引顺序文件）。
- **物理结构**（磁盘视角）：文件块 ↔ 磁盘块的分配方式（4.3）。
- **陷阱**：逻辑与物理**正交**——逻辑顺序文件可用链式物理存储。

## 4.2 目录、FCB 与索引结点 inode

- **溯源**：早期 FCB 把文件全部属性 + 文件名放目录项 → 检索目录时要反复读入大量属性，慢 → **索引结点 inode**：把**除文件名外**的属性都搬进 inode，**目录项只剩"文件名 + inode 指针"** → 一个盘块能放更多目录项、**加快按名检索**。
- **落地**：目录结构（单级/两级/树形/无环图）；FCB；硬/软链接（4.5）。
- **[联动计组]**：目录检索 = 多次磁盘 IO（链 5），是 open 慢的根源。

## 4.3 物理结构（缺陷迭代，链路 5 核心）

| 方式 | 溯源/优点 | 短板 → 衍生 |
|---|---|---|
| **连续分配** | 随机访问快、顺序读快 | **外部碎片、文件难增长** → 链接 |
| **隐式链接** | 解决碎片、易增长 | **不能随机访问、指针损坏全断** → 显式 |
| **显式链接 FAT** | 指针集中成 **FAT 表常驻内存**、可随机访问 | **FAT 占内存、大盘 FAT 巨大** → 索引 |
| **索引分配** | 索引块记录所有物理块、随机+增长 | **小文件浪费整索引块、大文件单块装不下** → 混合 |
| **多级/混合索引(inode)** | 直接块+一/二/三级间接块，小文件省、大文件撑 | 间接块多则访盘次数增 |

- **落地（必算）**：① 索引表项**隐含逻辑块号**（顺序存、每项定长、起址+偏移算出，用户笔记重点）；② **混合索引**算"最大文件大小"、"访问第 K 逻辑块需几次磁盘 IO"（看落在直接/一级/二级间接）。
- **陷阱**：小文件用多级索引反而**多次访盘**（用户笔记：1KB 文件两层索引要 3 次读盘）。

## 4.4 文件操作与打开文件表

- **open**：按路径**逐级目录检索（按名查找）**→ 找到 FCB/inode → 装入**系统级打开文件表（全局，含 inode、打开计数）** + **进程级打开文件表（fd → 系统表项指针、读写指针、权限）** → 返回 **fd**。
- **read/write**：用 **fd** 查表直达 inode，**不再按名检索**（这是 open 的全部价值）。
- **delete**：删目录项 → inode 链接计数 −1 → 计数为 0 才回收 inode + 数据块（改空闲管理结构）。
- **[联动计组]**：read 真正读盘走链 4（DMA + 缓冲 + 中断）。

## 4.5 文件共享与保护

- **硬链接**：多个目录项**共享同一 inode**，inode 存**链接计数**，删到 0 才真删；不能跨文件系统。
- **软链接（符号链接）**：独立文件，内容是**目标路径**；目标删除则**悬空失效**；可跨文件系统。
- **保护**：口令 / 加密 / **访问控制表 ACL**（最小权限原则）。

## 4.6 空闲空间管理 + 文件系统布局

- **空闲管理**：空闲表法（连续）/ 空闲链表法（盘块/盘区链）/ **位示图**（字号位号 ↔ 块号公式互算，必考）/ **成组链接（Unix）**（超级块记录一组空闲块 + 下组指针；超级块在内存，分配/回收只动内存中那组，省 IO）。
- **布局**：**物理格式化（低级）**分扇区+校验 → **逻辑格式化（高级）**建文件系统、根目录、空闲管理 → 外存结构（引导块 / 超级块 / inode 区 / 数据区）。
- **VFS**：屏蔽不同文件系统差异，提供统一接口（open/read/write）。
- **[联动计组]**：位示图/块号计算最终落到磁盘 CHS 物理寻址（[[计算机组成原理强化]] 3.6）。

# 第5章 IO 管理（A，与计组第7章合看，链路 4 OS 侧）

## 5.1 IO 控制方式四级演化（= 链路 4，与计组同源）

| 方式 | 前级短板 → 它 | CPU 参与 | 残留 → 衍生 |
|---|---|---|---|
| 程序直接控制(轮询) | 最原始 | **忙等**全程、串行 | 利用率低 → 中断 |
| 中断驱动 | 消除忙等 | **每字/字节**中断 | 中断频繁 → DMA |
| DMA | 成块直传不经 CPU | 仅传送**前后** | 争总线/一致性 → 通道 |
| 通道 | IO 处理机执行通道程序 | 几乎不管 | 硬件复杂 |

- **[联动计组]**：硬件通路（DMA 控制器/总线/中断）见 [[计算机组成原理强化]] 第7章；OS 管策略，计组管通路。

## 5.2 IO 软件四层（自顶向下）

1. **用户层**：库函数、**SPOOLing**。
2. **设备独立性软件**：**逻辑设备名 → 物理设备（LUT 表）**、设备保护、差错处理、缓冲管理、设备分配。
3. **设备驱动程序**：把抽象 IO 请求翻译成设备寄存器操作（厂商相关）。
4. **中断处理程序**：唤醒阻塞进程、搬数据。
- **陷阱**：**设备独立性** = 用户用逻辑设备名、不绑定具体物理设备 → 提高可移植与利用率。

## 5.3 缓冲与 SPOOLing

- **缓冲**：**溯源**——CPU 与设备速度不匹配、中断太频、生产消费需解耦 → 单缓冲（`单缓冲处理一块 ≈ max(C,T)+M`）→ 双缓冲（可边传边算）→ 循环缓冲 → **缓冲池**（系统级、按收发用途动态分配）。
- **SPOOLing（假脱机）**：**溯源**——独占设备（打印机）利用率低 → 用**输入井/输出井 + 输入/输出进程**把独占设备**虚拟成共享设备**（多用户"同时"打印，实际排队）。
- **[联动计组]**：缓冲区 = 内核内存，DMA 先把数据搬到内核缓冲区再拷用户区（链 4）。

## 5.4 设备分配

- **数据结构**：DCT（设备控制表）、COCT（控制器）、CHCT（通道）、SDT（系统设备表）。
- **安全分配 vs 不安全分配**（不安全需配死锁避免）；**LUT** 实现逻辑→物理映射。

## 5.5 磁盘调度（必算磁头移动距离）

| 算法 | 思想 | 短板 |
|---|---|---|
| FCFS | 先来先服务 | 距离可能很大 |
| SSTF | 就近最短寻道 | **可能饥饿**（远磁道） |
| **SCAN（电梯）** | 单向扫到**头**再回 | 对刚过位置不公平 |
| LOOK | SCAN 改进，到**最远请求**就回 | — |
| **C-SCAN** | 单向服务，回程**不服务**直接快退 | 平均更公平 |
| C-LOOK | C-SCAN + LOOK | 最常用之一 |

- **落地**：`一次读写时间 = 寻道 + 旋转延迟(平均½圈) + 传输`；减少延迟：**扇区交替编号（间隔因子）、相邻盘面错位命名、磁盘臂调度**。
- **[联动计组]**：磁头移动 = 物理寻道（[[计算机组成原理强化]] 3.6），调度算法优化的是寻道时间。

## 5.6 磁盘管理（B）

- 物理格式化（分扇区/ECC）→ 逻辑格式化（建 FS）；**引导块**（含引导程序，链 1 起点之一）；**坏块处理**（屏蔽/备用扇区替换）。
