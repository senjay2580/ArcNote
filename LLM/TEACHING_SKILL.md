# Karpathy 式 ML/DL 教学 Skill

> 基于 Andrej Karpathy 的 "Neural Networks: Zero to Hero" 教学方法论提炼
> 适用于：AI 辅助教学 ML/DL 知识点

---

## 核心哲学

**"What I cannot create, I do not understand." — Richard Feynman**

教学的目标不是让学生"感觉学到了"，而是让学生**能从零构建出来**。如果不能用代码实现，就没有真正解释清楚。

---

## 15 条教学规则

### 规则 1：代码先行，理论跟随
永远不要从理论开始。顺序是：
1. 给一个具体例子（带真实数字）
2. 跑代码看到结果
3. 然后解释为什么
4. 最后（可选）给数学公式

### 规则 2：先做最笨的版本
在给出优雅方案之前，永远先实现一个暴力/笨办法版本。
- 自注意力：先用 for 循环做平均，再矩阵乘法，再 softmax 加权，最后才引入 Q/K/V
- 反向传播：先用数值梯度 `(f(x+h)-f(x))/h`，再推解析梯度
- 语言模型：先用字典计数做 bigram，再用神经网络做同样的事

### 规则 3：每次只改一个东西
保持问题不变只换方法，或保持方法不变只换问题。绝不同时改两个变量。

### 规则 4：数值验证一切
任何解析结果都必须用数值计算交叉验证：
- 解析梯度 vs 数值梯度
- 手写实现 vs PyTorch 内置实现
- 两个数字必须匹配，否则有 bug

### 规则 5：可视化中间状态
不要只展示输入和输出，要展示中间发生了什么：
- 计算图可视化
- 每层激活值的分布
- 梯度的统计量
- 损失曲线

### 规则 6：从标量开始
先在最简单的标量维度讲清楚概念，再扩展到向量/矩阵/张量。去掉所有不必要的维度。

### 规则 7：搭完手工版再展示框架版
从零构建后，展示 PyTorch 等框架的等价实现，验证结果一致。桥接教学与实战。

### 规则 8：物理类比先于数学公式
每个数学概念出现前，先给一个日常/物理类比：

| 概念 | 类比 |
|------|------|
| 反向传播 | "在输出端施加一个力，这个力通过整个电路传回输入端" |
| 梯度下降 | "蒙眼站在山上，用脚感受坡度，往低处走" |
| 链式法则 | "汽车速度是自行车的 2 倍，自行车是步行的 3 倍，所以汽车是步行的 6 倍" |
| 正则化 | "参数上挂了弹簧，拉回零点" |
| 自注意力 | "每个字广播一个请求：谁能帮我预测下一个？其他字用钥匙回应" |

### 规则 9：版本递进，显式标注
复杂机制分解为多个版本（v0→v1→v2→...），每一步只新增一个想法。

### 规则 10：设置验证检查点
关键节点必须验证：
- 初始化时损失是否等于 `-log(1/类别数)`？
- 能否在单个 batch 上过拟合到 loss≈0？
- 解析梯度和数值梯度是否匹配？

### 规则 11：展示错误，不要隐藏
犯错和调试过程是教学的一部分。展示思考过程，不是只给最终修复。

### 规则 12：同一数据集贯穿一个概念序列
不要中途换数据集。数据不变，只变模型——这样学习者只需吸收一个新变量。

### 规则 13：明确前置知识
每节课开头声明：本课假设你已经知道 X 和 Y。

### 规则 14：代码就是讲义
不要用幻灯片做教学的主体。Jupyter Notebook 的 cell 进展就是教学结构。

### 规则 15：按需学习，深度优先
不要广度优先扫知识点。选一个具体项目，一路深入，遇到什么学什么。

## 教学节奏模板

每讲一个新概念，按以下节奏：

```
1. 提出问题（你想解决什么）
2. 直觉类比（日常生活中的对应）
3. 最笨的代码实现（暴力版，v0）
4. 运行 → 看结果 → 打印中间状态
5. 指出 v0 的问题（慢/不准/不能扩展）
6. 引入改进（v1），只加一个新想法
7. 运行 → 对比 v0 和 v1 结果（数值验证）
8. 继续改进（v2, v3...），每步一个新想法
9. 最终版本 → 对比 PyTorch 框架实现
10. 数学公式总结（这时候学生已经理解了，公式只是记号）
```

## 关键反模式（绝不能做的事）

| 反模式 | 为什么不行 |
|--------|-----------|
| 先讲 30 分钟理论再写代码 | 学生在理论阶段就丢失了，代码变成抄写 |
| 一次性展示 100 行完整代码 | 学生无法消化，不知道哪行是关键 |
| 跳过数值验证 | 学生可能代码跑通了但理解是错的 |
| 只展示最终正确版本 | 学生不知道为什么这样设计，不知道其他方案为什么不行 |
| 用"显然"、"容易看出"等措辞 | 对学生来说一点都不显然 |
| 同时改数据和模型 | 无法隔离变量，不知道效果来自哪里 |
