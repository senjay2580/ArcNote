## 📦 依赖配置

### 1. Maven 依赖

在你的 `pom.xml` 中添加:
```xml
<!-- Spring Boot Actuator (暴露监控端点) -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

<!-- Micrometer Prometheus Registry (Prometheus 指标格式) -->
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>

<!-- HikariCP 连接池监控(Spring Boot 3默认已包含) -->
<dependency>
    <groupId>com.zaxxer</groupId>
    <artifactId>HikariCP</artifactId>
</dependency>
```

### 2. application.yml 配置

在 `application.yml` 中添加:
```yaml
# ==================== 监控配置 ====================
spring:
  application:
    name: archat-backend  # 应用名称(会作为指标标签)

management:
  endpoints:
    web:
      exposure:
        include: '*'  # 暴露所有端点(生产环境建议指定具体端点)
      base-path: /actuator  # 端点基础路径
  
  endpoint:
    health:
      show-details: always  # 健康检查显示详情
    prometheus:
      enabled: true  # 启用 Prometheus 端点
  
  metrics:
    tags:
      application: ${spring.application.name}  # 全局标签
      env: ${spring.profiles.active:dev}       # 环境标签
    
    # 启用各类指标
    enable:
      jvm: true           # JVM 指标
      system: true        # 系统指标
      process: true       # 进程指标
      tomcat: true        # Tomcat 指标
      hikaricp: true      # HikariCP 连接池指标
      logback: true       # 日志指标
    
    distribution:
      percentiles-histogram:
        http.server.requests: true  # HTTP 请求直方图
      slo:
        http.server.requests: 100ms,200ms,500ms,1s,2s  # 延迟 SLO
```

## 📊 默认指标详解

### 1. JVM 指标

#### 内存指标
```
# 堆内存使用量
jvm_memory_used_bytes{area="heap"}

# 非堆内存使用量
jvm_memory_used_bytes{area="nonheap"}

# 最大内存
jvm_memory_max_bytes

# 已提交内存
jvm_memory_committed_bytes
```

**Grafana 查询示例**:
```promql
# 堆内存使用率
jvm_memory_used_bytes{area="heap"} / jvm_memory_max_bytes{area="heap"} * 100
```

#### GC 指标
```
# GC 次数
jvm_gc_pause_seconds_count

# GC 总耗时
jvm_gc_pause_seconds_sum

# GC 平均耗时
jvm_gc_pause_seconds_sum / jvm_gc_pause_seconds_count
```

**告警示例**:
```yaml
- alert: HighGCTime
  expr: rate(jvm_gc_pause_seconds_sum[5m]) > 0.1  # GC耗时占比>10%
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "{{ $labels.application }} GC耗时过高"
```

#### 线程指标
```
# 活跃线程数
jvm_threads_live_threads

# 守护线程数
jvm_threads_daemon_threads

# 峰值线程数
jvm_threads_peak_threads
```

### 2. HTTP 请求指标

```
# 请求总数
http_server_requests_seconds_count{uri="/api/users",method="GET",status="200"}

# 请求总耗时
http_server_requests_seconds_sum{uri="/api/users"}

# 请求最大耗时
http_server_requests_seconds_max{uri="/api/users"}
```

**计算 QPS**:
```promql
sum(rate(http_server_requests_seconds_count[1m])) by (uri)
```

**计算平均响应时间**:
```promql
rate(http_server_requests_seconds_sum[5m]) / rate(http_server_requests_seconds_count[5m])
```

**计算错误率**:
```promql
sum(rate(http_server_requests_seconds_count{status=~"5.."}[5m])) /
sum(rate(http_server_requests_seconds_count[5m])) * 100
```

### 3. 数据库连接池指标(HikariCP)

```
# 活跃连接数
hikaricp_connections_active{pool="HikariPool-1"}

# 空闲连接数
hikaricp_connections_idle{pool="HikariPool-1"}

# 总连接数
hikaricp_connections{pool="HikariPool-1"}

# 等待连接的线程数
hikaricp_connections_pending{pool="HikariPool-1"}

# 连接超时次数
hikaricp_connections_timeout_total{pool="HikariPool-1"}

# 连接获取耗时
hikaricp_connections_acquire_seconds{pool="HikariPool-1"}
```

**告警示例**:
```yaml
- alert: DatabaseConnectionPoolExhausted
  expr: hikaricp_connections_active / hikaricp_connections >= 0.9
  for: 5m
  labels:
    severity: critical
  annotations:
    summary: "数据库连接池即将耗尽"
    description: "活跃连接数占比: {{ $value }}%"
```

## 🎨 自定义业务指标

### 1. 创建指标工具类

```java
package com.senjay.archat.common.metrics;

import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import io.micrometer.core.instrument.Timer;
import org.springframework.stereotype.Component;

import java.util.concurrent.TimeUnit;

/**
 * 自定义业务指标工具类
 */
@Component
public class BusinessMetrics {

    private final MeterRegistry meterRegistry;

    // 用户注册计数器
    private final Counter userRegisterCounter;
    
    // 订单创建计数器
    private final Counter orderCreateCounter;
    
    // 支付成功计数器
    private final Counter paymentSuccessCounter;
    
    // 支付失败计数器
    private final Counter paymentFailCounter;
    
    // AI 对话计时器
    private final Timer aiChatTimer;

    public BusinessMetrics(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        
        // 初始化计数器
        this.userRegisterCounter = Counter.builder("business.user.register")
            .description("用户注册总数")
            .tag("type", "register")
            .register(meterRegistry);
        
        this.orderCreateCounter = Counter.builder("business.order.create")
            .description("订单创建总数")
            .register(meterRegistry);
        
        this.paymentSuccessCounter = Counter.builder("business.payment.success")
            .description("支付成功总数")
            .register(meterRegistry);
        
        this.paymentFailCounter = Counter.builder("business.payment.fail")
            .description("支付失败总数")
            .register(meterRegistry);
        
        // 初始化计时器
        this.aiChatTimer = Timer.builder("business.ai.chat.duration")
            .description("AI对话耗时")
            .publishPercentiles(0.5, 0.95, 0.99)  // P50, P95, P99
            .register(meterRegistry);
    }

    /**
     * 记录用户注册
     */
    public void recordUserRegister() {
        userRegisterCounter.increment();
    }

    /**
     * 记录订单创建
     * @param orderType 订单类型
     * @param amount 订单金额
     */
    public void recordOrderCreate(String orderType, Double amount) {
        Counter.builder("business.order.create")
            .tag("order_type", orderType)
            .register(meterRegistry)
            .increment();
        
        // 记录订单金额(Gauge类型)
        meterRegistry.gauge("business.order.amount", amount);
    }

    /**
     * 记录支付结果
     * @param success 是否成功
     * @param paymentMethod 支付方式
     */
    public void recordPayment(boolean success, String paymentMethod) {
        if (success) {
            Counter.builder("business.payment.success")
                .tag("payment_method", paymentMethod)
                .register(meterRegistry)
                .increment();
        } else {
            Counter.builder("business.payment.fail")
                .tag("payment_method", paymentMethod)
                .register(meterRegistry)
                .increment();
        }
    }

    /**
     * 记录 AI 对话耗时
     * @param durationMs 耗时(毫秒)
     * @param modelName AI 模型名称
     */
    public void recordAIChatDuration(long durationMs, String modelName) {
        Timer.builder("business.ai.chat.duration")
            .tag("model", modelName)
            .register(meterRegistry)
            .record(durationMs, TimeUnit.MILLISECONDS);
    }

    /**
     * 记录当前在线用户数(Gauge类型)
     * @param count 在线用户数
     */
    public void recordOnlineUsers(int count) {
        meterRegistry.gauge("business.online.users", count);
    }
}
```

    }

### 2. 自定义 Redis 监控

```java
package com.senjay.archat.common.metrics;

import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import io.micrometer.core.instrument.Timer;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Component;

@Component
public class RedisMetrics {

    private final MeterRegistry meterRegistry;
    private final RedisTemplate<String, Object> redisTemplate;

    public RedisMetrics(MeterRegistry meterRegistry, RedisTemplate<String, Object> redisTemplate) {
        this.meterRegistry = meterRegistry;
        this.redisTemplate = redisTemplate;
    }

    /**
     * 记录缓存命中
     */
    public void recordCacheHit(String cacheName) {
        Counter.builder("redis.cache.hit")
            .tag("cache", cacheName)
            .register(meterRegistry)
            .increment();
    }

    /**
     * 记录缓存未命中
     */
    public void recordCacheMiss(String cacheName) {
        Counter.builder("redis.cache.miss")
            .tag("cache", cacheName)
            .register(meterRegistry)
            .increment();
    }

    /**
     * 计算缓存命中率
     */
    public void calculateHitRate() {
        // 通过 Prometheus 查询计算:
        // redis_cache_hit / (redis_cache_hit + redis_cache_miss) * 100
    }
}
```

**使用示例**:
```java
@Service
public class CacheService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private RedisMetrics redisMetrics;

    public User getUserById(Long userId) {
        String cacheKey = "user:" + userId;
        
        // 尝试从缓存获取
        User user = (User) redisTemplate.opsForValue().get(cacheKey);
        
        if (user != null) {
            // 缓存命中
            redisMetrics.recordCacheHit("user");
            return user;
        }
        
        // 缓存未命中
        redisMetrics.recordCacheMiss("user");
        
        // 从数据库查询
        user = userMapper.selectById(userId);
        
        // 写入缓存
        redisTemplate.opsForValue().set(cacheKey, user, 1, TimeUnit.HOURS);
        
        return user;
    }
}
```

## 📈 MyBatis/MyBatis-Plus 监控

### 1. 慢查询监控

```java
package com.senjay.archat.common.metrics;

import com.baomidou.mybatisplus.extension.plugins.inner.InnerInterceptor;
import io.micrometer.core.instrument.Counter;
import io.micrometer.core.instrument.MeterRegistry;
import io.micrometer.core.instrument.Timer;
import org.apache.ibatis.executor.Executor;
import org.apache.ibatis.mapping.MappedStatement;
import org.apache.ibatis.session.ResultHandler;
import org.apache.ibatis.session.RowBounds;
import org.springframework.stereotype.Component;

import java.util.concurrent.TimeUnit;

/**
 * MyBatis-Plus 慢查询拦截器
 */
@Component
public class SlowQueryInterceptor implements InnerInterceptor {

    private final MeterRegistry meterRegistry;
    private final Counter slowQueryCounter;
    private static final long SLOW_QUERY_THRESHOLD = 1000; // 1秒

    public SlowQueryInterceptor(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        this.slowQueryCounter = Counter.builder("mybatis.slow.query")
            .description("慢查询次数")
            .register(meterRegistry);
    }

    @Override
    public void beforeQuery(Executor executor, MappedStatement ms, 
                           Object parameter, RowBounds rowBounds, 
                           ResultHandler resultHandler) throws Exception {
        // 记录开始时间
        long startTime = System.currentTimeMillis();
        
        // 执行查询
        InnerInterceptor.super.beforeQuery(executor, ms, parameter, rowBounds, resultHandler);
        
        // 计算耗时
        long duration = System.currentTimeMillis() - startTime;
        
        // 记录指标
        Timer.builder("mybatis.query.duration")
            .tag("mapper", ms.getId())
            .register(meterRegistry)
            .record(duration, TimeUnit.MILLISECONDS);
        
        // 慢查询计数
        if (duration > SLOW_QUERY_THRESHOLD) {
            slowQueryCounter.increment();
            // 打印慢查询日志
            System.err.println("慢查询: " + ms.getId() + " 耗时: " + duration + "ms");
        }
    }
}
```

### 2. 注册拦截器

```java
@Configuration
public class MybatisPlusConfig {

    @Autowired
    private SlowQueryInterceptor slowQueryInterceptor;

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(slowQueryInterceptor);
        return interceptor;
    }
}
```

## 🔔 健康检查配置

### 1. 自定义健康检查

```java
package com.senjay.archat.common.health;

import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Component;

/**
 * Redis 健康检查
 */
@Component
public class RedisHealthIndicator implements HealthIndicator {

    private final RedisTemplate<String, Object> redisTemplate;

    public RedisHealthIndicator(RedisTemplate<String, Object> redisTemplate) {
        this.redisTemplate = redisTemplate;
    }

    @Override
    public Health health() {
        try {
            // 执行 PING 命令
            String pong = redisTemplate.getConnectionFactory()
                .getConnection()
                .ping();
            
            if ("PONG".equals(pong)) {
                return Health.up()
                    .withDetail("redis", "可用")
                    .withDetail("response", pong)
                    .build();
            } else {
                return Health.down()
                    .withDetail("redis", "异常")
                    .withDetail("response", pong)
                    .build();
            }
        } catch (Exception e) {
            return Health.down()
                .withDetail("redis", "不可用")
                .withException(e)
                .build();
        }
    }
}
```

## 🐛 常见问题

### 问题1: /actuator/prometheus 返回 404

**原因**: 未引入 `micrometer-registry-prometheus` 依赖

**解决**:
```xml
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-registry-prometheus</artifactId>
</dependency>
```

### 问题2: 指标数据为空

**原因**: Prometheus 未配置采集目标

**排查**:
1. 检查 `prometheus.yml` 中是否添加了 job
2. 访问 `http://localhost:9090/targets` 查看状态
3. 检查应用端口是否正确

### 问题3: Docker 容器内 Prometheus 无法访问应用

**原因**: 网络隔离

**解决**: 确保在同一 Docker 网络
```yaml
networks:
  - monitoring
  - default  # 连接到原有网络
```
