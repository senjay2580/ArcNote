## Part 0 · TL;DR 五句话

1. **上位机 = 大脑决策；下位机 = 脊髓执行；IO = 手脚；通讯 = 神经；ROS = 普通话。**
2. Arduino 是玩具，**STM32 是工业界下位机主力**，ESP32 = STM32 + WiFi/蓝牙。
3. **树莓派系 ≠ 单片机派系**。树莓派/香橙派/Jetson 都是跑 Linux 的"微型电脑"（上位机）；STM32/ESP32 是"裸机微控制器"（下位机）。两者不是竞争关系，是**搭档**。
4. ROS2（不是 ROS1）是 2026 年机器人系统的行业标准，跑在上位机上。
5. PLC / PCS 是工厂流水线的独立王国，**和机器人/嵌入式不是一条路**。

## Part 1 · 主干认知地图

### 1.1 核心类比：机器人 = 数字化的人

| 人体部位 | 对应技术 | 在干嘛 |
|---|---|---|
| 🧠 大脑皮层 | **上位机**（PC / 工控机 / Jetson / 树莓派） | 跑 AI / 视觉 / 路径规划 / HMI |
| 🧬 小脑-脊髓 | **下位机**（STM32 / Arduino / ESP32 / PLC） | 实时控制电机 / 读传感器 / 发脉冲 |
| 🛰️ 神经 | **通讯总线**（串口 / CAN / 以太网 / ROS topic） | 上下位机对话 |
| 👁️ 五感 | **传感器**（IO 的 Input 部分） | 编码器 / IMU / 摄像头 / 雷达 |
| 💪 肌肉 | **执行器**（IO 的 Output 部分） | 电机 / 气缸 / 舵机 / 继电器 |
| 🗣️ 语言 | **ROS / ROS2** | 给所有模块制定统一对话协议 |

### 1.2 主干数据流

```
人类操作者
    │ 点击界面 / 发指令
    ▼
┌────────────────┐
│ 上位机（大脑） │  ← Python / C++ / ROS2
│ PC · Jetson    │    做视觉、规划、调度
└────────────────┘
    │ 串口 / CAN / 以太网（神经）
    ▼
┌────────────────┐
│ 下位机（脊髓） │  ← C / C++ 裸机 or RTOS
│ STM32 / PLC    │    把指令翻译成 PWM / 脉冲
└────────────────┘
    │ GPIO / IO 口
    ▼
┌──────────────────────┐
│ 驱动器 + 电机 + 传感器 │  ← 真正动起来的东西
└──────────────────────┘
    │ 编码器反馈
    └───────────────► 回传到上位机，形成闭环
```

## Part 2 · 下位机"微控制器"派系

这层芯片的特点：**没有操作系统（或只跑 RTOS），不跑 Linux，直接裸机控硬件，实时性好，功耗低**。

### 2.1 三巨头对比（核心差异）

| 维度 | Arduino | STM32 | ESP32 |
|---|---|---|---|
| **定位** | 教学 / DIY 玩具 | 工业级下位机主力 | IoT 专用芯（自带无线） |
| **主频** | 16 MHz（UNO） | 48–480 MHz | 双核 240 MHz |
| **架构** | AVR 8 位为主 | ARM Cortex-M | Xtensa LX6/LX7 |
| **无线** | 无（需外接） | 大多无（需外接） | **自带 WiFi + 蓝牙** |
| **开发环境** | Arduino IDE（傻瓜） | Keil / STM32CubeIDE / VS Code | Arduino IDE / PlatformIO / ESP-IDF / MicroPython |
| **学习曲线** | 最平 | 陡（HAL 库 + 时钟树） | 中等 |
| **产品量产** | 几乎不用 | **90% 工业产品的选择** | IoT 产品的默认选择 |
| **价格** | ~30 元 | 核心芯片 5–50 元 | ~15 元 |

**一句话判断**：
- 上课 / 毕设糊弄 → Arduino
- 做真东西 / 求职敲门砖 → **STM32**
- 需要 WiFi/蓝牙 / 智能家居 / BCI 便携设备 → **ESP32**

### 2.2 容易混淆的其它小字辈

| 名字 | 一句话定位 |
|---|---|
| **51 单片机** | 教学鼻祖，现在只用来入门打地基，实际产品基本淘汰 |
| **GD32** | 国产 STM32 兼容芯片，引脚代码通用，供应链不卡脖子 |
| **STM8** | STM32 的 8 位弱化版，用于极简成本场景 |
| **K210** | 国产 AI 视觉芯片（嘉楠），比 Arduino 强但生态小 |
| **OpenMV** | 基于 STM32H7 的"傻瓜机器视觉相机"，拍照 + 识别一体 |

## Part 3 · 上位机 "派" 家族（跑 Linux 的小电脑）

这层的特点：**能跑 Linux / Android，可以编程 Python、装 OpenCV、跑深度学习模型，功耗中等，实时性一般**。

### 3.1 核心三兄弟（2026 视角）

| 型号 | 厂商 | 定位 | 算力 | 内存 | 价格 | 典型用途 |
|---|---|---|---|---|---|---|
| **树莓派 5 / 4B** | 英国 Raspberry Pi 基金会 | 通用小电脑，**生态王** | 弱（CPU 强，无独立 AI 算力） | 4–8 GB | 400–800 | 教育、DIY、家庭服务器、轻量 AI（配 Hailo 加速器） |
| **Jetson Orin Nano / Nano Super** | NVIDIA | **边缘 AI 专用** | 强（40–67 TOPS） | 4–8 GB | 2000–3500 | 机器人视觉、实时推理、ROS2 首选上位机 |
| **香橙派 AI Pro / 5 Plus** | 深圳迅龙软件 | 国产高性价比 | 中–强（看型号） | 4–16 GB | 500–1500 | 国产替代、大内存需求、AI 初学 |

### 3.2 还有哪些"派"

| 名字 | 一句话 |
|---|---|
| **香蕉派 (Banana Pi)** | 国产老牌，生态不如树莓派但便宜 |
| **荔枝派 (Sipeed)** | 嘉楠主推，K210/LicheeRV 等 RISC-V 方向 |
| **Rock Pi** | 瑞芯微芯片为主，偏工业嵌入式 |
| **Lattepanda** | x86 架构的树莓派，可跑 Windows |
| **Jetson Xavier NX / AGX Orin** | Jetson 高端系列，机器人正式量产方案 |

### 3.3 选型决策树

```
任务是什么？
 ├─ 纯学 Linux / 搭网站 / 家庭NAS
 │    → 树莓派 5（生态最好）
 ├─ 实时控制电机、读传感器，不需要 AI
 │    → 根本别买"派"，直接 STM32 就够
 ├─ 要跑 YOLO / PyTorch 模型做机器人视觉
 │    → Jetson Orin Nano（有 CUDA，唯一正确选择）
 ├─ 国产 + 预算敏感 + AI 推理
 │    → 香橙派 AI Pro
 └─ 便携 BCI 头盔（你未来的场景）
      → ESP32 采集 + 手机/PC 上位机；不用"派"
```

### 3.4 派系 vs 单片机派系 —— 最重要的澄清

很多人学了一圈还分不清，**就记这张表**：

| 问题 | 树莓派 / Jetson / 香橙派 | STM32 / ESP32 / Arduino |
|---|---|---|
| 跑操作系统吗？ | ✅ Linux | ❌ 裸机 or RTOS |
| 能装 Python/PyTorch 吗？ | ✅ | ❌（只有 MicroPython 残血版） |
| 精确到微秒的实时控制？ | ❌（Linux 抢占不可靠） | ✅ |
| 功耗？ | 几瓦到几十瓦 | 毫瓦级 |
| 角色 | **上位机（大脑）** | **下位机（脊髓）** |
| 典型搭档 | 上位机搭配一块 STM32 做实时层 | 下位机搭配一块 Jetson 跑 AI |

**完整机器人里，两层都有，各司其职**。

## Part 4 · PLC / PCS —— 工厂自动化的"隔壁王国"

### 4.1 PLC (Programmable Logic Controller)
- **一句话**：工厂流水线上的"工业级 STM32"，皮实、傻瓜、专做开关/定时/顺控。
- 编程语言：**梯形图 (Ladder)**、结构化文本 ST、功能块图 FBD —— 电工也能写。
- 四大品牌：西门子（德国）、三菱（日）、欧姆龙（日）、汇川（国产崛起）。
- **岗位**：电气自动化工程师，传统制造业刚需。

### 4.2 PCS (Process Control System) = DCS
- **一句话**：化工厂、发电厂那种"几千个阀门 / 温度点"的大型过程控制系统。
- PLC 的放大强化版 + 冗余容错 + 工程师站 + SCADA。
- 品牌：霍尼韦尔、艾默生、横河、和利时、中控。
- **岗位**：化工 / 电力国企稳定岗。

### 4.3 PLC/PCS vs 机器人 —— 选边警示

| 维度 | PLC / PCS 世界 | STM32 / ROS 世界 |
|---|---|---|
| 编程语言 | 梯形图、ST | C / C++ / Python |
| 典型公司 | 制造业、国企、系统集成商 | 机器人初创、大厂、产品公司 |
| 薪资 | 稳定偏低 | 两极分化 |
| 加班 | 出差多（调试现场） | 研发加班 |
| 天花板 | 一般（除非做项目经理） | 高（尤其 AI+ 硬件） |

**⚠️ 警告**：这两个圈子几乎不流通，一旦选错 3 年就很难回头。

## Part 5 · ROS / ROS2 —— 机器人世界的"普通话"

### 5.1 ROS 是什么
**ROS 不是芯片、不是系统、不是框架** —— 它是**中间件 (Middleware)**，跑在上位机的 Linux 上，让不同模块用统一协议对话。

### 5.2 没有 ROS vs 有 ROS

```
没 ROS：每个厂家的相机/雷达/机械臂接口都不同，换一个就得重写一切

有 ROS：
  相机节点  ──┐
  雷达节点  ──┼──► topic (发布/订阅) ──► SLAM 节点 ──► 规划节点 ──► 底盘节点
  IMU 节点  ──┘
  （所有节点用同一套 pub/sub 协议对话，即插即换）
```

### 5.3 ROS1 vs ROS2（2026 就业必考）

| 维度 | ROS1（Noetic） | ROS2（Humble / Jazzy） |
|---|---|---|
| 底层协议 | 自家 TCPROS | **DDS**（工业标准） |
| 实时性 | 差 | 好，可量产 |
| 多机 | 单 master 脆弱 | 去中心化 |
| 支持状态 | Noetic 2025-05 已 EOL | **主流** |
| 行业态度 | 看老代码用 | 新项目全用 |
| 具身智能 | 不用 | **ROS2 + Zenoh + Isaac 是 2026 趋势** |

### 5.4 ROS2 学习姿势
- **主教材**：古月居《ROS2 入门 21 讲》（B 站 + 图文版 + Gitee 代码）
- **一键安装**：fishros 一键安装工具 + 鱼香 ROS《动手学 ROS2》
- **仿真环境**：Gazebo Harmonic / Ignition、NVIDIA Isaac Sim（人形机器人趋势）
- **项目路线**：小海龟 turtlesim → URDF 建模 → Gazebo 仿真 → SLAM 建图 → 导航避障 → 真机部署

## Part 6 · 协作机器人（Cobot）

**定义**：可与人共享工作空间、带力觉传感和安全模式的机械臂。

| 品牌 | 国别 | 地位 |
|---|---|---|
| UR（优傲） | 丹麦 | 鼻祖、全球第一 |
| JAKA 节卡 | 中国 | 国产头部 |
| 遨博 AUBO | 中国 | 工业场景强 |
| 新松 SIASUN | 中国 | 老牌国企 |
| FANUC CRX | 日本 | 传统工业巨头 |

**架构本质**：工控机（上位机）+ 伺服驱动（下位机）+ EtherCAT 通讯 + 安全 PLC。

**典型岗位**：协作机器人应用工程师 —— 不是造机械臂，而是用现成机械臂给客户做**视觉抓取、打磨、装配、码垛**方案。二次开发接口：厂商 SDK（Python/TCP）或 **ROS2 driver**。

## Part 7 · 自动化控制工程 · 学科全景

这是个**学科名**，伞下三个分支：

```
自动化控制工程
├── 工厂自动化 (FA)     → PLC + HMI + 伺服 + 传感器
├── 过程控制 (PCS/DCS)  → 化工 / 电力 / 冶金
└── 运动控制 / 机器人   → STM32 + ROS + 视觉 AI（新赛道）
```

大部分老牌学校里，这个专业**偏电气**；新学校和 TOP 高校开始往机器人 + AI 偏。

## Part 8 · 学习路线（完整版）

### 8.1 嵌入式主线（业界公认路线）

```
Stage 0  基础
         ├─ C 语言（必须扎实）
         ├─ 数字电路 / 模拟电路（电阻电容二极管够用）
         ├─ 数据结构（链表、队列、状态机必会）
         └─ 计算机组成原理（408 里有）

Stage 1  51 单片机（1–2 个月）          ← 只为打地基，别死磕
         └─ 目标：LED / 按键 / 串口 / 定时器 / 中断搞懂

Stage 2  STM32（3–6 个月）              ← 工业主力，重点
         ├─ 选板：正点原子 F103 或 F407（零基础最友好）
         ├─ 学 HAL 库 + CubeMX 工具
         └─ 做项目：小车 / 无人机 / 电机控制

Stage 3  RTOS（1–2 个月）               ← 产品级必备
         └─ FreeRTOS 或 RT-Thread（国产，越来越强）

Stage 4  嵌入式 Linux（6 个月–若干年）  ← 深水区
         ├─ 应用层（APP 开发）
         ├─ 驱动层（字符设备 / 平台驱动）
         └─ 内核层（裁剪 / 移植）

Stage 5  特化方向（任选）
         ├─ 边缘 AI（Jetson + TensorRT）
         ├─ 汽车电子（AUTOSAR / CAN 总线）
         ├─ IoT（ESP32 + MQTT + 云平台）
         └─ 机器人（ROS2 + SLAM + 导航）
```

## Part 10 · 与 BCI 主攻方向的交汇点

完整 BCI 系统（研究生以后会遇到）：

```
脑电帽（电极）
  │ 模拟信号
  ▼
放大/滤波芯片（TI ADS1299 是行业标准）
  │ 数字信号
  ▼
ESP32 / STM32（下位机：采样 + 蓝牙传输）   ← 嵌入式
  │ WiFi / 蓝牙 / USB
  ▼
PC / Jetson（上位机）                     ← 你的主战场
  │ Python + MNE + PyTorch (EEGNet / Conformer)
  ▼
意图分类结果
  │
  ▼
ROS2 → 控制机械臂 / 轮椅 / 光标           ← 闭环 BCI
```

**你的研究重点在第 4–5 层（算法）**，但实际做"实时在线 BCI"或"脑控机械臂"类项目时，第 3 层嵌入式和第 6 层 ROS2 会撞上。**这是研究生阶段按需补的内容，不是现在。**

# 附录 B · 中文互联网常见坑与避坑清单

## B.1 学习阶段的坑

### 坑 1 · 死磕 51 单片机 3 个月
- **现象**：很多老教程让新手在 51 上反复刷 LED，浪费半年。
- **真相**：51 是地基，会用 LED / 按键 / 串口 / 定时器 / 中断就够了，**1 个月内必须过渡到 STM32**。
- **正确姿势**：51 快进，STM32 是主战场。

### 坑 2 · 纠结 "野火 vs 正点原子"
- **真相**：两家代码风格小差异，视频节奏差异，**买哪家都行**。关键是**买了要做完**。
- **避坑**：别两家来回切，浪费钱浪费时间。

### 坑 3 · 一上来学 HAL 库，不碰寄存器
- **真相**：HAL 上手快但黑盒，碰到怪问题不会调。工业界招聘重点考寄存器级理解。
- **正确姿势**：HAL 学完一轮后，至少写一个**寄存器版本**的串口 / 定时器，理解底层。

### 坑 4 · 只做小车小板子，从不做项目
- **真相**：简历上写 5 个"点灯 demo" 等于零。必须有**一个完整项目**（带 PCB、通讯、界面、闭环控制）。

### 坑 5 · 买树莓派学单片机
- **真相**：树莓派跑 Linux，**不是单片机**。学单片机应该买 STM32 开发板。
- 初学者常把"派"和"单片机"搞混，买错板子学了半天。

### 坑 6 · Arduino 学完觉得自己会嵌入式
- **真相**：Arduino 封装太厚，学完只懂 `digitalWrite`，不懂寄存器、时钟、中断。面试被问就露怯。
- **避坑**：Arduino 只是入门 2 周玩具，**必须进阶到 STM32**。

### 坑 7 · ROS1 和 ROS2 混学
- **真相**：2025-05 ROS1 Noetic 已 EOL。**新人直接学 ROS2**。看到 ROS1 资料果断跳过或当历史读。

### 坑 8 · Windows 下学 ROS
- **真相**：ROS2 虽然支持 Windows，但**所有工业项目都在 Ubuntu 上**。一定要上 Ubuntu（WSL2 也行，但远程开发更好）。

### 坑 9 · 追新版本（追 Jazzy / Kilted）
- **真相**：2026 年工业界主流仍是 **Humble（LTS）**。新人学 Humble 就对，别追最新。

### 坑 10 · 不做仿真直接买硬件
- **真相**：一块真机底盘 + 激光雷达 + 机械臂轻松上万。**先在 Gazebo / Isaac 仿真**跑通，再买硬件。

## B.2 就业阶段的坑

### 坑 11 · "嵌入式劝退论"
- **真相**：劝退的多是卷低端 51 岗位的人。**车载电子、边缘 AI、IoT 三大方向薪资溢价显著**。
- 应届 6–9K 是低端岗，5 年经验的车载嵌入式 25–40K。

### 坑 12 · 国企 vs 互联网的认知错位
- **真相**：嵌入式岗位大量在**制造业、军工、汽车**，不是互联网大厂。面试流程、薪资结构都不一样。
- 别用互联网的"大小周 + 期权"去衡量嵌入式岗位，嵌入式圈相对平稳。

### 坑 13 · 只会 C 不会硬件
- **真相**：嵌入式面试至少问**示波器、万用表使用，电源设计，抗干扰**等硬件知识。
- **避坑**：学习时一定要动手焊 / 画板，不能只写代码。

### 坑 14 · 只会 HAL 库调包
- **真相**：工业界量产代码大量写**寄存器级优化**，HAL 会被 rewrite。
- **避坑**：至少理解一次底层。

### 坑 15 · 对 PLC 不屑一顾
- **真相**：PLC 岗位很多（尤其二三线城市），工资虽然不如互联网但**抗失业**，年龄越大越吃香。如果你求稳，别瞧不起这个方向。

### 坑 16 · 盲目追"具身智能"热点
- **真相**：2026 年具身智能融资狂飙（一季度 200 亿元），但大部分公司没产品、拿不到量产订单。**本科进 = 被当苦力，硕士进 = 看公司造化**。
- **避坑**：考虑具身智能 → 优先宇树、智元、银河通用头部；二线公司谨慎。

### 坑 17 · 以为学了 ROS2 就能去机器人公司
- **真相**：机器人公司面试要你会 **C++ 现代特性 + 多线程 + 算法（SLAM/控制）**，ROS2 只是胶水。
- **避坑**：学 ROS2 之前先打 C++ 功底。

### 坑 18 · 忽视英文
- **真相**：ROS2 / Jetson / Isaac 这些新东西，**中文教程至少慢半年**。
- **避坑**：早读英文官方文档是硬技能。

## B.3 BCI 方向交叉时的坑（你未来会遇到）

### 坑 19 · 以为 BCI 只是算法
- **真相**：做出可发论文的 BCI，往往要**懂一点硬件**（采集卡、电极、阻抗）。连实验范式都搭不起来就没数据。
- **避坑**：研一入组第一年至少跟师兄跑一次完整 EEG 采集实验。

### 坑 20 · 买消费级 EEG（Muse、NeuroSky）做严肃研究
- **真相**：消费级信号差、通道少、论文拒稿率高。学术要用 **g.tec / BrainProducts / Neuracle 国产科研级**。
- **避坑**：入组先问老师有没有科研级设备。

# 附录 C · 2026 行业现状速览

## C.1 嵌入式市场
- 2021 全球规模 900 亿美元，2027 预计 1160 亿美元，CAGR ~5.6%。
- **车载电子、边缘 AI、IoT** 三大细分爆发；传统单片机岗位薪资平缓。
- 5 年以上一线嵌入式工程师月薪 25–40K，**年龄友好**，十年工程师比应届更抢手。

- 工信部 2026-03 发布《人形机器人与具身智能标准体系（2026 版）》—— 我国首个国家级标准。
- 全球具身智能市场：2025 年 44.4 亿美元，2030 预计 230 亿美元，CAGR ~39%。
- 中国 2026 一季度具身智能赛道融资 50+ 起，30+ 家企业，累计融资 ~200 亿元，同比 +60%。
- 2026 年人形机器人从 "功夫模式" → "打工模式"，量产应用可能启动。
- 技术趋势：**ROS2 + Zenoh + Isaac + VLA 模型**。

- **人形机器人**：宇树、智元、银河通用、傅里叶、星动纪元、众擎
- **协作机械臂**：UR、JAKA、遨博、新松、越疆
- **激光雷达**：禾赛、速腾聚创
