# 408 第一章：操作系统 x 计算机组成原理总览

基于《操作系统.md》第一章、《计算机组成原理.md》第一章，以及第一章配图 79 张整理。目标是把两门课第一章的考点、知识体系、联系、王道刷题计划和掌握标准合成一张可复习的大图。

## 1. 总目标与主线

第一章共同回答一个问题：一个程序为什么能从磁盘上的文件，变成 CPU 正在执行的一串机器指令，并且还能安全地访问内存、文件和设备？

主线：源代码 -> 编译/汇编/链接 -> 可执行文件 -> OS 装载 -> 进程与地址空间 -> CPU 取指/译码/执行 -> 访问文件/设备/进程/内存资源时系统调用 -> trap 进入内核态 -> OS 服务 -> 返回用户态。

计组第一章解释“硬件怎样自动取指、译码、执行”；操作系统第一章解释“谁管理硬件、谁分配资源、谁处理系统调用和中断”。两门课的接口是：中断、异常、陷入指令、特权级、系统调用、程序装入与运行上下文。

## 2. 操作系统第一章知识体系

### 2.1 操作系统概念、功能与接口

- 操作系统是控制和管理整个计算机系统硬件与软件资源，合理组织调度计算机工作和资源分配，并向用户和其他软件提供接口与环境的系统软件。
- 作为资源管理者：处理机管理、存储器管理、文件管理、设备管理。
- 向上层提供服务：图形用户界面、命令接口、程序接口。
- 命令接口：联机命令接口是交互式；脱机命令接口是批处理。
- 程序接口：系统调用是核心。应用程序通过系统调用请求 OS 内核服务。
- CPU 能直接访问内存，不能直接执行外存文件；外存访问必须通过 OS 的文件系统和 I/O 管理。

易错点：库函数不等于系统调用。库函数可能只是普通用户态代码，也可能封装系统调用；系统调用一定会请求内核服务。

### 2.2 操作系统四大特征

| 特征 | 含义 | 必考区分 | 例子 |
|---|---|---|---|
| 并发 | 多个事件在同一时间间隔内发生 | 宏观同时、微观交替；并行是同一时刻真正同时 | 单核 CPU 上多个程序轮流运行 |
| 共享 | 系统资源可供多个并发进程共同使用 | 互斥共享 vs 同时共享 | 摄像头通常互斥；磁盘文件可交替共享 |
| 虚拟 | 把一个物理实体变成多个逻辑实体，或反向抽象 | 时分复用、空分复用 | 一台 CPU 虚拟成多个处理机；内存抽象成连续地址空间 |
| 异步 | 进程走走停停，以不可预知速度推进 | 由资源有限、调度、中断、I/O 等造成 | 同一程序多次执行时间不同 |

并发和共享互为存在条件。没有并发，就谈不上多个进程同时需要资源；没有共享，就无法支撑多个进程并发推进。

### 2.3 操作系统发展与分类

| 阶段/类型 | 核心思想 | 优点 | 缺点/考点 |
|---|---|---|---|
| 手工操作阶段 | 用户人工装入程序、操作机器 | 系统简单 | 人机速度矛盾严重 |
| 单道批处理 | 脱机 I/O + 监督程序，一次一道作业 | 减少人工干预 | CPU 等待 I/O 时仍浪费 |
| 多道批处理 | 内存同时放多道程序，交替运行 | 提高 CPU 和资源利用率，OS 正式诞生 | 缺少交互性 |
| 分时系统 | 时间片轮流为多个用户服务 | 交互性强，响应较及时 | 不能优先处理紧急任务 |
| 实时系统 | 优先响应紧急任务 | 及时性、可靠性强 | 硬实时严格，软实时可偶尔延迟 |
| 网络/分布式/个人机 | 网络共享、分布协同、个人使用 | 扩展应用场景 | 分布式强调多机协同完成同一任务 |

### 2.4 运行机制：内核、特权级、中断

- 内核程序运行在内核态，应用程序运行在用户态。
- 特权指令会影响系统全局状态或他人资源，只允许内核态执行。
- 非特权指令是普通运算和一般用户态计算。
- CPU 用 PSW 或状态寄存器中的标志位区分用户态/内核态。
- 用户态到内核态：由中断、异常、陷入等事件触发，硬件自动完成状态转换并跳转内核入口。
- 内核态到用户态：OS 恢复现场并修改状态标志，返回用户程序继续执行。

| 中断类别 | 别名 | 来源 | 同步性 | 例子 | 处理重点 |
|---|---|---|---|---|---|
| 外中断 | 中断、异步中断 | CPU 外部设备或时钟 | 与当前指令无关，异步 | 时钟中断、I/O 完成、键盘、网卡 | 通常在一条指令执行结束后检测 |
| 内中断 | 异常、同步中断 | CPU 执行当前指令时产生 | 与当前指令有关，同步 | 除零、非法指令、缺页、系统调用 | 执行到相关指令或异常条件时发生 |
| 陷入 trap | 访管/陷入指令 | 用户程序主动请求内核 | 同步 | 系统调用 | 用户态执行陷入指令后进入内核态 |
| 故障 fault | 可恢复异常 | 执行中发现问题 | 同步 | 缺页故障 | 修复后可回到原指令继续执行 |
| 终止 abort | 不可恢复异常 | 严重错误 | 同步 | 硬件故障、严重非法状态 | 通常终止进程或系统处理 |

中断处理过程：当前指令执行 -> 检测事件 -> 保存断点 -> 查中断向量表 -> 进入内核态 -> 执行中断处理程序 -> 恢复现场 -> 返回用户态或调度其他进程。

### 2.5 虚拟机、系统调用与体系结构

- 第一类 VMM：直接运行在硬件上，性能更好，典型如 VMware ESXi、Xen、裸机 Hyper-V。
- 第二类 VMM：运行在宿主操作系统之上，依赖 Host OS，典型如 VirtualBox、VMware Workstation。
- 系统调用过程：传参 -> 执行陷入指令 -> 进入内核态 -> 内核服务 -> 返回用户态。
- 系统调用分类：设备管理、文件管理、进程控制、进程通信、内存管理。
- 大内核：主要功能模块都放内核态，性能高但结构复杂、维护难。
- 微内核：只把最基本机制放内核，策略尽量放用户态；结构清晰、可靠性好，但状态切换多，性能可能下降。
- 分层结构强调层次调用；模块化强调可加载与扩展；外核强调资源保护与抽象分离。

### 2.6 操作系统引导

1. CPU 从 ROM/BIOS 固定入口取第一条指令。
2. POST 通电自检，检查基本硬件。
3. 加载 MBR 或分区引导记录，定位操作系统所在分区。
4. 启动管理器加载内核。
5. 先加载核心内核，再按需加载文件系统驱动、USB 驱动等可加载内核模块。

## 3. 计算机组成原理第一章知识体系

### 3.1 硬件组成与冯诺依曼结构

- 冯诺依曼机特点：五大部件；指令和数据同等地位存于存储器；二进制表示；指令由操作码和地址码组成；以运算器为中心。
- 现代计算机结构：以存储器为中心，CPU 包括运算器和控制器；主机包括 CPU 和主存储器；I/O 设备包括输入设备和输出设备。
- 软件/硬件逻辑等价：同一功能既可用软件实现，也可用硬件实现；硬件快但成本高，软件灵活但可能慢。
- 按字节编址：一个地址编号对应 1 个字节。
- 机器字长：CPU 一次能处理多少位。
- 存储字长：主存一个存储单元存多少位。
- 指令字长：一条指令占多少位，由 ISA 设计决定，不一定等于机器字长。

易错点：MDR 表示一次从内存读出或写入的数据位数，通常与存储字长相关；MAR 位数决定可寻址的存储单元数量。

### 3.2 主存、运算器、控制器

| 部件 | 作用 | 王道常考点 |
|---|---|---|
| MAR | 地址寄存器，保存要访问的主存地址 | MAR 位数决定最多可寻址单元数 |
| MDR | 数据寄存器，保存与主存交换的数据 | MDR 位数通常等于存储字长 |
| ACC | 累加器，保存操作数或运算结果 | 加减乘除过程常用 |
| MQ | 乘商寄存器 | 乘除法中存放乘数、商等 |
| X | 通用寄存器/缓冲寄存器 | 暂存操作数，位宽要与 ALU 匹配 |
| ALU | 算术逻辑单元 | 执行算术运算和逻辑运算 |
| PC | 程序计数器，保存下一条指令地址 | 取指后通常自动加 1 |
| IR | 指令寄存器，保存当前指令 | 操作码送 CU，地址码用于形成操作数地址 |
| CU | 控制单元，分析指令并发出控制信号 | 控制取指、分析、执行阶段 |

### 3.3 计算机工作过程

典型过程：PC -> MAR -> 主存 -> MDR -> IR；IR 的操作码 -> CU；IR 的地址码 -> MAR；MDR/寄存器 -> ALU -> ACC/MQ/X 或写回主存。

- 取指阶段：PC 给地址，主存读出指令到 MDR，再送入 IR，PC 通常自增。
- 分析阶段：OP(IR) 送 CU，CU 分析操作码。
- 执行阶段：地址码可能送 MAR 取操作数，数据经 MDR 到寄存器或 ALU，结果写回。
- 硬件自动性：取指、译码、执行由控制器和时钟电路驱动，不需要 OS 指挥每一条指令。
- OS 的作用点：OS 修改 PC、寄存器上下文、页表和内存映射，决定下一轮 CPU 自动取到哪个进程的指令。

### 3.4 软件、ISA 与层次结构

| 概念 | 定义/作用 | 常考区分 |
|---|---|---|
| 系统软件 | 保证系统高效正确运行的基础软件 | OS、DBMS、编译器、标准库、服务程序 |
| 应用软件 | 用户为解决具体问题编写的软件 | 办公、工程、统计、科学计算程序 |
| 机器语言 | 二进制机器指令 | 计算机唯一能直接识别和执行的语言 |
| 汇编语言 | 用助记符表示机器指令 | 必须经汇编器翻译成机器语言 |
| 高级语言 | C/C++/Java/Python 等 | 通常需编译或解释 |
| 编译器 | 高级语言翻译为汇编或目标代码 | C/C++ 常继续汇编和链接 |
| 解释器 | 按执行顺序逐句翻译并执行 | Python、Shell 常作为例子 |
| ISA | 指令集体系结构，软硬件接口 | 规定指令、编码、寄存器、寻址方式 |

层次结构：高级语言机器 -> 汇编语言机器 -> 操作系统机器 -> 传统机器 -> 微程序机器 -> 硬件。下层是上层的基础，上层是下层功能的扩展。

### 3.5 性能指标与公式

| 指标 | 公式/含义 | 考点提醒 |
|---|---|---|
| CPU 时钟周期 | 一个时钟脉冲的时间 | 主频越高，周期越短 |
| 主频 | 主频 = 1 / CPU 时钟周期 | 性能/速率类单位常按 1000 进制 |
| CPI | 执行一条指令平均需要的时钟周期数 | CPI 会受 Cache、流水线、分支预测等影响 |
| CPU 执行时间 | T = 指令条数 x CPI / 主频 | 指令数、平均 CPI、主频共同决定 |
| IPS/MIPS | 每秒执行多少条指令 | IPS = 主频 / 平均 CPI |
| FLOPS | 每秒浮点运算次数 | K/M/G/T/P 按 10^3 递进 |
| 数据通路带宽 | 数据总线一次能并行传输的信息位数 | 影响部件间一次传送的数据量 |
| 吞吐量 | 单位时间完成任务数量 | 常与响应时间形成权衡 |
| 响应时间 | 从发出请求到获得结果所需时间 | 包括 CPU 时间、等待时间、I/O 时间、系统开销 |

存储容量相关通常按 1024；速度、频率、带宽、运算次数等性能指标通常按 1000。

## 4. 两门课怎么联系起来

| 共同问题 | 计组视角 | 操作系统视角 | 考试怎么问 |
|---|---|---|---|
| 系统调用为什么会切内核态 | 执行陷入指令产生内中断，CPU 改变状态并跳入口 | 内核接管请求，执行文件/设备/进程/内存服务 | 系统调用过程、trap 指令、用户态/核心态转换 |
| 中断为什么能实现多任务 | 硬件在指令周期边界响应中断，保存断点并查向量表 | 时钟中断进入 OS，OS 可进行调度和上下文切换 | 中断分类、中断响应时机、进程切换原因 |
| CPU 为什么能切到另一个程序 | CPU 只根据 PC 取指，寄存器决定当前上下文 | OS 保存/恢复寄存器、PC、页表等上下文 | 程序与进程、PCB、上下文切换 |
| 为什么普通程序不能直接操作硬件 | 特权指令需要更高 CPU 特权级 | OS 统一管理共享资源，防止冲突和破坏 | 特权指令、系统调用、资源管理者 |
| 程序如何从磁盘开始运行 | CPU 从内存取指，不能直接执行磁盘文件 | OS/引导程序把代码装入内存并建立运行环境 | 程序装入、OS 引导、CPU 取指 |


## 5. 高频考点清单与易混辨析

### 5.1 操作系统第一章逐小节考点

| 小节 | 必背考点 | 王道常见问法 | 你要会说的话 |
|---|---|---|---|
| OS 概念和功能 | OS 是系统软件；管理软硬件资源；提供接口 | “操作系统的目标/功能是什么？”“OS 管理哪些资源？” | OS 一边向下管理 CPU、内存、文件、设备，一边向上给用户和程序提供接口。 |
| OS 接口 | 命令接口、程序接口、GUI；系统调用属于程序接口 | “系统调用和命令接口的区别？” | 命令接口给用户直接用，系统调用给程序通过代码请求内核服务。 |
| 四大特征 | 并发、共享、虚拟、异步 | “并发和并行区别？”“共享方式有哪些？” | 并发是同一时间间隔内交替推进，并行是同一时刻真正同时执行。 |
| 发展分类 | 手工、单道批、多道批、分时、实时、网络、分布式 | “哪种系统解决交互问题？”“哪种系统强调及时性？” | 多道批提高资源利用率，分时强调交互，实时强调及时和可靠。 |
| 运行机制 | 内核程序/应用程序；特权指令/非特权指令；用户态/内核态 | “哪些指令必须在核心态执行？” | 会影响系统全局状态、硬件资源或其他进程的操作必须交给内核。 |
| 中断异常 | 外中断、内中断、trap、fault、abort；中断向量表 | “系统调用属于哪类中断？”“中断响应时机？” | 系统调用是 trap，属于内中断；外中断通常在指令结束后检测。 |
| 系统调用 | 参数传递、陷入指令、内核服务、返回用户态 | “系统调用与库函数区别？” | 库函数可能封装系统调用；系统调用一定会进入内核请求 OS 服务。 |
| 虚拟机 | 第一类 VMM、第二类 VMM、特权级 | “两类虚拟机管理程序区别？” | 第一类直接跑硬件上，第二类跑宿主 OS 上。 |
| 体系结构 | 大内核、微内核、分层、模块化、外核 | “微内核优缺点？” | 微内核结构清晰可靠，但状态切换多，性能可能下降。 |
| OS 引导 | ROM/BIOS、POST、MBR、PBR、启动管理器、内核 | “开机后 OS 如何被加载？” | CPU 先从 ROM 取指，逐级找到并加载操作系统内核。 |

### 5.2 计组第一章逐小节考点

| 小节 | 必背考点 | 王道常见问法 | 你要会说的话 |
|---|---|---|---|
| 硬件基本组成 | 五大部件；现代计算机以存储器为中心 | “主机包括哪些？”“CPU 包括哪些？” | CPU=运算器+控制器；主机=CPU+主存；I/O 设备在主机外。 |
| 冯诺依曼结构 | 指令和数据同存储器；二进制；操作码+地址码；以运算器为中心 | “冯诺依曼机特点？” | 指令和数据在形式上都是二进制，CPU 按阶段解释它们。 |
| 主存结构 | MAR、MDR、存储体、存储单元、存储字、存储字长 | “MAR/MDR 位数分别决定什么？” | MAR 位数决定地址范围，MDR 位数决定一次交换的数据宽度。 |
| 运算器 | ACC、MQ、X、ALU | “乘除法中 ACC/MQ 的作用？” | 运算器负责数据加工，寄存器临时保存操作数和结果。 |
| 控制器 | PC、IR、CU；取指、分析、执行 | “PC 和 IR 的区别？” | PC 保存下一条指令地址，IR 保存当前正在执行的指令。 |
| 工作过程 | PC->MAR->MDR->IR；OP(IR)->CU；Ad(IR)->MAR | “一条指令如何执行？” | 先取指，再分析操作码，最后取操作数并执行。 |
| 软件分类 | 系统软件、应用软件；DBMS 是系统软件 | “数据库管理系统是不是系统软件？” | DBMS 是系统软件，数据库系统不是单纯一个软件。 |
| 语言翻译 | 机器语言、汇编语言、高级语言；编译、汇编、解释 | “哪种语言 CPU 直接执行？” | CPU 只能直接执行机器语言，汇编和高级语言都要翻译。 |
| ISA | 指令、编码、寄存器、寻址方式 | “ISA 为什么是软硬件接口？” | ISA 规定软件能使用哪些机器指令，也规定硬件必须实现什么。 |
| 性能指标 | 主频、CPI、CPU 时间、IPS/MIPS、FLOPS、吞吐量、响应时间 | “A 主频高就一定快吗？” | 不一定，性能由指令数、CPI、主频共同决定。 |

### 5.3 高频易混点

| 易混点 | 正确区分 | 快速判断 |
|---|---|---|
| 并发 vs 并行 | 并发是同一时间间隔内都在推进；并行是同一时刻同时执行 | 单核可并发，多核才可能真正并行。 |
| 同步/异步 vs OS 异步性 | 同步/异步常说事件等待关系；OS 异步性说进程推进速度不可预知 | 看到“走走停停、不可预知”就是 OS 异步性。 |
| 系统调用 vs 库函数 | 系统调用进入内核；库函数是语言/运行库提供的函数 | printf 可能最终调 write，但 printf 本身不是系统调用。 |
| 中断 vs 异常 | 外中断来自 CPU 外部；异常来自 CPU 执行指令内部 | 键盘/时钟/I/O 完成是外中断；除零/缺页/trap 是内中断。 |
| trap vs fault vs abort | trap 主动请求；fault 可修复；abort 严重不可恢复 | 系统调用 trap；缺页 fault；严重硬件故障 abort。 |
| 用户态 vs 内核态 | 用户态不能执行特权指令；内核态可执行全部指令 | 会影响别人或全局状态的事，进内核。 |
| MAR vs MDR | MAR 存地址；MDR 存数据 | 地址范围看 MAR，数据宽度看 MDR。 |
| PC vs IR | PC 指下一条；IR 存当前条 | 取指后 PC 常自增，IR 用于当前指令译码。 |
| 机器字长 vs 存储字长 vs 指令字长 | 机器字长看 CPU；存储字长看主存；指令字长看 ISA | 三者不必相等。 |
| 主频高 vs 性能高 | 主频只是因素之一 | CPU 时间 = 指令数 x CPI / 主频。 |

### 5.4 第一章答题模板

| 题型 | 答题模板 |
|---|---|
| 问 OS 作用 | 先答“管理资源”，再列处理机、存储器、文件、设备；再答“提供接口”，列命令接口、程序接口、GUI。 |
| 问系统调用过程 | 用户程序传参 -> 执行陷入指令 -> CPU 切换到内核态 -> OS 查表并执行服务例程 -> 恢复现场返回用户态。 |
| 问中断过程 | 事件发生 -> 当前指令结束后检测或同步异常触发 -> 保存断点/现场 -> 查中断向量表 -> 转入处理程序 -> 恢复或调度。 |
| 问指令执行过程 | PC 给出地址 -> MAR 访存 -> MDR 取回指令 -> IR 保存指令 -> CU 分析操作码 -> 执行并写回。 |
| 问性能比较 | 不只看主频；用 CPU 时间公式比较：指令条数、平均 CPI、主频三者共同决定。 |

### 5.5 刷题优先级

1. 先刷“概念边界题”：系统调用/库函数、用户态/内核态、中断/异常、MAR/MDR、PC/IR。
2. 再刷“流程顺序题”：系统调用过程、中断处理过程、CPU 指令执行过程、操作系统引导过程。
3. 最后刷“计算题”：CPU 执行时间、CPI、主频、MIPS、存储容量和地址范围。
4. 错题本不要只写答案，要写“为什么另外三个选项错”。第一章选择题很多就是概念边界陷阱。
