408 操作系统 · 第一章
Explainer 01

计算机按下电源后，操作系统还没开始运行

真正先动起来的是固件和引导程序。它们像接力赛，把 CPU 从“只会从固定地址取指令”的状态，送到“内核接管机器”的状态。
上电CPU 复位，跳到固件入口
自检检查内存、设备、启动项
找盘定位可启动设备
引导加载 bootloader
内核OS 开始接管
Step 1 · 固件先启动

BIOS / UEFI 做的事：让硬件进入“可启动”状态

CPU 从复位地址取第一条指令
固件初始化主板、内存和基本设备
按照启动顺序查找可启动介质
把控制权交给引导程序

考点抓法

这里不是“操作系统启动了”，而是“操作系统被启动”。固件负责找到入口，引导程序负责把内核搬进内存。

Step 2 · 引导程序接力

Bootloader 的核心任务：定位内核、加载内核、跳转内核入口

定位在磁盘或 EFI 分区里找到内核映像与启动配置。
加载把内核读入内存，并准备必要的参数。
跳转把 CPU 控制权交给内核入口地址。
Step 3 · 内核接管

内核一旦运行，就开始建立真正的操作系统环境

中断建立中断向量和异常处理，让 CPU 能响应事件。
内存建立页表、管理物理内存和虚拟地址空间。
进程创建第一个系统进程，用户态程序随后启动。
408 记忆框架

一句话记住：固件找入口，引导搬内核，内核管全局

做题时看到“系统启动、引导、装入内核、初始化”，先判断题目问的是哪一棒。不要把 BIOS/UEFI、Bootloader、OS Kernel 的职责混在一起。
固件硬件初始化与启动设备选择
引导程序加载内核与启动参数
内核中断、内存、进程、设备
用户态服务、Shell、应用程序
答题按阶段拆职责
先记住一个大方向：按下电源时，操作系统并没有立刻运行。
第一棒是 BIOS 或 UEFI，它完成基本硬件初始化，并寻找可以启动的设备。
第二棒是引导程序，它把操作系统内核从磁盘加载到内存，再跳到内核入口。
第三棒才是内核：建立中断、内存管理、进程和设备管理，系统环境逐步成形。
所以做 408 题时，要按“固件、引导程序、内核”三层分清职责。
