# ① 计组 OS 底层主干总论

## 0. 一句话总主轴

> 一段磁盘上的二进制，凭什么能变成 CPU 正在跑的指令流，还能安全地碰内存、文件、设备？
> **答案就是一条"硬件给约束、OS 补窟窿、补出新窟窿再补"的迭代链。** 计组讲"硬件怎么自动取指执行"，OS 讲"谁来管这堆硬件、谁分配资源、谁接系统调用和中断"。两科接口只有六个词：**中断、异常、陷入、特权级、系统调用、上下文**。

## 1. 因果引擎：缺陷迭代范式（全书统一思维模型）

任何一个 OS 机制都不是凭空设计的，它是被硬件逼出来的。强化阶段必须能对任意知识点跑通这条四段链：

```
硬件原生短板 ──> 早期方案(够用但有洞) ──> 暴露残留缺陷 ──> 衍生配套机制(补洞) ──> 新洞...
```

**为什么强化阶段要这样背**：408 综合大题考的从来不是"是什么"，而是"为什么必须有它、没它会怎样、它和上一环下一环怎么咬合"。把链路记住，选择题的"概念边界陷阱"和大题的"流程顺序"同时解决。

## 2. 硬件压给 OS 的五道硬约束（所有 OS 机制的总源头）

| # | 硬件原生短板（计组事实） | 逼出的 OS 抽象 | 残留缺陷 → 衍生 |
|---|---|---|---|
| 约束1 | **CPU 只按 PC 从主存取指、不能直接执行磁盘文件**；磁盘不可按字节供 CPU 取指 | 程序**装入**、进程、地址空间 | 逻辑≠物理地址 → 重定位 → 动态重定位 → 虚拟内存 |
| 约束2 | **主存容量有限、易失、单一物理地址空间**，多程序会互相踩内存 | **虚拟内存 + 页表 + 地址隔离** | 页表太大/访存翻倍 → 多级页表 → TLB |
| 约束3 | **访存延迟远慢于 CPU（存储墙/冯诺依曼瓶颈）** | **Cache + 局部性 + TLB** | 抖动、一致性 → 组相联、写策略、MESI |
| 约束4 | **外设慢且异步，CPU 轮询=纯浪费** | **中断 + DMA + 缓冲** | 中断开销/争总线/数据丢失 → DMA→通道、缓冲池 |
| 约束5 | **任何指令物理上都能碰全局资源，多进程共享必冲突** | **特权级 + 系统调用 + 同步互斥 + 文件系统** | 死锁、临界区竞争 → PV、银行家、原子指令 |

> **强化记忆点**：OS 的四大管理（处理机/存储/文件/设备）= 上面五道硬约束的兜底工程。任意 OS 算法被问"为什么这样设计"，先回到它对应的硬件短板。

## 3. 主演化总表（硬件短板 → OS 方案 → 残留缺陷 → 下一方案）

| 演化轴 | 硬件短板/前置缺陷 | 当代方案 | 原生局限（丢分点） | 衍生下一环 |
|---|---|---|---|---|
| 装入 | CPU 不认磁盘文件 | 绝对装入 | 只能单道、编译期定死物理地址 | 静态重定位 → 动态重定位（[[计算机组成原理强化]] 链1） |
| 并发 | 单核同刻只跑一条指令流 | 多道批处理 | 无交互、无法抢回死循环进程的 CPU | 分时（靠**时钟中断**强制夺权） |
| 内存 | 物理内存有限+需隔离 | 固定/动态分区 | 外部碎片、装不下大程序 | 分页 → 请求分页（虚拟内存） |
| 页表 | 单级页表占内存巨大 | 单级页表 | 进程页表可达数 MB | 多级页表（缺页式建表）→ TLB |
| 访存 | 多级页表访存翻倍 + 存储墙 | 直接查主存页表 | 一次访问要 N+1 次访存 | TLB（地址翻译 Cache）+ 数据 Cache |
| 设备 | 设备慢、CPU 不能逐字节搬 | 程序查询 | CPU 忙等、利用率极低 | 中断驱动 → DMA → 通道 |
| 文件 | 磁盘按块存、文件是字节流 | 连续分配 | 外部碎片、文件不能增长 | 链接 → 索引 → 混合索引(inode) |
| 共享 | 指令物理上能碰全局资源 | 无保护 | 进程互相破坏 | 特权级/系统调用 → PV/管程 → 死锁处理 |

## 4. 两科接口断面（计组↔OS 的五个咬合点）

| 共同问题 | 计组视角（硬件怎么做到） | OS 视角（软件拿它干什么） | 命题断面 |
|---|---|---|---|
| 系统调用为何切内核态 | 陷入指令产生内中断，CPU 改 PSW 跳入口 | 内核接管，做文件/设备/进程/内存服务 | trap、用户态↔核心态、参数传递 |
| 中断为何能撑多任务 | 指令周期**边界**响应中断、查中断向量表 | 时钟中断进 OS → 调度+上下文切换 | 中断分类、响应时机、切换原因 |
| CPU 为何能换进程 | CPU 只认 PC 取指，寄存器决定上下文 | OS 保存/恢复 PC、GPR、PSW、PTBR | PCB、上下文、切换开销 |
| 普通程序为何不能直接操硬件 | 特权指令需更高 CPU 特权级 | OS 统一管共享资源防破坏 | 特权指令、资源管理者 |
| 程序如何从磁盘跑起来 | CPU 从主存取指、不执行磁盘文件 | 引导/装入把代码搬进内存建环境 | 装入、引导、取指（链1） |

# ② 五大强化全链路（408 综合大题必考，缺陷迭代式拆解）

> 每条链按「**前置方案缺陷 → 诞生新技术 → 新技术残留短板 → 衍生配套机制**」分步；附**计组强化考点 / OS 强化考点 / 命题坑点溯源**。

## 链路 1：磁盘可执行文件 → 内存加载 → CPU 取指执行

### 分步流程（缺陷迭代）

1. **硬约束起点**：CPU 只能从主存按 PC 取指执行。磁盘文件是块设备数据，CPU 无法直接取指 → **必须先装入主存**。
2. **编译链接产物 = 逻辑地址**（可重定位目标）。链接方式三级演化：**静态链接**（运行前合并，缺陷:库冗余、更新难）→ **装入时动态链接**（装入边链，缺陷:装入慢）→ **运行时动态链接**（用到才链，衍生:DLL/共享库、节省内存）。
3. **装入方式三级演化（核心）**：
   - **绝对装入**：编译期就定死物理地址 → 缺陷：**只能单道**。
   - **静态重定位**：装入期一次性把逻辑地址改成物理地址 → 缺陷：**必须一次连续装完、运行中不能再移动**（无法换入换出）。
   - **动态重定位**：运行期由**重定位寄存器(基址)/MMU** 实时加偏移 → 衍生：**支持分页、换入换出、虚拟内存**（链 3 的地基）。
4. **建进程映像**：OS 建 PCB、页表、内核栈，代码/数据/堆/栈段就位，PC 置入口。
5. **CPU 自动取指执行**（计组工作过程）：`PC→MAR→主存→MDR→IR`，`OP(IR)→CU` 译码，`Ad(IR)→MAR` 取操作数，ALU 执行写回，PC 自增。**硬件自动性**：这一步不需要 OS 逐条指挥；OS 只在切换时改 PC/寄存器/页表。

### 强化考点

- **计组**：逻辑 vs 物理地址、重定位寄存器、指令周期/机器周期、PC 自增时机、`PC→MAR→MDR→IR` 数据通路。
- **OS**：三种装入、三种链接、进程映像组成、可执行文件 ≠ 进程。
- **命题坑**：① "装入时机决定能否多道"——绝对/静态装入做不了换入换出；② **只有动态重定位才能支撑虚拟内存**，省略它则链 3 整条塌；③ 可执行文件在磁盘是"程序"，装入并建 PCB 后才是"进程"。

> 详见 [[计算机组成原理强化]] 第1/4/5章、[[操作系统强化]] 第1/3章。

## 链路 2：单核 CPU 多进程分时并发调度

### 分步流程（缺陷迭代）

1. **硬约束起点**：单核**同一时刻只能执行一条指令流** → 想"并发"只能**分时复用**（宏观同时、微观交替）。
2. **多道批处理**：内存放多道、IO 时切换 → 缺陷：**无交互、无法从死循环用户进程手里抢回 CPU**。
3. **分时系统**：时间片轮转。物理前提是 **时钟中断**——没有时钟中断这个硬件，OS 永远拿不回 CPU 控制权。**时钟中断是分时的命根子**。
4. **上下文切换**（计组↔OS 咬合点）：保存/恢复 **PC、通用寄存器、PSW(状态字)、页表基址寄存器 PTBR**。这堆"上下文"本质是计组寄存器组。切换必须在**内核态**完成。
5. **调度算法权衡**：FCFS（公平/对短作业不利）→ SJF（周转最优/会饿死长作业/需预知运行时间）→ RR（响应好/时间片太小则切换开销占比飙升）→ 优先级 → **MLFQ**（综合，兼顾短作业与公平）。

### 强化考点

- **计组**：上下文 = 哪些寄存器（PC/GPR/PSW/PTBR）、时钟中断在**指令周期边界**被响应、PSW 各标志位。
- **OS**：进程五态转换、调度时机、切换 vs 调度、周转时间/带权周转/等待时间计算、时间片大小权衡。
- **命题坑**：① **进程切换 ≠ 进程调度**（调度是"选谁"，切换是"换上下文"）；② 时间片太小→切换开销吞吐崩，太大→退化成 FCFS；③ 只有核心态能执行切换涉及的特权操作；④ "上下文切换"开销主要来自寄存器/页表/TLB 刷新，不是算法本身。

> 详见 [[计算机组成原理强化]] 第5章（中断/数据通路）、[[操作系统强化]] 第2章。

## 链路 3：虚拟地址 → 多级页表 → TLB → Cache → 主存（408 压轴计算链）

### 分步流程（缺陷迭代，每一环都在补上一环的洞）

1. **物理内存有限 + 多进程要隔离** → **虚拟地址 + 页表**（每进程一张，VA→PA 映射）。残留：**页表本身占内存巨大**（32 位、4KB 页 → 单级页表 4MB/进程）。
2. → **多级页表**（按需建立、缺页式分配二级表，省内存）。残留：**一次访问要 N+1 次访存**（N 级页表 + 1 次取数）。
3. → **TLB（地址翻译 Cache，硬件 MMU 内）**：缓存最近的 VA→PA。残留：**TLB miss**、**进程切换 TLB 整体失效** → 衍生 **ASID（地址空间标识）** 避免全刷。
4. → 拿到物理地址后访存仍受**存储墙**拖累 → **数据 Cache**（局部性）。残留：**Cache 抖动、一致性**（链路见 ④ 故障溯源）。
5. → 若页表项无效 = **缺页**：触发**缺页异常进 OS** → 请求分页 + 页面置换。残留：**抖动 Thrashing** → 衍生**工作集模型**。

### 完整访问顺序（必须背到能默画）

```
VA ──拆──> [虚页号 VPN | 页内偏移]
           │
           ├─ 查 TLB ──命中──> PPN ─┐
           │         └未命中─> 查多级页表 ─有效─> PPN ─┤
           │                          └无效─> 缺页异常 → OS 调页 → 重执行
           ▼                                              │
        物理地址 PA = [PPN | 偏移] ───────────────────────┘
           │
           └─ 查 Cache(用 PA 的 Tag/Index/Offset) ─命中─> 取数
                                          └未命中─> 访主存 + 填 Cache
```

### 三条铁律辨析（高频陷阱）

- **TLB miss ≠ 缺页**：TLB 没有不代表页不在内存，可能页表里有。
- **Cache miss ≠ 缺页**：Cache miss 由**硬件**自动从主存补；缺页必须**进 OS** 异常处理程序。
- **缺页一定进 OS**；TLB/Cache 命中失败都是硬件自理（缺页是唯一要软件介入的）。

### 强化考点

- **计组**：Cache 三种映射、Tag/Index/Offset 位数划分、命中率、写直达/写回/写分配、**有效访问时间 EAT**。
- **OS**：VA 拆页号/偏移、多级页表层数→访存次数、缺页处理五步、置换算法(OPT/FIFO/LRU/Clock)、含缺页的 EAT。
- **命题坑**：① **访存次数计算**——TLB 命中=1 次访存取数；TLB 未命中+N 级页表=N+1 次；含缺页再叠加磁盘 IO；② VA/PA 位数、页大小决定偏移位数；③ **页大小与 Cache 行**关系（同义/别名问题，VIPT）；④ Belady 异常只在 FIFO 出现。

> 这是 408 最爱的跨科链。详见 [[计算机组成原理强化]] 第3章 + [[操作系统强化]] 第3章，二者必须合着看。

## 链路 4：用户 IO 系统调用 → 设备驱动 → DMA → 磁盘硬件

### 分步流程（缺陷迭代）

1. **硬约束起点**：外设慢且异步，CPU 若**逐字节轮询=纯浪费**。
2. **程序查询(轮询)**：CPU 死等设备就绪 → 缺陷：**CPU 利用率极低**。
3. **中断驱动**：设备好了发中断，CPU 平时干别的 → 缺陷：**每传一个字/字节就一次中断，中断开销大**（高速盘扛不住）。
4. **DMA**：成块传送，DMA 控制器直接在**设备↔主存**间搬数据，CPU 只在**传送前(初始化)和传送后(完成中断)**介入 → 缺陷：**DMA 与 CPU 争总线**（三种方式：停止 CPU 访存/周期窃取/交替访问）、**Cache 一致性**、**DMA 溢出**（见④）。
5. **通道**：进一步把 IO 控制权下放给独立处理机（大型机）。

### read() 完整数据流（必背）

```
用户 read(fd,buf,n)
  → trap 进内核（系统调用）
  → 文件系统按 inode/索引 定位磁盘物理块（链 5）
  → 设备驱动向磁盘控制器发命令
  → DMA 把数据搬到【内核缓冲区】
  → 传送完成 → 设备发中断
  → 中断服务程序：内核缓冲区 → 拷到【用户缓冲区】
  → 唤醒阻塞的进程，read 返回
```

### 缓冲演化

单缓冲 → 双缓冲（边搬边处理）→ 循环缓冲 → **缓冲池**（系统级、按用途动态分配）。**目的**：缓解 CPU 与设备速度不匹配、减少中断次数、解耦生产消费。

### 强化考点

- **计组**：DMA 三种方式及其与 CPU 争总线的代价、DMA 传送过程（预处理→数据传送→后处理）、中断响应阶段、总线周期。
- **OS**：IO 控制方式四级演化、缓冲技术、设备独立性、IO 软件四层（用户层/设备无关层/驱动/中断处理）、SPOOLing。
- **命题坑**：① **DMA 以"块"为传送单位**，中断方式以"字/字节"为单位；② DMA **数据传送阶段不占用 CPU**，但**传送前后**仍需 CPU；③ 周期窃取是"抢一个存取周期"不是抢整个总线占用期；④ DMA 请求**优先级高于**中断请求（数据会丢，时间敏感）。

> 详见 [[计算机组成原理强化]] 第7章 + [[操作系统强化]] 第5章，硬件通路与管理策略合看。

## 链路 5：open/read 系统调用 → inode → 磁盘物理块映射

### 分步流程（缺陷迭代）

1. **硬约束起点**：磁盘按**块**存储，文件是**逻辑字节流** → 必须有"**逻辑块号 → 物理块号**"映射结构。
2. **连续分配**：文件占连续块 → 优点随机访问快；缺陷：**外部碎片、文件难增长**。
3. **链接分配**：隐式链接（块尾存下一块指针）→ 缺陷：**不能随机访问、指针损坏全断**；显式链接 **FAT**（指针集中成表）→ 缺陷：**FAT 常驻内存、大盘 FAT 巨大**。
4. **索引分配**：每文件一个索引块记录所有物理块 → 支持随机访问 + 动态增长；缺陷：**小文件浪费整个索引块、大文件单索引块装不下**。
5. → **多级/混合索引（Unix inode）**：直接块 + 一级/二级/三级间接块。小文件直接块够用，大文件靠间接块扩展。**这是"小文件省、大文件能撑"的最终折中**。

### open/read 落地

```
open("/a/b/file")
  → 逐级目录检索（按名查找，IO 多次）→ 找到 file 的 inode/FCB
  → 装入【系统级打开文件表】+【进程级打开文件表】→ 返回 fd
read(fd, ...)
  → 用 fd 查进程打开文件表 → 指向 inode
  → 逻辑块号 →(inode 索引)→ 物理块号 →(链 4)→ DMA 读盘
  → 后续 read 不再按名检索（这是 open 的全部意义）
```

### 强化考点

- **计组**：磁盘物理结构（柱面/磁头/扇区 CHS）、**寻道时间 + 旋转延迟 + 传输时间**、磁盘地址格式、磁盘调度（FCFS/SSTF/SCAN/C-SCAN）的磁头移动距离。
- **OS**：三种物理结构对比、**inode 混合索引最大文件/访问块数计算**、目录检索、两级打开文件表、链接计数与删除。
- **命题坑**：① **open 做一次按名检索、read 不再查名**（fd 的价值）；② 混合索引"访问第 K 逻辑块需几次磁盘 IO"按落在直接/一级/二级间接计算；③ 删除文件改：目录项、inode、链接计数、空闲块位示图；④ 硬链接共享 inode（计数）、软链接是独立文件存路径。

> 详见 [[计算机组成原理强化]] 第3章(磁盘)/第7章 + [[操作系统强化]] 第4章。

# ③ 故障 & 丢分点溯源（对标软件测试 bug 溯源：底层根源 → 触发条件 → 上层兜底+公式）

> 408 综合题最爱在"系统为什么崩/为什么慢"上设坑。每个故障都能溯回某个硬件短板或算法原生漏洞。

## 故障 1：颠簸 Thrashing（抖动）

- **底层根源**：分配给进程的**页框数 < 工作集**，缺页率骤升，CPU 几乎全在换页、几乎不干活。本质是链 3 "请求分页"的原生漏洞。
- **触发条件**：多道程序度过高、局部性差、置换算法不当。
- **上层兜底**：**工作集模型**（驻留集 ≥ 工作集大小）、**缺页率 PFF 算法**（缺页率高则增页框、低则减）、挂起部分进程降低多道度。
- **公式**：`有效访问时间 EAT = (1-p)·访存时间 + p·缺页处理时间`，p=缺页率；p 极小但缺页处理是 ms 级，故 EAT 对 p 极敏感。

## 故障 2：死锁 Deadlock

- **底层根源**：四个必要条件**同时成立**——**互斥、占有并等待、不可剥夺、循环等待**。
- **触发条件**：PV 操作顺序错（多 mutex 嵌套加锁顺序不一致）、资源分配图成环。
- **上层兜底**：**预防**（破坏四条件之一，如资源有序分配破环）/ **避免**（**银行家算法**：找安全序列）/ **检测+解除**（资源分配图化简）。
- **判定**：死锁的充要——资源分配图**不可完全化简**；银行家用 `Need = Max − Allocation`，逐个找 `Need ≤ Available` 的进程。

## 故障 3：Cache 抖动（颠簸）

- **底层根源**：**映射冲突**——直接映射下多个块争同一 Cache 行，反复互相替换；或块太小局部性利用差。链 3 "Cache"的原生漏洞。
- **触发条件**：访问步长 = 组数的整数倍的数组（典型大题陷阱）、嵌套循环行优先/列优先不当。
- **上层兜底**：**组相联**（降低冲突）、合适的块大小、**循环分块(blocking)**、数组维度补齐避免步长共振。

## 故障 4：缺页连锁失效 / Belady 异常

- **底层根源**：置换算法不当。**FIFO 的 Belady 异常**——页框增多反而缺页增多（FIFO 不满足栈算法性质）。
- **触发条件**：用 FIFO 且访问串特定（如 1,2,3,4,1,2,5,1,2,3,4,5）。
- **上层兜底**：**LRU/Clock**（满足栈算法，无 Belady）、足够页框、工作集。
- **辨析**：**OPT 最优但不可实现**（需未来信息）；**LRU 近似 OPT 但开销大**（要时间戳/栈）；**Clock(二次机会)** 是 LRU 的低成本近似。

## 故障 5：DMA 溢出（数据丢失）

- **底层根源**：DMA 传送速率 > 主存/缓冲区接收速率，或 CPU 对 DMA 请求响应不及时，高速设备的数据在缓冲区被覆盖。链 4 "DMA"的原生漏洞。
- **触发条件**：高速设备 + 总线竞争激烈 + 缓冲不足。
- **上层兜底**：**DMA 请求优先级高于中断**、**周期窃取**保证及时、足够大的**双/多缓冲**、限制并发 DMA 通道数。

## 故障 6：IO 中断嵌套异常

- **底层根源**：中断处理期间又来更高优先级中断，若**中断屏蔽字设置不当 / 现场未保存好**，会导致现场破坏或优先级倒置。
- **触发条件**：多重中断、关中断时机错、中断服务程序未及时保存通用寄存器。
- **上层兜底**：**中断屏蔽字 + 优先级**控制嵌套层次、**关中断保存断点/现场**（硬件保存 PC/PSW，软件保存通用寄存器）、明确"中断响应阶段 vs 中断服务阶段"分工。
