## 1. 总体结论：这两科在 408 里的地位

操作系统和计组都不是“背概念就行”的科目。408 喜欢考它们的原因是：它们天然能把硬件、软件、地址、I/O、并发、性能串起来。

### 操作系统

操作系统的核心不是“名词解释”，而是三条主线：

1. 进程如何被管理：进程/线程、调度、同步互斥、死锁。
2. 内存如何被抽象：页表、地址变换、缺页、页面置换。
3. 外设与文件如何被组织：文件系统、磁盘、I/O、中断、DMA、缓冲。

### 计算机组成原理

计组的核心不是“硬件零件”，而是三条主线：

1. 数据怎么表示和运算：补码、浮点、溢出、ALU。

3. 存储和 I/O 怎么支撑性能：Cache、主存、TLB、总线、DMA、中断。

## 2. 操作系统章节重点

### OS 第1章：概述、运行机制、中断/异常、系统调用、引导

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 低到中，通常不单独大题，但可嵌入综合题 |
| 常驻考点 | 用户态/内核态、特权指令、系统调用、中断与异常、操作系统启动/引导流程 |
| 低频考点 | 批处理、分时、实时系统分类；OS 发展史 |
| 核心考察 | 能不能分清“谁触发、谁处理、是否陷入内核、是否可屏蔽、是否同步发生” |
| 学习优先级 | A-，概念必须清楚，但不用在名词分类上过度消耗 |

你要掌握的判断链：

- 系统调用：用户程序主动请求内核服务，本质是陷入内核。
- 异常：CPU 执行指令时同步发现问题，如缺页、溢出、非法指令。
- 中断：外部设备异步通知 CPU，如键盘、磁盘、定时器。
- 引导流程：ROM 中固化的引导程序先执行，再进入磁盘引导、分区引导、操作系统初始化。

大题可能怎么考：

- 2021 年这类题会把“引导流程、磁盘格式化、文件系统结构”串起来。
- 常见问法不是让你背定义，而是问“哪个阶段做什么、谁先谁后、扇区/分区/文件系统分别在哪一步形成”。

### OS 第2章：进程、线程、调度、同步互斥、死锁

| 维度 | 结论 |
|---|---|
| 选择题频率 | 极高 |
| 大题频率 | 极高 |
| 常驻考点 | 进程状态转换、PCB、线程共享资源、调度算法、信号量 PV、生产者消费者、读者写者、死锁条件、银行家算法 |
| 低频考点 | 多处理器调度细节、管程的复杂实现细节 |
| 核心考察 | 把“资源数量、执行顺序、互斥关系、同步关系”翻译成状态变化或信号量代码 |
| 学习优先级 | S，OS 第一优先级 |

选择题常考：

- 进程状态：就绪、运行、阻塞之间谁能直接转谁。
- 进程和线程：共享地址空间、文件描述符、代码段，线程私有栈/寄存器。
- 调度算法：FCFS、SJF、优先级、时间片轮转的周转时间/等待时间。
- 死锁：四个必要条件、资源分配图、银行家算法安全序列。

大题常考：

- 信号量同步互斥：先判断哪些操作互斥，哪些操作有先后。
- 调度计算：画时间轴，算完成时间、周转时间、带权周转时间。
- 死锁/银行家：Need、Available、Allocation 表格推安全序列。

本章最关键的能力：

> 看到题，不急着写 PV。先画出“共享资源”和“先后条件”。互斥用 mutex，同步用 empty/full/order 之类的信号量。

### OS 第3章：内存管理、页表、虚拟内存、页面置换

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 高 |
| 常驻考点 | 分页、分段、页表项、地址转换、TLB、缺页异常、页面置换、有效访问时间 |
| 低频考点 | 覆盖、交换的历史细节；复杂段页式管理 |
| 核心考察 | 把虚拟地址拆成页号/页内偏移，再沿 TLB、页表、主存、外存走完整访问流程 |
| 学习优先级 | S，OS 第二优先级，也是计组交叉核心 |

选择题常考：

- 页号位数、页内偏移位数、页表大小。
- TLB 命中/未命中、缺页/不缺页的区别。
- 页面置换算法：OPT、FIFO、LRU、Clock。
- 抖动、工作集、驻留集。

大题常考：

- 地址转换：虚拟地址 -> 页号 + 页内偏移 -> 页表项地址 -> 物理地址。
- 缺页处理：发生缺页异常，OS 分配页框，读入页面，更新页表项，再重新执行指令。
- 与 Cache 结合：数组访问、Cache 命中率、页大小、局部性。

本章必须和计组连起来：

- TLB 属于硬件加速地址转换，但页表由操作系统维护。
- 缺页是异常，不是普通 Cache miss。
- Cache miss 由硬件处理；缺页要进入 OS 缺页异常处理程序。

### OS 第4章：文件系统、目录、文件分配、空闲空间管理

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 中 |
| 常驻考点 | 文件控制块、目录项、索引分配、链接分配、连续分配、位示图、空闲链表、打开文件表 |
| 低频考点 | 具体文件系统实现名词的边角细节 |
| 核心考察 | 文件名如何映射到磁盘块，空闲块如何管理，打开/读写/删除会改哪些结构 |
| 学习优先级 | A |

选择题常考：

- 目录项和 FCB/inode 的关系。
- 文件打开后，后续 read/write 是否还需要按名查找。
- 删除文件时目录项、索引节点、链接计数、磁盘块如何变化。
- 位示图空间计算。

大题可能考：

- 给定索引结构，算文件最大长度或访问某逻辑块需要几次磁盘访问。
- 给定空闲块管理方式，判断分配/回收过程。

本章和第5章强相关：

- 文件系统决定“逻辑块怎么找”。
- I/O 和磁盘调度决定“找到后怎么读得快”。

### OS 第5章：I/O 管理、缓冲、磁盘调度、设备管理

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 中到低 |
| 常驻考点 | 程序查询、中断驱动、DMA、缓冲、设备独立性、磁盘调度、SCAN/C-SCAN |
| 低频考点 | 通道技术细节、设备分配复杂策略 |
| 核心考察 | CPU、内存、设备控制器、DMA 控制器、内核缓冲区之间的数据流 |
| 学习优先级 | A-，和计组 I/O 一起学 |

选择题常考：

- DMA 数据通路：设备接口和主存之间传数据，CPU 不逐字节搬运。
- 中断处理：保存断点、进入内核、执行中断服务例程。
- 键盘/磁盘数据先到内核缓冲区，再到用户缓冲区。
- 磁盘调度：FCFS、SSTF、SCAN、C-SCAN 的磁头移动距离。

学习时不要把 OS I/O 和计组 I/O 分开背：

- 计组问硬件通路：中断请求、DMA 控制器、总线周期、接口。
- OS 问管理策略：缓冲、设备分配、磁盘调度、文件读写流程。

## 3. 计算机组成原理章节重点

### 计组第1章：概述、性能指标、层次结构

| 维度 | 结论 |
|---|---|
| 选择题频率 | 中到高 |
| 大题频率 | 低 |
| 常驻考点 | CPI、主频、时钟周期、CPU 时间、MIPS、FLOPS、吞吐率、响应时间 |
| 低频考点 | 冯诺依曼结构的纯概念题 |
| 核心考察 | 能不能把“时间 = 指令数 × CPI × 时钟周期”熟练变形 |
| 学习优先级 | A-，性价比高 |

必会公式：

```text
CPU执行时间 = 指令条数 × CPI × 时钟周期
CPU执行时间 = 指令条数 × CPI / 主频
MIPS = 指令条数 / (执行时间 × 10^6)
```

这章题通常不难，但很容易因为单位错失分。所有 Hz、ns、ms、MB/s、GB/s 都要统一单位。

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 中到低 |
| 常驻考点 | 补码、移码、浮点数、IEEE 754、溢出判断、移位、ALU 标志位 |
| 低频考点 | 原码一位乘除法的复杂手算细节 |

| 学习优先级 | A |

选择题常考：

- short/int 的机器数表示。
- 补码加减法的 OF/CF 判断。

学习抓手：

- 无符号数看 CF。

### 计组第3章：存储系统、Cache、主存、虚拟存储

| 维度 | 结论 |
|---|---|
| 选择题频率 | 极高 |
| 大题频率 | 高 |
| 常驻考点 | Cache 映射、Tag/Index/Offset、命中率、替换策略、写策略、主存芯片扩展、TLB |
| 低频考点 | 磁表面存储器的纯参数记忆 |

| 学习优先级 | S |

选择题常考：

- 直接映射、全相联、组相联。

- 写直达、写回、写分配、不写分配。

大题常考：


### 计组第4章：指令系统、寻址方式、机器级程序

| 维度 | 结论 |
|---|---|
| 选择题频率 | 极高 |
| 大题频率 | 高，常作为 CPU 大题的前置 |
| 常驻考点 | 指令格式、操作码扩展、寻址方式、立即数、偏移量、机器码、RISC/CISC |
| 低频考点 | 某些具体汇编伪指令 |

| 学习优先级 | S |

选择题常考：

- CPU 能直接执行的是机器指令，不是高级语言/汇编/伪指令。
- 操作码和地址码长度变化。
- 寻址方式：立即、直接、间接、寄存器、基址、变址、相对。

大题常考：

- 给定指令格式，分析寄存器编号位数、立即数字段、shamt 位数。
- 给定机器码，判断是哪类指令，算访存地址。
- 与数据通路结合，写控制信号取值。

### 计组第5章：CPU、数据通路、控制器、流水线、中断异常

| 维度 | 结论 |
|---|---|
| 选择题频率 | 极高 |
| 大题频率 | 极高 |
| 常驻考点 | 数据通路、控制信号、硬布线/微程序、指令周期、流水线、数据冒险、控制冒险、中断异常 |
| 低频考点 | 微程序控制器过细的字段编码 |
| 核心考察 | 一条指令在 CPU 内部每一步怎么走，哪里需要控制信号，哪里会阻塞 |
| 学习优先级 | S+，计组第一优先级 |

选择题常考：

- PC、IR、MAR、MDR、GPR、ALU、MUX 属于什么部件。
- 取指、译码、执行、访存、写回。
- 流水线加速比、吞吐率、阻塞周期。
- 数据冒险：RAW 最常见；转发、插入气泡、编译器调整。
- 中断响应阶段和中断服务程序阶段分别做什么。

大题常考：

- 给一个简化 CPU 数据通路图，问某条指令控制信号。
- 给指令序列，分析流水线阻塞。
- 给机器码和寄存器/内存内容，算执行结果、溢出标志、访存地址。

这章与 OS 的联系：

- 进程切换本质上要保存/恢复 PC、寄存器、状态字等上下文。
- 系统调用/异常/中断最终都要靠 CPU 的异常中断机制进入内核。

| 维度 | 结论 |
|---|---|
| 选择题频率 | 中 |
| 大题频率 | 低到中 |

- 先掌握单位换算。

| 维度 | 结论 |
|---|---|
| 选择题频率 | 高 |
| 大题频率 | 中 |
| 常驻考点 | 程序查询、中断方式、DMA、I/O 接口、中断响应、中断屏蔽、优先级 |

| 学习优先级 | A |

选择题常考：

- 中断响应时保存断点和程序状态字。
- 通用寄存器保存通常由中断服务程序软件完成。
- DMA 方式下，数据传输通路在设备接口和主存之间。
- 中断屏蔽字、中断优先级、中断嵌套。

- 计组：硬件怎么通知 CPU，DMA 怎么占总线。

## 4. 选择题与大题的高频分布

### 选择题高频

操作系统选择题常驻：

- 系统调用、中断、异常、用户态/内核态。
- 进程状态、线程共享资源、调度算法。
- 死锁必要条件、银行家算法基本判断。
- 页表、TLB、缺页、页面置换。
- 文件目录、打开文件表、空闲块管理。
- DMA、中断驱动、磁盘调度。

计组选择题常驻：

- CPI/主频/CPU 时间。
- 补码、浮点、溢出标志。
- Cache 映射、主存芯片、TLB。
- 指令格式、寻址方式、机器指令。
- 数据通路部件、流水线冒险。
- 总线带宽、DMA、中断响应。

### 大题高频

操作系统大题高频：

1. 信号量同步互斥。
2. 调度算法计算。
3. 页式虚拟存储和缺页处理。
4. 文件系统索引/磁盘块计算。
5. 死锁/银行家算法。

计组大题高频：

1. 指令格式 + 机器码 + 数据通路控制信号。
2. 流水线冒险、阻塞、转发。
3. Cache 地址划分和命中率。
4. 数据表示、溢出、ALU 标志。
5. 虚拟存储、TLB、页表、Cache 联合题。

## 5. 两科之间最重要的联系

### 联系 1：中断、异常、系统调用

```text
外部设备中断 / CPU执行异常 / 用户系统调用
        ↓
CPU切换到内核态
        ↓
保存现场，转入中断/异常/系统调用处理程序
        ↓
OS完成服务或处理
        ↓
恢复现场，返回用户态或调度其他进程
```

计组负责解释“CPU 怎么跳过去”；OS 负责解释“内核进去以后干什么”。

### 联系 2：虚拟内存、TLB、页表、Cache

```text
进程给出虚拟地址
        ↓
MMU 查 TLB
        ↓
TLB 命中：得到物理页框
TLB 未命中：查页表
        ↓
页表有效：得到物理地址
页表无效：缺页异常，进入 OS
        ↓
物理地址访问 Cache/主存
```

这是 OS 第3章和计组第3章的核心交叉。你必须能说清：

- TLB miss 不一定缺页。
- Cache miss 不一定进入 OS。
- 缺页一定进入 OS 异常处理。

### 联系 3：I/O、DMA、磁盘、文件系统

```text
用户 read()
        ↓
系统调用进入内核
        ↓
文件系统根据目录/索引找到磁盘块
        ↓
设备驱动发起 I/O
        ↓
DMA 把数据搬到内核缓冲区
        ↓
中断通知 CPU
        ↓
内核把数据交给用户缓冲区
```

这里计组解释 DMA、中断、总线；OS 解释系统调用、文件系统、缓冲、调度。

### 联系 4：进程切换与 CPU 上下文

OS 说“进程切换”，计组问“切换的到底是什么”：

- PC：下一条要执行的指令。
- 通用寄存器：当前计算状态。
- PSW/状态寄存器：条件码、中断允许位、当前态等。
- 页表基址寄存器：切换地址空间。

所以 OS 进程切换题，经常能追到计组寄存器和地址转换。

## 6. 你应该怎么学：优先级路线

### 第一梯队：必须先拿下

1. 计组第5章 CPU/数据通路/流水线。
2. OS 第2章 进程/调度/同步/死锁。
3. 计组第3章 Cache/存储系统。
4. OS 第3章 内存管理/虚拟内存。
5. 计组第4章 指令系统。

理由：这些章节同时覆盖选择题和大题，而且互相连接。408 的综合题往往就在这里出。

### 第二梯队：稳定拿分

1. OS 第4章 文件系统。
2. OS 第5章 I/O 和磁盘。
3. 计组第7章 I/O。
4. 计组第2章 数据表示与运算。
5. 计组第1章 性能指标。

理由：选择题高频，题型相对固定，认真刷题回报高。

### 第三梯队：不要花过多时间

1. OS 发展史、系统分类细枝末节。
2. 计组总线标准细节。
3. 通道、外围设备历史实现。
4. 微程序控制器特别细的编码设计。

理由：不是不考，而是不值得作为第一轮主攻。

## 7. 王道刷题计划

### 阶段 1：建立框架，别急着刷整套真题

时间：3-5 天

任务：

- OS：第1章快速过，第2章重点学到信号量前。
- 计组：第1章性能公式、第2章补码浮点、第3章 Cache 基础。
- 每章只做王道基础选择题，不追求速度。

目标：

- 能说出每章主线。
- 能把错题归因到“概念不清、公式不会、流程不会、计算粗心”。

### 阶段 2：主攻核心大题

时间：10-14 天

顺序：

1. OS 第2章：调度 + 信号量 + 死锁。
2. 计组第5章：数据通路 + 流水线。
3. OS 第3章 + 计组第3章：页表/TLB/Cache 联合。
4. 计组第4章：指令格式、寻址、机器码。

刷题方法：

- 每个专题先做王道例题，再做课后综合题。
- 大题必须手写过程，不要只看答案。
- 每道错题写一句“它真正考什么”。

### 阶段 3：文件、I/O、总线补强

时间：5-7 天

顺序：

1. OS 第4章 文件系统。
2. OS 第5章 I/O/磁盘。
3. 计组第7章 I/O。
4. 计组第6章 总线。

目标：

- 能完整讲出 `read()` 从用户态到磁盘再回用户态的流程。
- 能区分 DMA、中断、缓冲、文件系统分别属于哪层。
- 能做磁盘调度和总线带宽计算。

### 阶段 4：真题专题复盘

时间：7-10 天

做法：

- 不按年份一套一套硬刷，先按专题刷。
- 专题顺序：进程同步 -> CPU/流水线 -> Cache/页表 -> 指令系统 -> 文件/I-O -> 数据表示。
- 每个专题整理一页模板。

模板例子：

```text
信号量题模板：
1. 共享资源是什么？
2. 哪些操作互斥？

```
