## 计算机软件

**1、系统软件和应用软件**

计算机软件，一般分为系统软件和应用软件
系统软件：是一组保证计算机系统高效、正确运行的基础软件，通常作为系统资源提供给用户使用。包括操作系统，数据库管理系统，语言处理系统(比如编译器)，分布式软件系统，网络软件系统，标准库系统，服务性系统(比如连接程序)。
应用软件：是指用户为解决某个应用领域中的各类问题而编制的程序。包括各种科学计算类程序，工程设计类程序，数据统计与处理程序。
**2、三个级别的语言**

**机器语言。** 又称二进制代码语言，需要编程人员记忆每条指令的二进制编码。机器语言是计算机唯一可以直接识别和执行的语言。
**汇编语言。** 汇编语言用英文单词或其缩写代替二进制的指令代码，更容易为人们记忆和理解。使用汇编语言编辑的程序，必须经过一个称为汇编程序的系统软件的翻译，将其转换为计算机的机器语言后，才能在计算机的硬件系统上执行。
**高级语言**。高级语言(如 C、C++、Java 等)是为方便程序设计人员写出解决问题的处理方案和解题过程的程序。通常高级语言需要经过编译程序编译成汇编语言程序，然后经过汇编操作得到机器语言程序，或直接由高级语言程序翻译成机器语言程序。
**注：数据库管理系统和数据库系统是不一样的。数据库管理系统是系统软件。而数据库系统一般是由数据库，数据库管理系统，数据库管理员和应用系统构成。所以只能说它里面有系统软件，但并不能说它为系统软件。**

**机器语言是唯一可以控制 CPU 的语言，** 但是它的符号不利于人识别和书写，为了方便理解和记忆，将机器语言换一些通俗易懂的符号，这就变成了 **汇编语言**。一般来说在在编译器中高级语言先转换为汇编在转换为机器语言，也有直接转换为机器语言的情况。

**机器语言是计算机唯一可以直接执行的语言，汇编语言用助记符编写，以便记忆。**

而正则语言是编译原理中符合正则文法的语言。

**各种翻译程序的概念：**

汇编程序（汇编器）：将汇编语言程序翻译成机器语言程序。
解释程序（解释器）：将源程序中的语句按执行顺序逐条翻译成机器指令并立即执行。（Python、Shell）
编译程序（编译器）：翻译一句执行一句，边翻译边执行：由高级语言转化为汇编语言的过程称为编译，把汇编语言源程序翻译成机器语言程序的过程称为汇编。（exe，比如 C/C++、Java）
**软件和硬件的逻辑功能等价性：**

对某一个功能来说，既可以有软件实现又可以由硬件实现，从用户的角度来看，它们在功能上是等价的。这一等价性被称为软/硬件逻辑功能等价性。（eg：浮点数运算）

**1.交换律: 对于任意两个位 a 和 b，有 ab = bÐa0**

**3.恒等律: 对于任意位 a ，有**
**aФ0 = a**
**4.自反律: 对于任意位 a ，有**
**aФa = 0 。**

**片选线的基本作用**

1. **芯片选择功能**

- **多芯片系统中的寻址**：当多个存储芯片共享同一总线时，CS 信号决定哪个芯片响应当前操作
- **示例**：一个系统连接了 8 片 Flash，通过 3-8 译码器生成 8 个 CS 信号，每次只有 1 个 CS 为有效电平

**2. 功耗管理**

- **非选中状态的低功耗**：当 CS 无效时，芯片进入待机模式（待机电流可降低至工作电流的 1/1000）
- **示例**：SDRAM 的 CS 无效时，自动进入自刷新状态

## CISC 和 RISC

![image-20250511150915273](https://cdn.jsdelivr.net/gh/kasahuki/os_test@main/img/image-20250511150915273.png)



**指令 微指令 微操作**

**指令** 是计算机程序中的基本操作单位，它定义了计算机硬件执行的具体操作。

指令通常包括操作码（Opcode）和操作数（Operands），操作码指示了执行的操作类型（如加法、乘法、存储等），操作数则是操作所需的数据或地址。

在高级语言或汇编语言中编写的代码会被编译成一系列机器指令，这些指令直接由中央处理器（CPU）执行。

**主存（主内存，Main Memory）通常存储的是指令**

| 类型        | 内容                   | 说明                                    |
| ----------- | ---------------------- | --------------------------------------- |
| **指令**    | `ADD R1, R2, R3`       | `R1 = R2 + R3`，把 R2 和 R3 相加存到 R1 |
| **微指令 1** | `Load R2 → ALU.inputA` | 把 R2 寄存器的值送到 ALU 的 A 输入端    |
| **微指令 2** | `Load R3 → ALU.inputB` | 把 R3 的值送到 ALU 的 B 输入端          |
| **微指令 3** | `ALU.perform(ADD)`     | 执行 ALU 加法操作                       |
| **微指令 4** | `ALU.output → R1`      | 将 ALU 输出写回到寄存器 R1              |

---



**微指令** 是指一组更加底层、更加具体的控制信号序列，用于控制计算机中的微操作或硬件级别的操作。

微指令通常由控制单元（Control Unit）根据指令集架构（Instruction Set Architecture, ISA）中的指令来生成，它们直接控制硬件中的寄存器、算术逻辑单元（ALU）、存储单元等。

微指令的执行可以理解为对硬件内部具体操作的控制，如存储器的地址选择、数据传输路径的控制等。

**微指令则通常存储在控制单元内部的控制存储器（Control Memory）中，用于实现对硬件中各种微操作的精细控制。**

**ALU 输入**：必须保证两路数据在时钟边沿同步到达（通过寄存器/时序控制实现）。

**ALU 输出**：必须加暂存器（输出寄存器），以保证结果稳定、避免总线竞争，并支持流水线处理

**结果未稳定就被使用**

- 假设 ALU 正在做加法 `R1 + R2 → R3`。
- ALU 内部的加法器电路传播延迟可能是 **10ns**。
- 在这 10ns 内，输出线上的信号还在“抖动”，不是稳定的结果。
- 如果这时结果直接放到总线上，其他单元可能读到的是错误值。

👉 **暂存器的作用**：在时钟沿到来时，锁存 ALU 输出，保证总线上拿到的是稳定结果




 输入端的信号不会同时到达 ALU 内部。例如：

- 输入 A 的信号先到；
- 输入 B 的信号稍微延迟几纳秒才到。
   在这段时间里，ALU 的输出就会根据“未完成”的输入先产生一个临时结果，随着输入逐步到齐，输出会不断变化。

**这种变化过程就是“抖动”**（Glitch）：
 在最终正确结果稳定前，输出可能会经历几次错误的高低电平变化

### **微操作是什么？**

- 微操作就是对寄存器级别的一次最小操作。
- 例如：
  - `PC → MAR`（把程序计数器送到存储器地址寄存器）
  - `MDR → IR`（把存储器数据寄存器的内容送到指令寄存器）
  - `AC ← AC + DR`（累加器加上数据寄存器内容）

> 这些都是最细粒度的操作，不能再拆分



### **控制信号是什么？**

- 微操作的实现，靠硬件控制信号来驱动。
- 比如：
  - 想执行 `PC → MAR`，硬件必须发出两个控制信号：
    - **PC 输出使能**
    - **MAR 写入使能**
- 控制信号来自 **控制器**（硬布线或微程序），它会根据当前指令的操作码 + 时序，生成对应的控制信号。

所以可以理解为：

> **微操作是逻辑动作，控制信号是物理手段**



一个微操作至少需要一个时钟周期

### **IR（Instruction Register，指令寄存器）是什么？**

- **定义**：IR 是 CPU 内部的一个关键寄存器，用来 **存放当前正在执行的指令**。
- **来源**：指令从主存（内存）取出时，首先会进入 **MDR（存储数据寄存器）**，然后再传送到 **IR**。
- **作用**：一旦指令进入 IR，就说明这条指令已经被 CPU 取出，接下来 CPU 就会根据 IR 的内容来进行译码和执行

### ==微程序控制器的基本原理==

控制器发出的 **控制信号的组合** 就是微命令

![image-20250822104448568](https://cdn.jsdelivr.net/gh/kasahuki/os_test@main/img/image-20250822104448568.png)



内存是由 **存储单元阵列** 组成的（行 × 列），每个存储单元都可以看作一个小的触发器（存 bit）。

- 如果直接把 MAR 输出的二进制信号连到内存，所有存储单元都会“看到”这个地址。
- 但我们只希望 **唯一的一个单元** 被选中进行读/写。

👉 所以，需要一个 **地址译码器（Decoder）** 来做“二进制地址 → 独热码（one-hot）”的转换：

- MAR 输出 `n` 位地址（比如 16 位 → 可以寻址 2^16 = 65536 个单元）。
- 译码器把这个 16 位地址 **翻译成唯一的 1 条选通信号**，只激活一个存储单元，其余全为 0。

#### 影响因素和分类

**IF、ID、EX、M、WB** 其实是 **处理器流水线（Pipeline）** 中对 **指令周期的进一步细分阶段**。它和你前面问的“取指周期、间址周期、执行周期、访存周期”是两种不同层次的划分：

- **取指/间址/执行/访存** → 是传统冯诺依曼体系结构里对整个 **指令周期（Instruction Cycle）** 的功能性划分。
- **IF/ID/EX/M/WB** → 是现代 CPU 在流水线技术中，把“执行一条指令”的工作拆成更细的流水阶段。

---



五级流水线阶段解释

1. **IF (Instruction Fetch, 取指阶段)**
   - 从内存（指令 Cache/主存）中取出指令。
   - 更新 PC（程序计数器）。
2. **ID (Instruction Decode, 指令译码阶段)**
   - 对指令进行译码，确定操作类型、寄存器地址、操作数位置等。
   - 读取寄存器堆（寄存器文件）中的源操作数。
3. **EX (Execute, 执行阶段)**
   - 由 ALU（算术逻辑单元）进行运算，例如加法、逻辑比较、地址计算等。
   - 如果是分支指令，会在这里计算跳转目标。
4. **M (Memory Access, 访存阶段)**
   - 如果指令需要访问数据内存（如 load/store），就在这一阶段完成。
   - 其他类型的指令可能跳过这一阶段。
5. **WB (Write Back, 回写阶段)**
   - 把运算结果或访存结果写回寄存器堆，以供后续指令使用。

---



**传统指令周期**（取指、间址、执行、访存）
 → 更偏概念化，适合解释计算机组成原理里的“顺序执行”。

**流水线阶段**（IF、ID、EX、M、WB）
 → 更偏现代 CPU 实现细节，用于解释“如何并行执行多条指令”

数据旁路是一种 **硬件解决方案**，核心思想是 **不等待写回阶段，把结果直接送到需要它的流水线阶段**。

**原理：**

- 当 EX 阶段需要一个寄存器的值，而这个值还没回写 WB 阶段时：
  - 从 **EX/MEM 或 MEM/WB 流水线寄存器** 直接取出结果
  - 送入 ALU 输入端，绕过寄存器堆
- 因此指令 **无需停顿**，冲突被消除

### 五段式指令流水线

![image-20250824230326813](https://cdn.jsdelivr.net/gh/kasahuki/os_test@main/img/image-20250824230326813.png)

**IF（Instruction Fetch，取指令）**

- 从指令存储器（内存或 Cache）中取出指令。
- 使用 PC（程序计数器）提供地址，然后 PC 自增。
- 关键数据：PC、指令寄存器（IR）。

✅ 输出：指令 IR。







------

**ID（Instruction Decode，译码/读寄存器）**

- 对 IR 中的二进制指令进行译码，识别指令类型（算术、访存、跳转等）。
- 从寄存器堆中读出源操作数（rs、rt）。
- 计算分支目标地址（用于分支预测/转移）。

✅ 输出**：控制信号、源操作数、目标寄存器号。**



ALU（算术逻辑单元）执行什么操作，是由 **控制信号（Control Signals）** 决定的，而这些控制信号来源于 **指令译码阶段（ID）** 的控制单元



ID阶段的寄存器堆是指在指令解码阶段（Instruction Decode, ID）中使用的寄存器集合。寄存器堆用于存储和管理处理器中的数据，允许指令在执行时快速访问所需的操作数。



在这个阶段，处理器会解码指令并确定需要访问哪些寄存器。寄存器堆的设计和实现对处理器的性能有着重要影响，因为它直接关系到数据的读取和写入速度。



寄存器堆通常由多个寄存器组成，每个寄存器可以存储一定数量的位（如32位或64位）。在ID阶段，寄存器堆的内容会被读取，以便为后续的执行阶段提供必要的数据。

---



**EX（Execute，执行/运算）**

- ALU 执行实际操作：
  - 算术运算（加减乘除）
  - 逻辑运算（与或非异或）
  - 地址计算（基址 + 偏移）
- 如果是分支指令，此时判断条件是否满足。

✅ 输出：ALU 结果（可能是数据或地址）。

------

**MEM（Memory Access，访存）**

- 如果指令是 **load/store**，需要访问数据存储器：
  - load：取出数据
  - store：写入数据
- 如果是算术/逻辑指令，则直接跳过此阶段。

✅ 输出：内存数据 or ALU 结果。

------

**WB（Write Back，写回寄存器）**

- 将结果写回寄存器堆（如 `rd` 或 `rt`）。
- load 的数据 / ALU 的结果最终存入寄存器，供后续指令使用。

✅ 输出：更新后的寄存器堆。

**把运算结果存回寄存器堆**

- 之前 ALU 或访存阶段得到的结果，需要写回寄存器，才能被后续指令使用

在五段流水线里，每一条指令在各阶段之间都有 **流水线寄存器（Pipeline Register）**

| 阶段间   | 寄存器名称 | 保存内容                                   |
| -------- | ---------- | ------------------------------------------ |
| IF → ID  | IF/ID      | 取出的指令及 PC                            |
| ID → EX  | ID/EX      | 译码后的指令信息、读取的寄存器值、立即数等 |
| EX → MEM | EX/MEM     | ALU 结果、目标寄存器地址等                 |
| MEM → WB | MEM/WB     | 内存读出数据或 ALU 结果                    |

**L1 在 EX 阶段**，它的数据已经存放在 **ID/EX 寄存器** 中；

**L2 在 ID 阶段**，它也会读取寄存器堆的值并写入 **ID/EX 寄存器**。

ALU从寄存器（例如ID/EX流水线寄存器）中读取输入数据（通常是操作数） 又由于题目 要求同一个寄存器 不能同时读写 

所以要等L1 ALU读完 L2才能写入**ID/EX流水线寄存器**

**数据总线**是一组“物理导线”，这些导线本身没有固定“谁发谁收”，而是**根据操作（读 or 写）动态决定方向**

**控制总线**是一堆**专用的控制信号线**，每一根线的方向是固定的



---



地址总线的作用：**CPU 发出“我要访问的地址”**。

- 地址总线只需要从 **CPU → 存储器/外设** 单向传递。
- 比如 CPU 想读内存地址 `0x1000`，就把 `0x1000` 放到地址总线上，内存芯片看到这个地址后，才知道应该去哪个存储单元读写。
- 所以地址总线只要单向，没必要“反过来传地址”。

结论：**地址总线是单向的，从 CPU 指向外设/内存。**



数据总线是用来搬运数据的。数据既可能是：

- **CPU → 内存/外设**（写操作，比如 CPU 把一个数写入内存）。
- **内存/外设 → CPU**（读操作，比如 CPU 从内存取指令，或从硬盘读数据）。

所以数据总线必须能双向传输。

结论：**数据总线是双向的，方向取决于读/写。**



控制总线就比较有意思，它包含了各种**控制信号**，方向上有双向的情况：

- **CPU → 外设/内存**
  - 读信号（RD）、写信号（WR）、内存读、内存写、I/O 读、I/O 写等。
  - 中断响应信号（INTA）等。
- **外设/内存 → CPU**
  - 中断请求（INT）、DMA 请求（DRQ）、准备好信号（READY）等。

所以控制总线的方向是 **双向的，但具体一根线是有方向性的**。

- 比如 `RD` 信号一定是 CPU 发的，
- 而 `INT` 信号一定是外设发的。

整体上，控制总线是一组既有输入线也有输出线的混合集合

### 3.DMA 方式



![image-20250823170514514](https://cdn.jsdelivr.net/gh/kasahuki/os_test@main/img/image-20250823170514514.png)

**I/O 设备地址（I/O Address）** —— CPU 发出的地址信号，决定访问哪个设备的哪个寄存器



**初始化阶段（CPU 参与）**

- CPU **配置** DMA 控制器：
  - **起始地址寄存器** → 要写入或读出的内存地址。
  - **计数寄存器** → 传输的数据字数（多少个字）。
  - **控制寄存器** → 传输方向（I/O → 内存，还是内存 → I/O），工作方式（突发、周期窃取、块传送等）。
- CPU 发出启动命令，让 DMA 控制器接管。



**总线申请与占用阶段**

- 外设准备好数据（或需要数据），向 DMA 控制器发出 **DMA 请求（DRQ）**。
- DMA 控制器向 CPU 发出 **总线请求（BR，Bus Request）**。
- CPU 执行完当前总线周期后，响应 **总线授权（BG，Bus Grant）**，把总线控制权交给 DMA 控制器。
- 此时 CPU 让出总线，暂停对内存的直接访问



**数据传送阶段（DMA 执行）**

DMA 控制器完全接管总线，直接在 **外设 ↔ 内存** 之间搬运数据：

1. **发出内存地址**（来自起始地址寄存器）。
2. **产生读/写控制信号**：
   - 若是 I/O→内存：DMA 先发出 I/O 读，再发内存写。
   - 若是 内存→I/O：DMA 先发内存读，再发 I/O 写。
3. **数据传输完成后**：
   - 地址寄存器 +1，指向下一个字。
   - 计数寄存器 -1，剩余字数减少。
4. **重复上述步骤**，直到计数器减到 0



**传输结束与恢复**

- 当计数寄存器 = 0，DMA 控制器完成一批传输：
  - 向外设发出 **DMA 结束信号（DACK）**。
  - 向 CPU 发送一个 **中断信号**，通知传输完成。
- CPU 收到中断后，执行相应的中断服务程序，继续正常运行。
- 总线控制权归还给 CPU
