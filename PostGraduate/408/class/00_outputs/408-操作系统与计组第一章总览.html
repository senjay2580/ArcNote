这两个第一章不要分开背。它们共同回答一个问题：一个程序为什么能从磁盘上的文件，变成 CPU 正在执行的一串机器指令，并且还能安全地访问内存、文件和设备？

操作系统第一章考点

王道操作系统第一章的核心不是让你记定义，而是让你理解：操作系统为什么必须存在，以及它如何在硬件之上形成一个“安全、有序、可共享”的运行环境。

操作系统概念、功能与接口

  • 定义：操作系统是控制和管理整个计算机系统硬件与软件资源，合理组织调度计算机工作和资源分配，并向用户和其他软件提供接口与环境的系统软件。
  • 作为资源管理者：处理机管理、存储器管理、文件管理、设备管理。配图中 QQ/视频聊天例子说明：程序要运行，先要被装入内存，再由处理机处理，还可能申请摄像头等设备。
  • 向上层提供服务：图形用户界面、命令接口、程序接口。命令接口给用户用，程序接口给程序员通过代码调用，系统调用是程序接口的核心。
  • 命令接口：联机命令接口是交互式，一条命令一条命令执行；脱机命令接口是批处理，用户一次提交任务，系统批量执行。
  • 系统调用：应用程序请求 OS 内核服务的方式，常见请求包括设备、文件、进程、进程通信、内存管理。
  • CPU 与外存：CPU 能直接访问的是内存；外存上的文件必须经过操作系统的文件系统和 I/O 管理间接访问。

易错点

库函数不等于系统调用。库函数可以只是普通用户态代码，也可以在底层封装系统调用；系统调用一定会请求内核服务，涉及资源分配或全局状态。

操作系统四大特征

特征含义必考区分例子
并发多个事件在同一时间间隔内发生宏观同时、微观交替；并行是同一时刻真正同时单核 CPU 上多个程序轮流运行
共享系统资源可供多个并发进程共同使用互斥共享 vs 同时共享摄像头通常互斥；磁盘文件可交替共享
虚拟把一个物理实体变成多个逻辑实体，或把多个物理实体抽象成一个逻辑实体时分复用、空分复用一台 CPU 虚拟成多个处理机；内存抽象成连续地址空间
异步进程走走停停，以不可预知速度推进由资源有限、调度、中断、I/O 等造成同一程序多次执行所需时间可能不同
要点

并发和共享互为存在条件：没有并发，就谈不上多个进程同时需要资源；没有共享，就无法支撑多个进程并发推进。虚拟和异步是在并发共享基础上进一步表现出来的系统特征。

发展与分类

阶段/类型核心思想优点缺点/考点
手工操作阶段用户人工装入程序、操作机器系统简单人机速度矛盾严重，CPU 等待人和慢速 I/O
单道批处理引入脱机 I/O 和监督程序，一次运行一道作业减少人工干预内存中只有一道程序，CPU 等待 I/O 时仍浪费
多道批处理内存同时放多道程序，交替运行提高 CPU 与资源利用率，操作系统正式诞生缺少交互性，用户不能及时干预作业
分时系统以时间片轮流为多个用户服务交互性强，响应较及时不能优先处理紧急任务
实时系统优先响应紧急任务及时性、可靠性强硬实时要求严格时限；软实时允许偶尔延迟
网络/分布式/个人机网络资源共享、分布协同、个人使用扩展应用场景分布式强调多机协同完成同一任务

运行机制：内核、特权级、中断

  • 内核程序 vs 应用程序：内核程序是 OS 最核心部分，运行在内核态；应用程序运行在用户态。
  • 特权指令：会影响系统全局状态或他人资源的指令，只允许内核态执行，例如 I/O 指令、关中断、修改页表、修改状态寄存器等。
  • 非特权指令：普通运算、访管前的用户态计算等，应用程序可直接执行。
  • 用户态与内核态：CPU 通常通过 PSW 或状态寄存器中的标志位区分当前特权级。
  • 用户态到内核态：由中断、异常、陷入等事件触发，硬件自动完成状态转换并跳转到内核入口。
  • 内核态到用户态：OS 处理完事件后，恢复现场并修改状态标志，返回用户程序继续执行。
中断类别别名来源同步性例子处理重点
外中断中断、异步中断CPU 外部设备或时钟与当前指令无关，异步时钟中断、I/O 完成、键盘、网卡通常在一条指令执行结束后检测
内中断异常、同步中断CPU 执行当前指令时产生与当前指令有关，同步除零、非法指令、缺页、系统调用执行到相关指令或异常条件时发生
陷入 trap访管/陷入指令用户程序主动请求内核同步系统调用用户态执行陷入指令后进入内核态
故障 fault可恢复异常执行中发现问题同步缺页故障修复后可回到原指令继续执行
终止 abort不可恢复异常严重错误同步硬件故障、严重非法状态通常终止进程或系统处理

虚拟机、系统调用与体系结构

主题必须掌握易错/补充
第一类 VMM虚拟机管理程序直接运行在硬件上性能更好，典型如 VMware ESXi、Xen、裸机 Hyper-V
第二类 VMM虚拟机管理程序运行在宿主操作系统之上依赖 Host OS，典型如 VirtualBox、VMware Workstation
系统调用过程传参、执行陷入指令、进入内核态、内核服务、返回用户态陷入指令在用户态执行，系统调用处理程序在内核态执行
系统调用分类设备管理、文件管理、进程控制、进程通信、内存管理涉及资源分配和共享资源统一管理时通常要系统调用
大内核主要功能模块都放内核态性能高，但结构复杂、维护难
微内核只把最基本机制放内核，策略尽量放用户态结构清晰、可靠性好，但用户态/内核态切换多，性能可能下降；更适合分布式思想
分层/模块化/外核体系结构组织方式分层强调层次调用，模块化强调可加载与扩展，外核强调把资源保护与抽象交给上层库

记忆法

只要某条指令或操作会“影响别人、改变全局状态、直接碰硬件资源”，就应该想到内核态、特权指令或系统调用。

计组第一章考点

计组第一章的任务是建立硬件视角：计算机由哪些部件组成，指令如何被硬件自动执行，软件如何变成机器能识别的指令，以及如何评价性能。

硬件组成与冯诺依曼结构

  • 冯诺依曼机特点：五大部件；指令和数据同等地位存于存储器；指令和数据用二进制表示；指令由操作码和地址码组成；以运算器为中心。
  • 现代计算机结构：以存储器为中心，CPU 包括运算器和控制器；主机包括 CPU 和主存储器；I/O 设备包括输入设备和输出设备。
  • 软件/硬件逻辑等价：同一功能既可用软件实现，也可用硬件实现；硬件通常快但成本高，软件灵活但速度可能慢。
  • 按字节编址：一个地址编号对应 1 个字节。地址空间大小看地址个数，存储容量看一共能存多少数据。
  • 字长区别：机器字长看 CPU 一次能处理多少位；存储字长看主存一个存储单元多少位；指令字长看 ISA 设计，不一定等于机器字长。

易错点

MDR 表示一次从内存读出或写入的数据位数，通常与存储字长相关；它不是“一个地址能存多少数据”的同义词。MAR 位数决定可寻址的存储单元数量。

计算机工作过程

配图从一段 C 代码开始，展示高级语言经过编译、汇编、链接形成可执行文件，装入主存后，CPU 根据 PC 指向的地址一条条取出机器指令并执行。

  • 取指阶段：PC -> MAR，主存读出指令到 MDR，再送入 IR，PC 通常自增。
  • 分析阶段：OP(IR) -> CU，CU 分析操作码，判断是什么指令。
  • 执行阶段：地址码可能送 MAR 取操作数，数据经 MDR 到寄存器或 ALU，执行后结果写回寄存器或主存。
  • 硬件自动性：取指、译码、执行由硬件控制器和时钟电路驱动，不需要 OS 指挥每一条指令。
  • OS 的作用点：OS 通过修改 PC、寄存器上下文、页表和内存映射，决定下一轮 CPU 自动取到哪个进程的指令。
要点

CPU 本身不理解“这是微信还是游戏”，它只按 PC 取指执行。所谓进程切换，本质是 OS 改变寄存器和内存上下文，让下一次取指落到另一个程序的代码上。

概念定义/作用常考区分

性能指标与公式

指标公式/含义考点提醒
CPU 时钟周期一个时钟脉冲的时间单位常用秒、微秒、纳秒；主频越高，周期越短

IPS/MIPS每秒执行多少条指令IPS = 主频 / 平均 CPI；MIPS 是百万条指令每秒
FLOPS每秒浮点运算次数K/M/G/T/P 等性能单位按 10^3 递进
数据通路带宽数据总线一次能并行传输的信息位数影响部件间一次传送的数据量
吞吐量单位时间完成任务数量常与响应时间形成权衡
响应时间从发出请求到获得结果所需时间包括 CPU 时间、等待时间、I/O 时间、系统开销等

单位规则

存储容量相关通常按 1024，即 2 的幂；速度、频率、带宽、运算次数等性能指标通常按 1000，即国际单位制。

操作系统逐小节考点

小节必背考点王道常见问法你要会说的话
OS 概念和功能系统软件；管理软硬件资源；提供接口OS 的目标/功能是什么？OS 向下管理 CPU、内存、文件、设备，向上给用户和程序提供接口。
OS 接口命令接口、程序接口、GUI；系统调用属于程序接口系统调用和命令接口区别？命令接口给用户用，系统调用给程序通过代码请求内核服务。
四大特征并发、共享、虚拟、异步并发和并行区别？共享方式有哪些？并发是时间间隔内交替推进，并行是同一时刻真正同时执行。
发展分类手工、单道批、多道批、分时、实时、网络、分布式哪种系统解决交互问题？哪种强调及时性？多道批提高资源利用率，分时强调交互，实时强调及时和可靠。
运行机制内核/应用；特权/非特权；用户态/内核态哪些指令必须在核心态执行？影响全局状态、硬件资源或其他进程的操作必须交给内核。
中断异常外中断、内中断、trap、fault、abort；中断向量表系统调用属于哪类中断？中断响应时机？系统调用是 trap，属于内中断；外中断通常在指令结束后检测。
系统调用参数传递、陷入指令、内核服务、返回用户态系统调用与库函数区别？库函数可能封装系统调用；系统调用一定会进入内核请求 OS 服务。
虚拟机第一类 VMM、第二类 VMM、特权级两类 VMM 区别？第一类直接跑硬件上，第二类跑宿主 OS 上。
体系结构大内核、微内核、分层、模块化、外核微内核优缺点？微内核结构清晰可靠，但状态切换多，性能可能下降。
OS 引导ROM/BIOS、POST、MBR、PBR、启动管理器、内核开机后 OS 如何被加载？CPU 先从 ROM 取指，逐级找到并加载操作系统内核。

计组逐小节考点

小节必背考点王道常见问法你要会说的话
硬件基本组成五大部件；现代计算机以存储器为中心主机包括哪些？CPU 包括哪些？CPU=运算器+控制器；主机=CPU+主存；I/O 设备在主机外。
冯诺依曼结构指令和数据同存储器；二进制；操作码+地址码冯诺依曼机特点？指令和数据形式上都是二进制，CPU 按阶段解释它们。
主存结构MAR、MDR、存储体、存储单元、存储字、存储字长MAR/MDR 位数分别决定什么？MAR 位数决定地址范围，MDR 位数决定一次交换的数据宽度。
运算器ACC、MQ、X、ALU乘除法中 ACC/MQ 作用？运算器负责数据加工，寄存器临时保存操作数和结果。
控制器PC、IR、CU；取指、分析、执行PC 和 IR 的区别？PC 保存下一条指令地址，IR 保存当前正在执行的指令。
工作过程PC->MAR->MDR->IR；OP(IR)->CU；Ad(IR)->MAR一条指令如何执行？先取指，再分析操作码，最后取操作数并执行。
软件分类系统软件、应用软件；DBMS 是系统软件数据库管理系统是不是系统软件？DBMS 是系统软件，数据库系统不是单纯一个软件。
语言翻译机器语言、汇编语言、高级语言；编译、汇编、解释哪种语言 CPU 直接执行？CPU 只能直接执行机器语言，汇编和高级语言都要翻译。
ISA指令、编码、寄存器、寻址方式ISA 为什么是软硬件接口？ISA 规定软件能使用哪些机器指令，也规定硬件必须实现什么。
性能指标主频、CPI、CPU 时间、IPS/MIPS、FLOPS、吞吐量、响应时间主频高就一定快吗？不一定，性能由指令数、CPI、主频共同决定。

高频易混点

易混点正确区分快速判断
并发 vs 并行并发是同一时间间隔内都在推进；并行是同一时刻同时执行单核可并发，多核才可能真正并行。
同步/异步 vs OS 异步性同步/异步常说事件等待关系；OS 异步性说进程推进速度不可预知看到“走走停停、不可预知”就是 OS 异步性。
系统调用 vs 库函数系统调用进入内核；库函数是语言/运行库函数printf 可能最终调 write，但 printf 本身不是系统调用。
中断 vs 异常外中断来自 CPU 外部；异常来自 CPU 执行指令内部键盘/时钟/I/O 完成是外中断；除零/缺页/trap 是内中断。
trap vs fault vs aborttrap 主动请求；fault 可修复；abort 严重不可恢复系统调用 trap；缺页 fault；严重硬件故障 abort。
用户态 vs 内核态用户态不能执行特权指令；内核态可执行全部指令会影响别人或全局状态的事，进内核。
MAR vs MDRMAR 存地址；MDR 存数据地址范围看 MAR，数据宽度看 MDR。
PC vs IRPC 指下一条；IR 存当前条取指后 PC 常自增，IR 用于当前指令译码。
机器字长 vs 存储字长 vs 指令字长机器字长看 CPU；存储字长看主存；指令字长看 ISA三者不必相等。
主频高 vs 性能高主频只是因素之一CPU 时间 = 指令数 x CPI / 主频。
