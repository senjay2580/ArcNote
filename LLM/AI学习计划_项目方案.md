> 调研来源：Boss直聘岗位分析、研究生论坛、清华/中科院招生信息、GitHub项目趋势、AI面试题库

| 方向 | 热度 | 我的匹配度 | 原因 |
|------|------|-----------|------|
| **AI Agent 工程化** | 最热 | 极高 | 有 Agent 后端经验，直接对口 |
| **Edge AI / 端侧部署** | 高 | 高 | 工程落地能力强需求，后端可迁移 |

### 关键市场结论

1. **"会用大模型"比"会训大模型"更有市场** — Agent 开发、RAG 应用是最大就业增量
2. **软硬结合人才溢价高** — 端侧 AI 和具身智能方向，同时懂算法和硬件的人才最抢手
3. **工程能力 > 论文数量** — 企业更看重能落地、能部署、能优化的实战能力
4. **跨学科背景加分** — 具身智能需要机器人+AI，端侧需要嵌入式+AI，复合背景竞争力更强

### 技术架构

```
┌─────────────┐    MQTT     ┌──────────────┐    HTTP/WS    ┌─────────────┐
│  ESP32 节点  │ ◄────────► │  树莓派5中控  │ ◄───────────► │  Web 管理端  │
│ 传感器+执行器│            │ LLM + Agent  │              │ (后端强项)   │
└─────────────┘            └──────────────┘              └─────────────┘
  温湿度/光照                Ollama 本地推理                 FastAPI 后端
  继电器/LED                 RAG 知识库                    设备管理 Dashboard
  MicroPython               意图识别+设备控制              历史数据可视化
```

### 技术栈分解

| 层次 | 技术 | 经验复用度 |
|------|------|-----------|

| **硬件端** | ESP32 + MicroPython + DHT22 | 新学，门槛最低的入口 |
| **前端** | React 简易 Dashboard | 可选，锦上添花 |

### AI 核心概念清单（面试必问 Top 12）

| 概念 | 一句话理解 |
|------|-----------|
| **Transformer** | 基于自注意力机制的序列模型，抛弃了RNN的循环结构 |
| **Self-Attention** | 计算序列中每个token对其他token的关注度，Q/K/V矩阵运算 |
| **多头注意力** | 并行多组注意力捕捉不同子空间特征 |
| **位置编码** | 给Transformer注入序列顺序信息 |
| **RAG** | 检索增强生成：先检索相关文档，再送入LLM生成答案 |
| **Prompt Engineering** | 通过设计输入提示词控制模型输出质量 |
| **LoRA/QLoRA** | 低秩适配微调，冻结大部分参数只训练小矩阵 |
| **Token化** | 将文本切分为模型可处理的最小单元（BPE/SentencePiece） |
| **向量数据库** | 存储Embedding，支持语义相似度检索（Milvus/FAISS） |
| **AI Agent** | 具备规划、记忆、工具调用能力的自主智能体 |
| **模型量化** | 将FP32权重压缩为INT8/INT4，减少内存和推理延迟 |
| **ZeRO优化器** | 分布式训练中切分优化器状态，降低显存需求 |
