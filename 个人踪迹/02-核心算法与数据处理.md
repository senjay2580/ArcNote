/**

public void processLocationEvent(LocationProcessEvent event) {

### 1.2 访问检测算法

**核心思想**: 基于空间聚类和时间连续性检测停留点

```
算法步骤:
1. 空间聚类: 使用数据库层面的空间聚类查询
2. 时间分割: 按时间间隔分割同一空间聚类
3. 持续时间过滤: 过滤掉停留时间过短的点
4. 创建访问记录
```

**关键参数**:
- `minimumStayTimeInSeconds`: 最小停留时间
- `maxMergeTimeBetweenSameStayPoints`: 同一停留点的最大合并时间间隔
- `minDistanceBetweenVisits`: 访问之间的最小距离

```java
private List<StayPoint> detectStayPointsFromTrajectory(
        Map<Integer, List<RawLocationPoint>> points,
        DetectionParameter.VisitDetection visitDetectionParameters) {
    
    // 按时间分割空间聚类
    for (List<RawLocationPoint> clusteredByLocation : points.values()) {
        clusteredByLocation.sort(Comparator.comparing(RawLocationPoint::getTimestamp));
        
        for (int i = 1; i < clusteredByLocation.size(); i++) {
            RawLocationPoint next = clusteredByLocation.get(i);
            // 如果时间间隔超过阈值，创建新的时间聚类
            if (Duration.between(currentTime, next.getTimestamp()).getSeconds() 
                    < visitDetectionParameters.getMaxMergeTimeBetweenSameStayPoints()) {
                currentTimedCluster.add(next);
            } else {
                currentTimedCluster = new ArrayList<>();
                clusters.add(currentTimedCluster);
            }
        }
    }
    
    // 按最小停留时间过滤
    return clusters.stream()
        .filter(c -> Duration.between(c.getFirst().getTimestamp(), 
                                      c.getLast().getTimestamp()).toSeconds() 
                     > visitDetectionParameters.getMinimumStayTimeInSeconds())
        .map(this::createStayPoint)
        .collect(Collectors.toList());
}
```

### 1.3 访问合并算法

**目的**: 将相邻的访问记录合并为处理后的访问 (ProcessedVisit)

```java
private List<ProcessedVisit> mergeVisitsChronologically(
        User user, String previewId, String traceId, List<Visit> visits,
        DetectionParameter.VisitMerging mergeConfiguration) {
    
    for (int i = 1; i < visits.size(); i++) {
        Visit nextVisit = visits.get(i);
        SignificantPlace nextPlace = findOrCreateSignificantPlace(...);
        
        boolean samePlace = nextPlace.getId().equals(currentPlace.getId());
        boolean withinTimeThreshold = Duration.between(currentEndTime, nextVisit.getStartTime())
            .getSeconds() <= mergeConfiguration.getMaxMergeTimeBetweenSameVisits();
        
        boolean shouldMerge = samePlace && withinTimeThreshold;
        
        // 如果是同一地点但超过时间阈值，检查实际移动距离
        if (samePlace && !withinTimeThreshold) {
            List<RawLocationPoint> pointsBetween = ...;
            double travelledDistance = GeoUtils.calculateTripDistance(pointsBetween);
            shouldMerge = travelledDistance <= mergeConfiguration.getMinDistanceBetweenVisits();
        }
        
        if (shouldMerge) {
            // 扩展当前访问的结束时间
            currentEndTime = nextVisit.getEndTime();
        } else {
            // 创建新的处理后访问
            result.add(createProcessedVisit(currentPlace, currentStartTime, currentEndTime));
            currentPlace = nextPlace;
            currentStartTime = nextVisit.getStartTime();
        }
    }
}
```

## 二、交通方式检测

### 2.1 基于速度的分段分类

```java
public TransportMode segmentAndClassifyTrip(List<RawLocationPoint> points, 
                                            List<TransportModeConfig> configs) {
    List<TripSegment> segments = new ArrayList<>();
    List<Double> speeds = calculateSpeeds(points);
    
    // 检测速度变化点进行分段
    for (int i = 1; i < points.size(); i++) {
        double prevSpeed = speeds.get(i - 2);
        double currSpeed = speeds.get(i - 1);
        
        // 速度变化超过50%时创建新分段
        if (prevSpeed > 0 && Math.abs(currSpeed - prevSpeed) / prevSpeed > 0.5) {
            TransportMode mode = classifySegment(currentSegmentPoints, configs);
            segments.add(new TripSegment(currentSegmentPoints, mode));
            currentSegmentPoints.clear();
        }
        currentSegmentPoints.add(points.get(i));
    }
    
    // 按持续时间加权选择主要交通方式
    Map<TransportMode, Double> modeDurationMinutes = new HashMap<>();
    for (TripSegment segment : segments) {
        double durationMinutes = Duration.between(
            segment.points().getFirst().getTimestamp(),
            segment.points().getLast().getTimestamp()
        ).toMillis() / (1000.0 * 60.0);
        modeDurationMinutes.merge(segment.dominantMode(), durationMinutes, Double::sum);
    }
    
    return modeDurationMinutes.entrySet().stream()
        .max(Map.Entry.comparingByValue())
        .map(Map.Entry::getKey)
        .orElse(TransportMode.UNKNOWN);
}
```

## 三、地理空间计算

### 3.1 距离计算 (Haversine 公式)

```java
public static double distanceInMeters(RawLocationPoint p1, RawLocationPoint p2) {
    // 使用 Haversine 公式计算两点间的球面距离
    double lat1 = Math.toRadians(p1.getLatitude());
    double lat2 = Math.toRadians(p2.getLatitude());
    double dLat = lat2 - lat1;
    double dLon = Math.toRadians(p2.getLongitude() - p1.getLongitude());
    
    double a = Math.sin(dLat/2) * Math.sin(dLat/2) +
               Math.cos(lat1) * Math.cos(lat2) *
               Math.sin(dLon/2) * Math.sin(dLon/2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    
    return EARTH_RADIUS_METERS * c;
}
```

### 3.2 行程距离计算

```java
public static double calculateTripDistance(List<RawLocationPoint> points) {
    double totalDistance = 0;
    for (int i = 1; i < points.size(); i++) {
        totalDistance += distanceInMeters(points.get(i-1), points.get(i));
    }
    return totalDistance;
}
```

### 3.3 米转度数 (纬度相关)

```java
public static double metersToDegreesAtPosition(double meters, double latitude) {
    // 在不同纬度，1度对应的米数不同
    double metersPerDegree = 111320 * Math.cos(Math.toRadians(latitude));
    return meters / metersPerDegree;
}
```

## 五、空间聚类查询

### 5.1 数据库层面的空间聚类

利用 PostGIS 的空间函数进行高效聚类:

```sql
-- 使用 ST_ClusterDBSCAN 进行空间聚类
SELECT 
    id,
    timestamp,
    geom,
    ST_ClusterDBSCAN(geom, eps := :distance, minpoints := :minPoints) 
        OVER (ORDER BY timestamp) as cluster_id
FROM raw_location_points
WHERE user_id = :userId
  AND timestamp BETWEEN :start AND :end
  AND NOT ignored
ORDER BY timestamp;
```

## 六、学习要点

### 6.1 算法设计亮点

1. **分层处理**: 原始点 → 访问 → 处理后访问 → 行程
2. **参数可配置**: 所有检测参数都可以按用户配置
3. **预览模式**: 支持参数调整预览，不影响生产数据
4. **乐观锁**: 使用版本号防止并发更新冲突

### 6.2 性能优化技巧

1. **批量处理**: 使用批量插入和更新
2. **空间索引**: PostGIS GIST 索引加速空间查询
3. **缓存策略**: Redis 缓存热点数据
4. **异步处理**: RabbitMQ 解耦数据摄入和处理

### 6.3 可扩展性设计

1. **用户级并行**: 不同用户的数据可以并行处理
2. **队列隔离**: 不同类型的任务使用不同队列
3. **死信处理**: 失败消息进入死信队列，便于排查
