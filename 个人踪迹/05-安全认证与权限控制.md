# 安全认证与权限控制

## 一、认证方式概览

Reitti 支持多种认证方式，满足不同场景需求：

```
┌─────────────────────────────────────────────────────────────────┐
│                        认证方式                                  │
├─────────────────────────────────────────────────────────────────┤
│  1. 表单登录 (Form Login)        - Web UI 用户登录              │
│  2. API Token 认证               - 移动应用/API 调用            │
│  3. OIDC/OAuth2 认证             - 第三方身份提供商             │
│  4. Magic Link 认证              - 分享链接访问                 │
│  5. Remember Me                  - 持久化登录状态               │
└─────────────────────────────────────────────────────────────────┘
```

## 二、Spring Security 配置

### 2.1 安全过滤器链

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                // 公开访问
                .requestMatchers("/login", "/access", "/error").permitAll()
                .requestMatchers("/css/**", "/js/**", "/images/**").permitAll()
                .requestMatchers("/actuator/health").permitAll()
                
                // 角色权限控制
                .requestMatchers("/settings/logging/**").hasRole(Role.ADMIN.name())
                .requestMatchers("/settings/**").hasAnyRole(Role.ADMIN.name(), Role.USER.name())
                
                // Magic Link 特殊权限
                .requestMatchers("/api/v1/photos/**").hasAnyRole(
                    Role.ADMIN.name(), Role.USER.name(),
                    "MAGIC_LINK_FULL_ACCESS",
                    "MAGIC_LINK_ONLY_LIVE_WITH_PHOTOS",
                    "MAGIC_LINK_MEMORY_VIEW_ONLY"
                )
                
                // 其他请求需要认证
                .anyRequest().authenticated()
            )
            // 过滤器顺序
            .addFilterBefore(magicLinkSessionValidationFilter, AuthorizationFilter.class)
            .addFilterBefore(magicLinkAuthenticationFilter, MagicLinkSessionValidationFilter.class)
            .addFilterBefore(bearerTokenAuthFilter, MagicLinkAuthenticationFilter.class)
            .addFilterBefore(urlTokenAuthenticationFilter, TokenAuthenticationFilter.class)
            .addFilterBefore(setupFilter, MagicLinkSessionValidationFilter.class)
            
            // 禁用 CSRF (API 使用)
            .csrf(AbstractHttpConfigurer::disable)
            
            // 表单登录
            .formLogin(form -> form
                .loginPage("/login")
                .successHandler(customAuthenticationSuccessHandler)
            )
            
            // Remember Me
            .rememberMe(rememberMe -> rememberMe
                .key("uniqueAndSecretKey")
                .tokenValiditySeconds(2592000) // 30 天
                .rememberMeParameter("remember-me")
            )
            
            // 登出
            .logout(logout -> logout
                .deleteCookies("JSESSIONID", "remember-me")
                .permitAll()
            );

        return http.build();
    }
}
```

### 2.2 过滤器执行顺序

```
请求 → SetupFilter → UrlTokenAuthFilter → TokenAuthFilter 
     → MagicLinkAuthFilter → MagicLinkSessionValidationFilter 
     → AuthorizationFilter → Controller
```

## 三、API Token 认证

### 3.1 Token 认证过滤器

```java
public class TokenAuthenticationFilter extends BaseTokenAuthenticationFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) {
        // 从 Authorization 头获取 Token
        String authHeader = request.getHeader("Authorization");
        
        if (authHeader != null && authHeader.startsWith("Bearer ")) {
            String token = authHeader.substring(7);
            
            // 验证 Token
            Optional<ApiToken> apiToken = apiTokenService.validateToken(token);
            
            if (apiToken.isPresent()) {
                // 创建认证对象
                User user = apiToken.get().getUser();
                UsernamePasswordAuthenticationToken authentication = 
                    new UsernamePasswordAuthenticationToken(
                        user, null, 
                        List.of(new SimpleGrantedAuthority("ROLE_API_ACCESS"))
                    );
                SecurityContextHolder.getContext().setAuthentication(authentication);
                
                // 更新最后使用时间
                apiTokenService.updateLastUsed(apiToken.get());
            }
        }
        
        filterChain.doFilter(request, response);
    }
}
```

### 3.2 URL Token 认证 (便捷方式)

```java
public class UrlTokenAuthenticationFilter extends BaseTokenAuthenticationFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) {
        // 从 URL 参数获取 Token
        String token = request.getParameter("token");
        
        if (token != null) {
            // 验证并认证
            authenticateWithToken(token);
        }
        
        filterChain.doFilter(request, response);
    }
}
```

### 3.3 Token 服务

```java
@Service
public class ApiTokenService {
    
    public ApiToken createToken(User user, String name) {
        String token = generateSecureToken();
        ApiToken apiToken = new ApiToken(name, hashToken(token), user);
        apiTokenJdbcService.save(apiToken);
        return apiToken.withPlainToken(token); // 只返回一次明文
    }
    
    public Optional<ApiToken> validateToken(String plainToken) {
        String hashedToken = hashToken(plainToken);
        return apiTokenJdbcService.findByHashedToken(hashedToken);
    }
    
    private String generateSecureToken() {
        byte[] bytes = new byte[32];
        new SecureRandom().nextBytes(bytes);
        return Base64.getUrlEncoder().withoutPadding().encodeToString(bytes);
    }
    
    private String hashToken(String token) {
        return DigestUtils.sha256Hex(token);
    }
}
```

### 4.2 自定义 OIDC 用户服务

```java
@Service
public class CustomOidcUserService extends OidcUserService {
    
    @Override
    public OidcUser loadUser(OidcUserRequest userRequest) {
        OidcUser oidcUser = super.loadUser(userRequest);
        
        // 提取用户信息
        String externalId = oidcUser.getIssuer() + ":" + oidcUser.getSubject();
        String username = oidcUser.getPreferredUsername();
        String displayName = oidcUser.getFullName();
        String pictureUrl = oidcUser.getPicture();
        
        // 查找或创建用户
        User user = userService.findByExternalId(externalId)
            .or(() -> userService.findByUsername(username))
            .orElseGet(() -> {
                if (signUpEnabled) {
                    return userService.createUser(username, displayName, externalId);
                }
                throw new OAuth2AuthenticationException("User not found");
            });
        
        // 更新用户信息
        user = user.withDisplayName(displayName)
                   .withExternalId(externalId);
        
        // 下载头像
        if (pictureUrl != null) {
            avatarService.downloadAndSaveAvatar(user, pictureUrl);
        }
        
        return new CustomOidcUser(oidcUser, user);
    }
}
```

## 五、Magic Link 认证

### 5.1 Magic Link 概念

Magic Link 是一种无密码分享机制，允许用户通过特殊链接分享数据：

```
https://reitti.example.com/access?token=abc123xyz
```

### 5.2 Magic Link 类型

```java
public enum MagicLinkType {
    FULL_ACCESS,           // 完全访问权限
    ONLY_LIVE_MODE,        // 仅实时模式
    ONLY_LIVE_WITH_PHOTOS, // 实时模式+照片
    MEMORY_VIEW_ONLY,      // 仅查看记忆
    MEMORY_EDIT_ACCESS     // 编辑记忆权限
}
```

### 5.3 Magic Link 认证过滤器

```java
public class MagicLinkAuthenticationFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) {
        String token = request.getParameter("token");
        
        if (token != null && request.getRequestURI().equals("/access")) {
            Optional<MagicLink> magicLink = magicLinkService.validateToken(token);
            
            if (magicLink.isPresent()) {
                MagicLink link = magicLink.get();
                
                // 创建 Magic Link 认证
                MagicLinkAuthenticationToken authentication = 
                    new MagicLinkAuthenticationToken(
                        link.getUser(),
                        link.getType(),
                        link.getResources()
                    );
                
                // 存储到 Session
                request.getSession().setAttribute("MAGIC_LINK_AUTH", authentication);
                
                // 重定向到目标页面
                response.sendRedirect(link.getTargetUrl());
                return;
            }
        }
        
        filterChain.doFilter(request, response);
    }
}
```

### 5.4 Magic Link Session 验证

```java
public class MagicLinkSessionValidationFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) {
        MagicLinkAuthenticationToken auth = 
            (MagicLinkAuthenticationToken) request.getSession()
                .getAttribute("MAGIC_LINK_AUTH");
        
        if (auth != null) {
            // 验证 Magic Link 是否仍然有效
            if (magicLinkService.isValid(auth.getToken())) {
                SecurityContextHolder.getContext().setAuthentication(auth);
            } else {
                // Magic Link 已失效，清除 Session
                request.getSession().removeAttribute("MAGIC_LINK_AUTH");
            }
        }
        
        filterChain.doFilter(request, response);
    }
}
```

## 七、初始化设置流程

### 7.1 Setup 过滤器

```java
public class SetupFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) {
        // 检查是否需要初始化设置
        if (!setupService.isSetupComplete()) {
            String uri = request.getRequestURI();
            
            // 允许访问设置页面和静态资源
            if (!uri.startsWith("/setup") && !isStaticResource(uri)) {
                response.sendRedirect("/setup");
                return;
            }
        }
        
        filterChain.doFilter(request, response);
    }
}
```

### 7.2 初始化控制器

```java
@Controller
@RequestMapping("/setup")
public class SetupController {
    
    @GetMapping
    public String showSetup() {
        if (setupService.isSetupComplete()) {
            return "redirect:/";
        }
        return "setup";
    }
    
    @PostMapping
    public String completeSetup(@RequestParam String password) {
        // 设置管理员密码
        userService.setAdminPassword(password);
        setupService.markSetupComplete();
        return "redirect:/login";
    }
}
```

## 八、学习要点

### 8.1 多认证方式设计

1. **过滤器链**: 多个认证过滤器按顺序执行
2. **优先级**: Token 认证优先于 Session 认证
3. **降级处理**: 认证失败时继续尝试下一种方式

### 8.2 Token 安全最佳实践

1. **哈希存储**: Token 只存储哈希值
2. **一次显示**: 明文 Token 只在创建时显示一次
3. **安全生成**: 使用 SecureRandom 生成 Token
4. **使用追踪**: 记录 Token 最后使用时间

### 8.3 OIDC 集成要点

1. **用户匹配**: 先按 external_id，再按 username
2. **自动注册**: 可配置是否允许自动创建用户
3. **信息同步**: 每次登录更新用户信息
4. **头像下载**: 自动下载 OIDC 提供的头像

### 8.4 Magic Link 设计

1. **细粒度权限**: 不同类型的 Magic Link 有不同权限
2. **资源限制**: 可以限制访问特定资源
3. **有效期**: Magic Link 可以设置过期时间
4. **Session 绑定**: Magic Link 认证绑定到 Session
