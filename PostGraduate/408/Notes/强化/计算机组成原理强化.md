# 计算机组成原理 · 强化分章树（块③·计组）

> 对标 **2027 王道 408 强化**。主干见 [[408强化总纲]]，五大链路在总纲②。
> 每个强化点统一四项范式：**溯源（硬件/前置缺陷推出它）→ 落地（怎么考）→ 短板（边界坑）→ 衍生（催生谁、省略隐患）**，并挂 **[联动OS]**。

# 第1章 计算机系统概述（A-，性价比高，选择常驻）

## 1.1 冯诺依曼结构 → 现代以存储器为中心

- **溯源**：早期冯诺依曼机**以运算器为中心**，所有数据进出都过 ALU → CPU 成为数据搬运瓶颈。**现代改为以存储器为中心**，解放 CPU。
- **落地**：五大部件、存储程序、指令=操作码+地址码、指令与数据同存且同为二进制（靠"何时取"区分）。
- **短板**：以运算器为中心时 IO 与运算抢 ALU；指令/数据同存导致冯诺依曼瓶颈（CPU 与存储器之间的带宽墙）。
- **衍生**：存储器中心结构、Cache（链路 3 的存储墙）、哈佛结构（指令/数据 Cache 分离，第 5 章流水线）。
- **[联动OS]**：CPU 只认 PC 取主存指令、不执行磁盘文件 → 逼出 OS 的"装入"（[[408强化总纲]] 链 1）。

## 1.2 性能指标与公式（必拿计算分）

- **溯源**：单看主频会误判性能 → 引入指令数×CPI 综合度量。
- **落地（必背变形）**：
  - `CPU 执行时间 = 指令条数 × CPI × 时钟周期 = 指令条数 × CPI / 主频`
  - `MIPS = 指令条数 / (执行时间 × 10⁶) = 主频 / (CPI × 10⁶)`
  - `FLOPS`：每秒浮点运算次数，K/M/G/T/P 按 **10³** 递进。
- **短板/陷阱**：① **主频高 ≠ 性能高**（指令数、CPI 共同决定）；② **单位**——存储容量按 **1024**，速度/频率/带宽/运算次数按 **1000**；③ MIPS 不能跨指令集比较。
- **衍生**：基准程序（benchmark）评测、吞吐量 vs 响应时间权衡。
- **[联动OS]**：吞吐量/响应时间正是 OS 调度算法的优化目标；CPI 受 Cache 命中率、流水线阻塞、缺页影响（跨链路 3）。

## 1.3 易混速查

| 易混点 | 正确区分 |
|---|---|
| 机器字长 / 存储字长 / 指令字长 | 看 CPU / 看主存单元 / 看 ISA，三者不必相等 |
| MAR / MDR | MAR 位数决**可寻址单元数**；MDR 位数决**一次读写数据宽度**(≈存储字长) |
| PC / IR | PC 存**下一条**地址(取指后自增)；IR 存**当前**指令 |
| 系统软件 / 应用软件 | DBMS、编译器、OS 是系统软件 |

# 第2章 数据的表示与运算（A，选择高频，细节多）

## 2.1 补码 —— 为何统治定点运算

- **溯源**：原码做加减要**先判符号、绝对值大小再决定加还是减**，电路复杂；且 +0/−0 两个零 → **补码把减法统一成加法**（`A−B = A + (−B)补`），符号位参与运算，零唯一。
- **落地**：补码加减电路（一个加法器 + 求补）、标志位 **OF/CF/SF/ZF**；变补（连同符号位取反加一）。
- **短板**：补码只统一了加减；**乘除仍需专门算法**（手算繁、易错）。
- **衍生**：Booth 算法（补码一位乘）、原码/补码移位乘除、阵列乘法器。
- **[联动OS]**：纯硬件层，OS 不介入；但整数溢出/类型转换是系统编程隐患源。

## 2.2 溢出判断（选择必考陷阱）

- **落地**：① 用**两个符号位**（变形补码）——结果两符号位不同则溢出；② **最高位进位 ⊕ 次高位进位 = 1** 则溢出；③ **有符号看 OF、无符号看 CF**。
- **陷阱**：正+正得负、负+负得正才溢出；正+负永不溢出。

## 2.3 浮点数与 IEEE 754

- **溯源**：定点数表示范围/精度受限 → 浮点用"阶码定范围、尾数定精度"。
- **落地**：拆三段 **S（符号）/ E（阶码，移码）/ M（尾数，隐藏位 1）**；规格化（尾数最高位为 1）；舍入四种（就近偶/向 0/向 +∞/向 −∞）；特殊值：阶全 0=0/非规格化、阶全 1 尾 0=∞、阶全 1 尾≠0=NaN。
- **短板**：精度有限 → **大数吃小数**、不结合律；上溢→∞（中断），下溢→0。
- **衍生**：双精度、非规格化数（渐进下溢）、舍入控制。
- **陷阱**：阶码用**移码**（=补码符号位取反）便于比较大小；规格化方向；真值=`(-1)^S ×1.M× 2^(E-127)`。

## 2.4 加法器与 ALU（B+，了解推导）

- **溯源**：串行进位（行波）加法器**进位逐位传播、延迟大** → **并行进位（先行进位 CLA）**用进位生成/传递函数一次算出所有进位。
- **落地**：ALU 由加法器 + 逻辑 + 控制构成；标志位输出。
- **[联动OS]**：无直接联动，属计组本科。

# 第3章 存储系统（S，选择极高、大题高发，链路 3 核心）

## 3.1 存储层次 —— 一切的起点

- **溯源**：速度、容量、价格**三者不可兼得** → 用金字塔分层：寄存器→Cache→主存→辅存。两个核心局部矛盾各自解决：**Cache–主存层**解决"主存太慢"，**主存–辅存层（虚存）**解决"主存太小"。
- **落地**：Cache 对程序员**透明、硬件**自动管理；虚存对程序员**透明、OS + 硬件(MMU)**共管。
- **短板**：层次越多、一致性与命中管理越复杂。
- **衍生**：Cache（3.5）、虚拟存储（见 [[操作系统强化]] 第3章）、TLB。
- **[联动OS]**：**页式/虚拟存储"见操作系统"**——计组负责硬件查表（MMU/TLB/Cache），OS 负责建页表、处理缺页。这是链路 3 的分工线。

## 3.2 DRAM / SRAM 与刷新

- **溯源**：SRAM 用触发器（双稳态）→快但**面积大、贵**，做 Cache；DRAM 用**电容存电荷**→密度高便宜，做主存。
- **落地**：DRAM **破坏性读出 → 读后必须立即重写（刷新）**；电容漏电 → 每 **64ms** 全刷一遍；刷新按**行**进行；刷新方式：集中刷新（有死区）/分散刷新（无死区但慢）/异步刷新（折中）。行列地址复用（降低地址引脚）。
- **短板**：DRAM 需周期性刷新占用存储周期；速度仍远慢于 CPU。
- **衍生**：SDRAM/DDR（与时钟同步、突发传输）、多模块存储器（3.4）、Cache（3.5）。
- **陷阱**：刷新对 CPU 透明、由存储器控制器做；刷新以**行**为单位、与片选无关；SRAM 不需刷新。
- **[联动OS]**：刷新/访存周期是 OS 算不到的底层，但访存延迟决定有效访问时间 EAT。

## 3.3 主存与 CPU 连接：位扩展 / 字扩展（选择常考）

- **溯源**：单片存储芯片**字长不够**（数据线带不满）→ **位扩展**；**容量不够**（地址空间用不满）→ **字扩展**；两者皆缺 → **字位同时扩展**。
- **落地**：算芯片数 = 总容量 / 单片容量；片选用译码器（线选法/译码片选法）；给地址范围反推芯片连接。
- **短板**：地址译码不当 → 地址重叠/寻址能力浪费。
- **陷阱**：位扩展**共享片选与地址、并接数据**；字扩展**共享地址数据、片选区分**；译码器使能端保证地址稳定后再选通（用户笔记重点）。

## 3.4 多模块存储器

- **溯源**：单体存储**一次访问要等一个存取周期**，CPU 干等 → 用**并行**思想：单体多字（一次取多个字）/ 多体并行（多个独立模块，**低位交叉编址**轮流启动）。
- **落地**：交叉存储器**流水线式**启动各体，连续访问吞吐≈ m 倍；连续取 n 个字耗时 `T + (n-1)·r`。
- **短板**：单体多字灵活性差（读入冗余）；多体对非连续访问收益小。
- **[联动OS]**：提升主存带宽，缓解链路 3 访存压力。

## 3.5 Cache（重中之重 S+）

### 3.5.1 Cache 为何诞生

- **溯源**：CPU 与主存**速度矛盾**（CPU 空等主存）+ 程序的**局部性原理**（时间局部性：刚访问的还会访问；空间局部性：邻近的将被访问）→ 把热点块放进高速小容量 Cache。
- **落地**：命中率 h、缺失率、`平均访问时间 = h·t_c + (1-h)·t_m`（或含层次）。
- **短板**：容量小 → 必有替换、冲突、抖动、一致性问题（见下及 [[408强化总纲]] 故障 3）。

### 3.5.2 三种映射（必考地址划分）

| 映射 | 溯源/动机 | 地址划分 | 短板 → 衍生 |
|---|---|---|---|
| 直接映射 | 实现最简、比较最快 | **Tag + 行号 + 块内偏移** | 冲突最严重（多块争一行）→ 组相联 |
| 全相联 | 任意块任放、冲突最少 | **Tag + 块内偏移** | 需并行比较所有 Tag、硬件贵慢 → 组相联 |
| 组相联 | 折中 | **Tag + 组号 + 块内偏移** | 路数选择权衡（2/4/8 路） |

- **落地**：给地址位数/Cache 容量/块大小/相联度，算 **Tag/Index/Offset 位数、行数、Cache 总位数（含 Tag+有效位+脏位）**。
- **陷阱**：**Cache 行（块）大小 = 主存块大小**；块内偏移位数由块大小定；组数 = 行数 / 路数。

### 3.5.3 替换算法

- 随机（无视局部性、命中低）/ FIFO（可能 Belady）/ **LRU（栈算法，无 Belady，硬件计数器/堆栈实现）** / LFU。
- **陷阱**：LRU 用计数器，命中清零其余加一；只在**组相联/全相联**才需替换算法（直接映射无选择）。

### 3.5.4 写策略（成对记）

- **写命中**：**写直达 write-through**（同时写 Cache+主存，慢、一致性好，配**写缓冲**）/ **写回 write-back**（只写 Cache、置**脏位**，换出时才写主存，快、一致性差）。
- **写不命中**：**写分配 write-allocate**（调块入 Cache，配写回）/ **非写分配**（直接写主存不调块，配写直达）。
- **衍生**：多级 Cache（L1/L2/L3）、**多级/多核 Cache 一致性问题 → MESI 协议**（用户笔记已点"各级 Cache 存在一致性问题"）。
- **[联动OS]**：DMA 直写主存绕过 Cache → **Cache 与内存不一致**（链路 4 DMA 一致性）；写策略影响多进程共享内存可见性。

### 3.5.5 故障：Cache 抖动

见 [[408强化总纲]] 故障 3：直接映射下步长=组数倍数的数组访问反复冲突 → 组相联/循环分块。

## 3.6 磁盘存储器（与 OS 链路 5 接驳）

- **溯源**：需要大容量非易失存储 → 磁记录、串行逐位读写、攒块后并行送内存。
- **落地**：地址 = **柱面(磁道)号 + 磁头(盘面)号 + 扇区号 (CHS)**；**存取时间 = 寻道时间 + 旋转延迟(平均转半圈=½×60/转速) + 传输时间**；扇区是最小读写单位。
- **短板**：机械寻道/旋转慢（ms 级）→ 是整个链路 5 的性能瓶颈。
- **衍生**：磁盘调度（FCFS/SSTF/SCAN/C-SCAN，见 [[操作系统强化]] 第5章）、RAID（0 条带提速无冗余 / 1 镜像 / 5 分布校验）、SSD（无机械、按页读写、有写放大与磨损均衡）。
- **[联动OS]**：磁盘物理参数 → OS 磁盘调度算法的计算源（磁头移动距离题）。

# 第4章 指令系统（S，选择极高、常作 CPU 大题前置）

## 4.1 指令格式与扩展操作码

- **溯源**：定长操作码在**指令字短、指令多**时浪费编码空间 → **扩展操作码（变长操作码）**：地址多的指令用短操作码、地址少的用长操作码，充分利用编码。
- **落地**：给定各类指令条数，分配操作码位数；判断指令字长。
- **短板**：变长操作码**译码电路复杂**；**短操作码不能是任何长操作码的前缀**（否则歧义）。
- **衍生**：RISC 定长指令（4.5）便于流水线译码。
- **陷阱**：n 位操作码最多 2ⁿ 条；扩展时每"借"一个长码要扣掉对应短码空间。

## 4.2 指令寻址 vs 数据寻址

- **指令寻址**：顺序（PC + "1"，"1"=指令字长字节数）/ 跳跃（转移指令给出）。**变长指令字靠操作码先判长度**再定 PC 增量。
- **数据寻址（7+ 种）**：

| 方式 | 有效地址 EA | 访存次数(取操作数) | 用途/特点 |
|---|---|---|---|
| 立即 | 操作数在指令中 | 0 | 常数，最快 |
| 直接 | EA = A | 1 | A 即地址 |
| 间接 | EA = (A) | ≥2 | 一次间址 2 次、多级更多 |
| 寄存器 | 操作数在 Rᵢ | 0 | 最快 |
| 寄存器间接 | EA = (Rᵢ) | 1 | 地址在寄存器 |
| 隐含 | 隐含在指令/约定寄存器 | — | 如除法用 EDX:EAX |
| 偏移 | EA = (基/变/PC) + A | 1 | 见下 |
| 堆栈 | SP 隐含 | 1 | 入栈出栈 |

## 4.3 偏移寻址三兄弟（高频辨析）

- **基址寻址**：`EA = (BR) + 形式地址`。**溯源**：多道程序每个程序在内存位置不定 → 需要"整段浮动"。**BR 由 OS 在程序运行前置为该程序起始地址（存在 PCB 中）**（用户笔记重点）。**面向操作系统、程序整段在内存中浮动**。
- **变址寻址**：`EA = (IX) + 形式地址`，IX 用户可改 → **面向用户、循环访问数组**。
- **相对寻址**：`EA = (PC) + 偏移`。**程序内部一段代码的浮动**，用于**转移指令、与位置无关代码**。
- **[联动OS]**：基址寄存器 BR ≡ **重定位寄存器**，正是链路 1 **动态重定位**的硬件实现；程序浮动 = OS 能把进程装到任意空闲内存。**省略动态重定位则虚拟内存整条塌。**
- **陷阱**：相对寻址目标 = **取指后已自增的 PC** + 偏移（注意 PC 当前值）。

## 4.4 机器级程序（A）

- 一条**汇编 = 一条机器**指令；一条高级语句 = 多条汇编。
- 选择/循环 → `cmp` 改标志 + `jxx` 条件转移（看 ZF/SF/OF/CF）。
- **函数调用**：`call` 压返回地址跳转、`ret` 弹出返回；**栈帧**用 EBP（基址）/ESP（栈顶）；**调用者保存 vs 被调用者保存**寄存器 = 恢复现场（与中断/进程切换"保存现场"同源思想）。
- **[联动OS]**：栈帧 ↔ 进程用户栈；call/ret 的压栈保存 ↔ 上下文保存（链路 2）。

## 4.5 CISC vs RISC

- **溯源**：CISC 指令复杂、变长、靠**微程序**实现 → 译码慢、难流水；**RISC** 精简定长、Load/Store 结构、寄存器多 → **天然适配流水线**（MIPS 五级流水线是教科书范本，ARM 继承）。
- **陷阱**：RISC 指令数多但 CPI 低且可流水；CISC 单条功能强但 CPI 高。
- **[联动OS]**：RISC 定长指令使"指令周期边界响应中断"更规整（链路 2/4）。

# 第5章 中央处理器（S+，计组第一优先级，大题高发）

## 5.1 CPU 结构与寄存器分类

- 运算器（ALU + 通用寄存器 + 暂存器 + ACC/PSW）+ 控制器（PC/IR/CU + MAR/MDR）。
- **用户可见**：通用寄存器、PSW、PC（部分）；**用户不可见（透明）**：MAR、MDR、IR、暂存寄存器。
- **数据为何必经寄存器**：内存↔ALU 太慢且总线一次只能一读或一写，故"内存→寄存器→ALU→寄存器→内存"（用户笔记重点）。**ALU 输出加暂存器**避免抖动 Glitch、支持流水线。
- **[联动OS]**：用户可见寄存器 + PC + PSW + 页表基址寄存器 = **进程上下文**（链路 2）。

## 5.2 指令周期与数据流

- **溯源**：一条指令要分阶段完成 → 指令周期 = **取指 → (间址) → 执行 → (中断)** 周期；机器周期含多个时钟周期；最小单位是**微操作**。
- **落地**：默画四种数据流——
  - 取指：`PC→MAR→M→MDR→IR`，`OP(IR)→CU`，PC+1。
  - 间址：`Ad(IR)→MAR→M→MDR→Ad(IR)`。
  - 执行：随指令而异。
  - 中断：`关中断→保存PC/PSW→向量→入口PC`。
- **陷阱**：**每条指令执行末尾 CPU 才检测中断**（用户笔记强调）；间址周期取的是有效地址不是操作数。

## 5.3 数据通路 + 控制器（硬布线 vs 微程序）

- **数据通路**：单总线（结构简单、冲突多、慢）/ 专用通路（快、贵）。**微操作 ⇔ 控制信号**。
- **硬布线控制器**：**溯源**——追求速度，用组合逻辑直接产生控制信号。**短板**——不灵活、指令多则**电路爆炸、修改困难** → **衍生微程序**。RISC 多用硬布线。
- **微程序控制器**：把每条机器指令翻成一段**微程序**存于**控制存储器 CM**。
  - 微指令编码：**直接编码**（快、字长长）/ **字段直接编码**（互斥微命令分组，省位）/ 字段间接编码（更省、更慢）。
  - 微地址形成、微指令格式（水平型/垂直型/混合型）。
  - **短板**：每条微指令要访问 CM → **比硬布线慢**。CISC 多用微程序。
- **陷阱**：水平型微指令并行性高、微程序短；垂直型相反。控制存储器用 **ROM**。

## 5.4 指令流水线（大题压舱石）

### 5.4.1 为何流水

- **溯源**：顺序执行时**各功能部件大量空闲**（取指时 ALU 闲） → 流水线**时空重叠**：把指令分 **IF/ID/EX/M/WB** 五段，像工厂流水线让各段同时干不同指令。
- **落地（必算）**：
  - 吞吐率 `TP = n / T_k`；理想 k 段流水连续执行 n 条：`T = (k + n − 1)·Δt`。
  - 加速比 `S = 顺序时间 / 流水时间`，**上限 = 段数 k**。
  - 效率 `E = 时空有效面积 / 总时空面积`。

### 5.4.2 三大冒险（短板 → 衍生，必考）

| 冒险 | 根源（短板） | 衍生（解决） |
|---|---|---|
| **结构冒险** | 同一周期争用同一部件（如取指与访存都用存储器） | **哈佛结构**（指令/数据 Cache 分离）、资源加倍、停顿 |
| **数据冒险** | 后条指令需要前条还没写回的结果（**RAW 最常见**，还有 WAR/WAW） | **数据旁路/转发 forwarding**（从流水寄存器直送 ALU）、插入气泡 stall、编译器指令调度 |
| **控制冒险** | 分支指令要到 EX(或 ID) 才知跳哪 → 已预取的指令作废 | **分支预测**（静态/动态 BHT/BTB）、延迟槽、提前生成转移地址 |

- **陷阱**：**数据旁路不能完全消除 load-use 冒险**（load 的数据在 M 段才得到，仍需停 1 拍）；加速比受冒险/装入时间拖累达不到 k。
- **衍生（高级）**：超标量（多发射）、超流水（段更细）、VLIW（编译期打包）。
- **[联动OS]**：进程切换/中断会**冲刷流水线**，是上下文切换开销的硬件来源（链路 2）。

## 5.5 中断机制（计组实现，OS 链路总开关）

- **溯源**：外设异步、CPU 不能轮询 → 中断让 CPU 在**指令末尾**响应。
- **落地（中断响应=中断隐指令，硬件自动）**：① **关中断**（防现场保存被打断，保证原子）；② **保存断点 PC、PSW** 入栈；③ 识别中断源、取**中断向量**；④ 入口送 PC。**通用寄存器需中断服务程序用软件保存**（用户笔记重点）。结尾 **开中断 → IRET（同时恢复 PSW）**。
- **短板**：单重中断不能嵌套急件 → **多重中断 + 中断屏蔽字 + 优先级**；屏蔽字设置不当 → 嵌套异常（[[408强化总纲]] 故障 6）。
- **陷阱**：中断隐指令**不保存通用寄存器**（软件做）；自己不能屏蔽比自己优先级低 …（屏蔽字决定可嵌套谁）；不可屏蔽中断（掉电）绕过屏蔽。
- **[联动OS]**：这是 OS 进内核的唯一硬件通道——系统调用(trap)、缺页(fault)、时钟中断全靠它（链路 1/2/3/4）。

## 5.6 多处理器（B，了解）

- Flynn：SISD/SIMD/MISD/MIMD；硬件多线程：细粒度/粗粒度/**同时多线程 SMT**（超线程）。
- **[联动OS]**：多核是 OS 多处理器调度、真并行的硬件前提。

# 第6章 总线（B+，中频，主攻计算）

- **溯源**：部件两两专线 → 连线爆炸 → **总线**共享传输，代价是**分时、需仲裁**。
- **落地（必算）**：`总线带宽 = 总线频率 × 总线宽度(位) / 8`（B/s）；一次传输看"地址周期 + 等待 + 数据周期"。
- **要点**：同步定时（统一时钟、快、设备需同速）/ 异步定时（全互锁/半互锁/不互锁握手、灵活慢）；仲裁（链式查询/计数器/独立请求、集中 vs 分布）；突发(burst)传送。
- **🔴 PCI vs PCIe**（用户笔记标必考）：PCI 并行共享总线；PCIe **串行、点对点、全双工、按 lane（×1/×4/×16）**，带宽 = 每 lane 速率 × lane 数 × 编码效率。
- **[联动OS]**：DMA 与 CPU **争总线**（停止 CPU/周期窃取/交替访问，链路 4 + 故障 5）。

# 第7章 输入输出系统（A，选择高频，链路 4 硬件侧）

## 7.1 IO 控制方式四级演化（= 链路 4 主轴）

| 方式 | 前级短板 → 它 | CPU 介入粒度 | 残留短板 → 衍生 |
|---|---|---|---|
| **程序查询(轮询)** | 最原始 | CPU **忙等**全程 | 利用率极低 → 中断驱动 |
| **程序中断** | 消除忙等 | **每字/字节**一次中断 | 中断开销大、高速盘扛不住 → DMA |
| **DMA** | 成块直传 | 仅**传送前(初始化)+后(完成中断)** | 争总线、Cache 一致性、溢出 → 通道 |
| **通道** | IO 处理机 | CPU 几乎不管 | 硬件复杂（大型机） |

## 7.2 程序中断方式（细节见 5.5）

- **中断响应**即**中断隐指令**：硬件自动关中断、保存断点 PC/PSW、取向量、转 ISR。
- **向量中断**：中断源送向量号 → 查中断向量表（固定低地址，每项存 ISR 入口）→ 地址 = 向量号 × 项长。
- **多重中断条件**：开中断 + 现场已保护 + 来者优先级更高；**中断屏蔽字**控制嵌套；**EOI** 不发则同级后续中断被堵。
- **陷阱**：中断响应只能在**指令执行结束**时刻；DMA 请求**优先级高于中断**（数据时间敏感会丢）。

## 7.3 DMA 方式（必考）

- **溯源**：中断驱动每字节打扰 CPU，高速设备受不了 → DMA 控制器直接在**设备↔主存**间成块搬运。
- **落地（DMA 传送过程）**：**预处理**（CPU 初始化 DMA：源/目的地址、字数）→ **数据传送**（DMA 接管总线，**不需 CPU**）→ **后处理**（传送完发中断，CPU 善后）。
- **三种总线占用方式**：**停止 CPU 访存**（简单、CPU 全停）/ **周期窃取（cycle stealing）**（DMA 抢一个存取周期、最常用）/ **交替访问**（分时不冲突、效率最高、需硬件配合）。
- **短板**：与 CPU **争总线**；**DMA 绕过 Cache 致一致性问题**；高速设备 **DMA 溢出**丢数据（[[408强化总纲]] 故障 5）。
- **陷阱**：DMA 以**块**为单位、中断方式以**字/字节**为单位；DMA **数据传送阶段不占 CPU**，但**预处理/后处理占 CPU**；周期窃取是"偷一个周期"非占整个传输期。
- **[联动OS]**：DMA 把数据搬到**内核缓冲区** → 再拷到用户缓冲区（链路 4）；机器字长决定 DMA 缓冲寄存器一次搬运宽度（用户笔记）。

## 7.4 IO 接口与端口编址

- **统一编址（内存映射 IO）**：端口占用内存地址、用访存指令、无专用 IO 指令 → 占内存空间。
- **独立编址**：专用 IO 指令、单独 IO 地址空间 → 指令多。
- **[联动OS]**：设备独立性、IO 软件分层见 [[操作系统强化]] 第5章。
