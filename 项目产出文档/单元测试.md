## 单元测试【能力边界】

**单元测试能做什么**：

- ✅ 测试单个方法的业务逻辑
- ✅ 验证异常情况的处理
- ✅ 确保边界条件的正确性
- ✅ 模拟外部依赖的行为
- ✅ 提供代码使用示例

**单元测试不能做什么**：

- ❌ 测试数据库连接（那是集成测试）
- ❌ 测试网络请求（需要Mock）
- ❌ 测试UI界面交互
- ❌ 测试性能问题
- ❌ 替代人工验收测试

## 基本语法（知道能干什么）

### Stubbing（定义行为）

| 场景       | 语法                        | 示例                                                         |
| ---------- | --------------------------- | ------------------------------------------------------------ |
| 返回值     | `when().thenReturn()`       | `when(dao.select(1)).thenReturn(user)`                       |
| 返回多个值 | `thenReturn().thenReturn()` | `when(dao.count()).thenReturn(1).thenReturn(2)`              |
| 抛异常     | `when().thenThrow()`        | `when(dao.insert(null)).thenThrow(new IllegalArgumentException())` |
| void方法   | `doNothing().when()`        | `doNothing().when(service).sendEmail()`                      |
| void抛异常 | `doThrow().when()`          | `doThrow(new RuntimeException()).when(service).method()`     |

### Verification（验证调用）

| 场景     | 语法                     | 示例                                   |
| -------- | ------------------------ | -------------------------------------- |
| 验证调用 | `verify()`               | `verify(dao).insert(user)`             |
| 调用1次  | `verify(times(1))`       | `verify(dao, times(1)).insert(any())`  |
| 从未调用 | `verify(never())`        | `verify(dao, never()).delete(any())`   |
| 至少1次  | `verify(atLeastOnce())`  | `verify(dao, atLeastOnce()).select(1)` |
| 无交互   | `verifyNoInteractions()` | `verifyNoInteractions(dao)`            |

### 参数匹配器

| 匹配器        | 说明       | 示例                                                   |
| ------------- | ---------- | ------------------------------------------------------ |
| `any()`       | 任意对象   | `when(dao.insert(any())).thenReturn(1)`                |
| `anyString()` | 任意字符串 | `when(service.find(anyString())).thenReturn(result)`   |
| `anyInt()`    | 任意整数   | `when(dao.select(anyInt())).thenReturn(data)`          |
| `eq()`        | 精确匹配   | `verify(dao).insert(eq(user))`                         |
| `isNull()`    | null值     | `when(service.method(isNull())).thenThrow(ex)`         |
| `isNotNull()` | 非null     | `when(service.method(isNotNull())).thenReturn(result)` |

`@Spy` 的常见用途

1. **测试回调或监听器**：当你需要验证某个事件发生时，一个回调方法是否被正确触发。
2. **部分模拟复杂对象**：对于一个拥有很多方法的复杂对象，你只想关注其中一两个方法的调用情况，而其他方法的行为保持不变。
3. **在真实环境中替换部分逻辑**：在集成测试中，你可能希望大部分逻辑都是真实的，只将某个外部依赖（如数据库查询、API 调用）的返回值固定下来，以确保测试的稳定性和可重复性。
4. **性能测试中的监控**：可以用 Spy 来统计某个方法的调用次数或执行时间，而不改变其功能。

### 断言语法大全

#### 1. 基本断言

```java
@Test
void basicAssertions() {
    // 相等断言
    assertEquals(4, 2 + 2);
    assertEquals(4, 2 + 2, "2 + 2 应该等于 4");
    
    // 不相等断言
    assertNotEquals(3, 2 + 2);
    
    // 布尔断言
    assertTrue(2 < 3);
    assertFalse(2 > 3);
    
    // null断言
    assertNull(null);
    assertNotNull("not null");
    
    // 相同对象断言
    String str = "test";
    assertSame(str, str);
    assertNotSame(new String("test"), new String("test"));
}
```

#### 2. 异常断言

```java
@Test
void exceptionTesting() {
    // 断言抛出特定异常
    Exception exception = assertThrows(
        ArithmeticException.class, 
        () -> {
            int result = 10 / 0;
        }
    );
    
    // 验证异常消息
    assertEquals("/ by zero", exception.getMessage());
    
    // 断言不抛出异常
    assertDoesNotThrow(() -> {
        int result = 10 / 2;
    });
}
```

#### 3. 超时断言

```java
@Test
void timeoutTesting() {
    // 断言在指定时间内完成
    assertTimeout(Duration.ofMillis(100), () -> {
        Thread.sleep(50);
        return "result";
    });
    
    // 超时立即失败
    assertTimeoutPreemptively(Duration.ofSeconds(1), () -> {
        // 长时间运行的代码
    });
}
```

#### 4. 组合断言

```java
@Test
void groupedAssertions() {
    Person person = new Person("John", "Doe");
    
    // 所有断言都会执行
    assertAll("person properties",
        () -> assertEquals("John", person.getFirstName()),
        () -> assertEquals("Doe", person.getLastName()),
        () -> assertNotNull(person.getEmail())
    );
}
```

### 参数化测试

> 核心目的都是**用多组不同的输入数据来重复执行同一个测试逻辑**，以提高测试覆盖率和代码复用性。

#### 1. 值源参数化

```java
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.*;

class ParameterizedTests {
    
    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3, 4, 5})
    void testWithInts(int value) {
        assertTrue(value > 0 && value < 6);
    }
    
    @ParameterizedTest
    @ValueSource(strings = {"", "  ", "\t", "\n"})
    void testBlankStrings(String value) {
        assertTrue(value.isBlank());
    }
    
    @ParameterizedTest
    @NullSource
    @EmptySource
    @ValueSource(strings = {" ", "\t"})
    void testNullEmptyAndBlank(String text) {
        assertTrue(text == null || text.trim().isEmpty());
    }
}
```

#### 2. 方法源参数化

```java
@ParameterizedTest
@MethodSource("provideTestData")
void testWithMethodSource(String input, int expected) {
    assertEquals(expected, input.length());
}

static Stream<Arguments> provideTestData() {
    return Stream.of(
        Arguments.of("hello", 5),
        Arguments.of("world", 5),
        Arguments.of("JUnit 5", 7)
    );
}
```

#### 3. CSV源参数化

```java
@ParameterizedTest
@CsvSource({
    "1, 1, 2",
    "2, 3, 5",
    "10, 20, 30"
})
void testAddition(int a, int b, int expected) {
    Calculator calc = new Calculator();
    assertEquals(expected, calc.add(a, b));
}

@ParameterizedTest
@CsvFileSource(resources = "/test-data.csv", numLinesToSkip = 1)
void testWithCsvFile(String name, int age) {
    assertNotNull(name);
    assertTrue(age > 0);
}
```

        assertEquals(4, 2 + 2);
    }

    @DisplayName("除法测试")

            });

### 动态测试

```java
import org.junit.jupiter.api.DynamicTest;
import org.junit.jupiter.api.TestFactory;

class DynamicTestsDemo {
    
    @TestFactory
    Stream<DynamicTest> dynamicTests() {
        return Stream.of(1, 2, 3, 4, 5)
            .map(n -> DynamicTest.dynamicTest(
                "测试 " + n + " 是否小于 10",
                () -> assertTrue(n < 10)
            ));
    }
    
    @TestFactory
    Collection<DynamicTest> dynamicTestsFromCollection() {
        return Arrays.asList(
            DynamicTest.dynamicTest("第一个动态测试", 
                () -> assertTrue(true)),
            DynamicTest.dynamicTest("第二个动态测试", 
                () -> assertEquals(4, 2 + 2))
        );
    }
}
```



## 编写开始

编写测试前问自己：

- [ ] 我要测试的类是什么？（应该用 @InjectMocks）
- [ ] 这个类依赖哪些对象？（应该用 @Mock）
- [ ] 需要测试哪些场景？（正常、异常、边界）
- [ ] 如何准备测试数据？（常量、辅助方法）
- [ ] 需要 stub 哪些方法？（只 stub 会被调用的）
- [ ] 需要 verify 什么？（核心方法调用、参数）
- [ ] 测试名称是否清晰？（使用 @DisplayName）



## 测试覆盖率指标

	测试覆盖率指标主要用于衡量软件测试的完整性，可从以下维度分类总结：

### 一、代码覆盖率（Code Coverage）
- **语句覆盖率（Statement Coverage）**：被执行的代码语句数占总语句数的比例，反映代码行的执行程度。
- **分支覆盖率（Branch Coverage）**：被执行的代码分支数占总分支数的比例，如`if-else`、`switch`的每个分支是否都被触发。
- **函数覆盖率（Function Coverage）**：被执行的函数/方法数占总函数数的比例。
- **条件覆盖率（Condition Coverage）**：复杂条件中每个子条件的真假情况被覆盖的比例，如`a && b || c`中`a`、`b`、`c`的真假组合。
- **路径覆盖率（Path Coverage）**：被执行的代码路径数占总路径数的比例，覆盖所有可能的执行流程，是最严格的代码覆盖率指标。

### 二、功能覆盖率（Functional Coverage）
衡量测试用例对需求文档中功能点的覆盖比例，即验证的功能数占总功能数的比例。

### 三、需求覆盖率（Requirements Coverage）
涵盖对所有需求（包括功能和非功能需求，如性能、安全性、兼容性等）的覆盖程度，即针对需求设计的测试用例数占总需求数的比例。

### 四、接口覆盖率（Interface Coverage）
针对系统接口（如API、函数调用接口），覆盖的接口输入参数组合、返回值情况等的比例。
