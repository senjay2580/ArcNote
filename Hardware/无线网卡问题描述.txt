[TL;DR]
Lenovo Legion Y7000P IRH8 (2023, i7-13620H, Win11 22H2).
The WiFi (and occasionally Bluetooth) icon disappears from the
Quick Settings panel. The wireless adapter (likely Intel Wi-Fi 6E
AX211) intermittently shows a yellow exclamation mark in Device
Manager, and sometimes vanishes from the device list entirely.

The fault is reliably triggered by pressing the BUILT-IN
keyboard with moderate force - ANY key, not a specific one.
Using an EXTERNAL USB keyboard NEVER triggers the issue.
A 30-40s power drain temporarily restores it. All software-level
fixes have failed. Temperature is normal; the fault occurs even
on a cold machine. Strongly suspect a physical issue (loose M.2
WiFi card or detached u.FL antenna connector).

[Hardware]
- Model: Lenovo Legion Y7000P IRH8 (MT 82YA)
- CPU: Intel Core i7-13620H (13th Gen, 10C/16T)
- RAM: 16 GB
- BIOS: LENOVO M2CN26WW, dated 2023-07-06, UEFI mode
- OS: Windows 11 Home 22H2, Build 22621.1413
- WiFi card (as seen in Device Manager): Intel(R) Wi-Fi 6E AX211 160MHz
- Age: ~2-3 years, never opened, never cleaned, original thermal paste

[Symptoms]
1. The WiFi toggle is missing from the Win11 Quick Settings flyout.
   The Bluetooth toggle also disappears occasionally, but far
   less frequently than WiFi.
2. When the fault occurs:
   - No "WLAN" option under Settings > Network & Internet.
   - The wireless adapter shows a yellow exclamation mark in
     Device Manager, OR disappears completely from the
     "Network adapters" list - as if the card were physically
     unplugged from the PCIe/CNVi bus.

[Trigger - the key clue]
- Pressing the BUILT-IN laptop keyboard with slightly more force
  than normal typing reliably triggers the fault.
- The triggering key is NOT specific - ANY key can cause it.
- Light typing rarely triggers it.
- Using an EXTERNAL USB keyboard: has NEVER triggered the fault.
- Temperature is normal. It happens on a completely cold machine
  as well, so thermal throttling / overheating is ruled out.
- No clear correlation with uptime or workload.

[What I have already tried]
1. Unplug ALL peripherals + hold power button 30-40s for full
   drain + reboot
   -> Works TEMPORARILY. The fault always returns.
2. Windows "Network reset"                         -> no effect
3. Disable / Enable adapter in Device Manager      -> no effect
4. Uninstall driver + reboot                       -> no effect;
   sometimes the adapter does not re-appear at all after reboot
5. Toggle Airplane mode                            -> no effect
6. Fn + wireless hotkey                            -> no effect

[Not yet tried]
- Updating / rolling back Intel AX211 driver
- Checking BIOS for WLAN module presence during a fault
- Booting a Linux Live USB or WinPE to rule out Windows/driver
- Full OS reinstall

[My hypothesis]
The combination of:
  (a) built-in keyboard press (ANY key) triggers the fault,
  (b) external USB keyboard NEVER triggers it,
  (c) full power drain temporarily restores it,
  (d) the device can completely disappear from PCIe enumeration,
  (e) temperature is irrelevant,
points strongly to a PHYSICAL contact problem, not software.

  Most likely: the M.2 2230 WiFi card is loose in its slot,
               or one/both u.FL antenna connectors have come
               off. Pressing ANY key on the built-in keyboard
               flexes the palm-rest / chassis just enough to
               disturb the card or antenna. External USB
               typing puts zero force on the chassis, so it
               never triggers - this fits perfectly.
  Possible:    cold solder joint / hairline crack on the CNVi
               interface or the motherboard in that area.
  Unlikely:    driver bug, ESD, dust/overheating.

[What I would like advice on]
1. Does my hypothesis sound correct given this exact symptom set?
2. Any additional diagnostic steps I should run BEFORE opening
   the laptop (specific event log sources, driver versions,
   BIOS options for WLAN, etc.)?
3. Intel AX211 is a CNVi-only module, correct? What replacement
   cards (if any) are compatible with the Y7000P IRH8 motherboard?
4. Has anyone seen the exact same "press built-in keyboard ->
   WiFi card vanishes from PCIe" pattern on a Legion Y7000P /
   Legion 5 / Legion Slim of this generation?
5. If it turns out to be a motherboard-side CNVi issue rather
   than the card itself, what are the realistic repair options?
