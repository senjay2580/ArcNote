# 学习总结与项目亮点

## 一、项目亮点总结

### 1.1 架构设计亮点

| 亮点 | 描述 | 学习价值 |
|------|------|---------|
| 事件驱动架构 | RabbitMQ 解耦数据摄入和处理 | 高可扩展性设计 |
| 空间数据处理 | PostGIS + JTS 地理空间计算 | GIS 应用开发 |
| 多认证方式 | Token/OIDC/MagicLink | 企业级安全设计 |
| HTMX 前端 | 服务端渲染 + 动态交互 | 现代 Web 开发新范式 |
| 预览模式 | 参数调整预览不影响生产 | 用户体验设计 |

### 1.2 代码质量亮点

| 亮点 | 描述 | 学习价值 |
|------|------|---------|
| 不可变对象 | 领域模型使用 wither 模式 | 函数式编程思想 |
| 乐观锁 | 版本号并发控制 | 高并发设计 |
| 纯 JDBC | 避免 ORM 开销 | 性能优化 |
| 70+ 迁移脚本 | 渐进式数据库演进 | 数据库版本管理 |
| 自定义测试注解 | @IntegrationTest 封装配置 | 测试工程化 |

### 1.3 运维友好亮点

| 亮点 | 描述 | 学习价值 |
|------|------|---------|
| Docker 一键部署 | docker-compose 完整编排 | 容器化部署 |
| 健康检查 | 所有服务配置健康检查 | 生产级部署 |
| 动态 UID/GID | 容器用户映射 | 安全最佳实践 |
| 多语言支持 | 8种语言国际化 | 全球化应用 |
| 自托管地理编码 | Photon 隐私保护 | 隐私优先设计 |

## 四、设计模式应用

### 4.1 创建型模式

| 模式 | 应用场景 |
|------|---------|
| Builder | 复杂对象构建 (QueueBuilder) |
| Factory | 服务创建 (GeocodeServiceFactory) |

### 4.2 结构型模式

| 模式 | 应用场景 |
|------|---------|
| Adapter | 数据格式转换 (OwnTracks → LocationPoint) |
| Decorator | 过滤器链 (Security Filters) |

### 4.3 行为型模式

| 模式 | 应用场景 |
|------|---------|
| Observer | 事件发布订阅 (RabbitMQ) |
| Strategy | 地理编码服务选择 |
| Template Method | 基础认证过滤器 |
| Chain of Responsibility | 安全过滤器链 |

## 五、最佳实践总结

### 5.1 代码规范

```java
// 1. 不可变对象 + Wither 模式
public RawLocationPoint markProcessed() {
    return new RawLocationPoint(this.id, ..., true, ...);
}

// 2. 乐观锁并发控制
UPDATE table SET version = version + 1 WHERE id = ? AND version = ?

// 3. 批量操作 + 去重
INSERT INTO ... ON CONFLICT DO NOTHING

// 4. 自定义测试注解
@IntegrationTest
class MyServiceTest { }
```

### 5.2 安全规范

```
1. Token 哈希存储，明文只显示一次
2. 非 root 容器运行
3. CSRF 保护 (表单) / 禁用 (API)
4. 细粒度权限控制 (Magic Link)
5. OIDC 外部身份集成
```

### 5.3 运维规范

```
1. 所有服务配置健康检查
2. 依赖服务就绪后再启动
3. 死信队列处理失败消息
4. 定期备份数据库和数据卷
5. 合理的日志级别配置
```
