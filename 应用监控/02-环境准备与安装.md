## 🔧 前置环境检查

### 1. 操作系统要求
- ✅ Windows 10/11 (WSL2) / Linux / macOS
- ✅ 64位系统
- ✅ 至少 4GB 可用内存
- ✅ 至少 20GB 可用磁盘空间

### 2. 必备软件
```bash
# 检查 Docker 版本(建议 20.10+)
docker --version
# 输出示例: Docker version 24.0.7

# 检查 Docker Compose 版本(建议 2.0+)
docker-compose --version
# 输出示例: Docker Compose version v2.23.0
```

### 3. 端口占用检查
```bash
# Windows PowerShell 检查端口
netstat -ano | findstr "9090"   # Prometheus
netstat -ano | findstr "3000"   # Grafana
netstat -ano | findstr "9100"   # Node Exporter
netstat -ano | findstr "9104"   # MySQL Exporter
```

**如果端口被占用**:
- 方案1: 停止占用端口的程序

## 🚀 快速部署(一键启动)

### 方式一:独立 docker-compose 文件(推荐)

#### 步骤1:创建配置文件

**1.1 创建 Prometheus 配置**
```bash
# 在 ArcHat 根目录执行
mkdir -p monitoring/prometheus/rules
```

创建文件: `monitoring/prometheus/prometheus.yml`
```yaml
# Prometheus 全局配置
global:
  scrape_interval: 15s          # 每15秒采集一次数据
  evaluation_interval: 15s      # 每15秒评估一次告警规则
  external_labels:
    cluster: 'archat-dev'       # 集群标签(多环境区分)
    env: 'development'

# 告警规则文件
rule_files:
  - '/etc/prometheus/rules/*.yml'

# 采集目标配置
scrape_configs:
  # ==================== 1. Prometheus 自身监控 ====================
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']
        labels:
          instance: 'prometheus-server'

  # ==================== 2. 系统资源监控 ====================
  - job_name: 'node-exporter'
    static_configs:
      - targets: ['node-exporter:9100']
        labels:
          instance: 'archat-server'
          service: 'node'

  # ==================== 3. MySQL 监控 ====================
  - job_name: 'mysql'
    static_configs:
      - targets: ['mysql-exporter:9104']
        labels:
          instance: 'mysql-main'
          service: 'mysql'

  # ==================== 4. Redis 监控 ====================
  - job_name: 'redis'
    static_configs:
      - targets: ['redis-exporter:9121']
        labels:
          instance: 'redis-main'
          service: 'redis'

  # ==================== 5. RabbitMQ 监控 (可选) ====================
  - job_name: 'rabbitmq'
    static_configs:
      - targets: ['rabbitmq-exporter:9419']
        labels:
          instance: 'rabbitmq-main'
          service: 'rabbitmq'

  # ==================== 6. Spring Boot 应用监控 ====================
  - job_name: 'spring-boot'
    metrics_path: '/actuator/prometheus'  # Spring Boot Actuator 指标路径
    static_configs:
      - targets: ['archat-backend:8080']  # 替换为你的后端容器名和端口
        labels:
          application: 'archat-backend'
          service: 'java-app'
```

**1.2 创建基础告警规则**
创建文件: `monitoring/prometheus/rules/node_alerts.yml`
```yaml
groups:
  - name: node_alerts
    interval: 30s
    rules:
      # CPU 使用率告警
      - alert: HighCPUUsage
        expr: 100 - (avg by(instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.instance }} CPU使用率过高"
          description: "{{ $labels.instance }} CPU使用率超过80% (当前值: {{ $value }}%)"

      # 内存使用率告警
      - alert: HighMemoryUsage
        expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100 > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.instance }} 内存使用率过高"
          description: "{{ $labels.instance }} 内存使用率超过85% (当前值: {{ $value }}%)"

      # 磁盘使用率告警
      - alert: HighDiskUsage
        expr: (1 - (node_filesystem_avail_bytes{fstype!="tmpfs"} / node_filesystem_size_bytes{fstype!="tmpfs"})) * 100 > 85
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "{{ $labels.instance }} 磁盘使用率过高"
          description: "{{ $labels.instance }} 磁盘 {{ $labels.mountpoint }} 使用率超过85% (当前值: {{ $value }}%)"
```

**1.3 创建 Grafana 数据源配置**
```bash
mkdir -p monitoring/grafana/provisioning/datasources
mkdir -p monitoring/grafana/provisioning/dashboards
```

创建文件: `monitoring/grafana/provisioning/datasources/prometheus.yml`
```yaml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: true
    jsonData:
      timeInterval: 15s
```

创建文件: `monitoring/grafana/provisioning/dashboards/dashboards.yml`
```yaml
apiVersion: 1

providers:
  - name: 'default'
    orgId: 1
    folder: ''
    type: file
    disableDeletion: false
    updateIntervalSeconds: 10
    allowUiUpdates: true
    options:
      path: /etc/grafana/provisioning/dashboards
```

#### 步骤2:创建监控 Docker Compose 文件

创建文件: `docker-compose.monitoring.yml`
```yaml
version: '3.8'

services:
  # ==================== Prometheus (监控核心) ====================
  prometheus:
    image: prom/prometheus:v2.48.0
    container_name: archat_prometheus
    restart: unless-stopped
    command:
      - '--config.file=/etc/prometheus/prometheus.yml'
      - '--storage.tsdb.path=/prometheus'
      - '--storage.tsdb.retention.time=15d'  # 数据保留15天
      - '--web.console.libraries=/usr/share/prometheus/console_libraries'
      - '--web.console.templates=/usr/share/prometheus/consoles'
      - '--web.enable-lifecycle'  # 允许热加载配置
    ports:
      - "9090:9090"
    volumes:
      - ./monitoring/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - ./monitoring/prometheus/rules:/etc/prometheus/rules
      - prometheus-data:/prometheus
    networks:
      - monitoring

  # ==================== Grafana (可视化) ====================
  grafana:
    image: grafana/grafana:10.2.2
    container_name: archat_grafana
    restart: unless-stopped
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=admin123  # 修改为强密码
      - GF_USERS_ALLOW_SIGN_UP=false
      - GF_SERVER_ROOT_URL=http://localhost:3000
      - GF_INSTALL_PLUGINS=redis-datasource  # 安装Redis数据源插件
    volumes:
      - ./monitoring/grafana/provisioning:/etc/grafana/provisioning
      - grafana-data:/var/lib/grafana
    depends_on:
      - prometheus
    networks:
      - monitoring

  # ==================== Node Exporter (系统监控) ====================
  node-exporter:
    image: prom/node-exporter:v1.7.0
    container_name: archat_node_exporter
    restart: unless-stopped
    command:
      - '--path.procfs=/host/proc'
      - '--path.sysfs=/host/sys'
      - '--path.rootfs=/rootfs'
      - '--collector.filesystem.mount-points-exclude=^/(sys|proc|dev|host|etc)($$|/)'
    ports:
      - "9100:9100"
    volumes:
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /:/rootfs:ro
    networks:
      - monitoring

  # ==================== MySQL Exporter ====================
  mysql-exporter:
    image: prom/mysqld-exporter:v0.15.1
    container_name: archat_mysql_exporter
    restart: unless-stopped
    ports:
      - "9104:9104"
    environment:
      - DATA_SOURCE_NAME=root:root@(mysql:3306)/
    command:
      - '--collect.info_schema.innodb_metrics'
      - '--collect.info_schema.processlist'
      - '--collect.info_schema.query_response_time'
      - '--collect.info_schema.tables'
      - '--collect.perf_schema.tableiowaits'
      - '--collect.perf_schema.tablelocks'
    depends_on:
      - mysql
    networks:
      - monitoring
      - default  # 连接到原有网络访问MySQL

  # ==================== Redis Exporter ====================
  redis-exporter:
    image: oliver006/redis_exporter:v1.55.0
    container_name: archat_redis_exporter
    restart: unless-stopped
    ports:
      - "9121:9121"
    environment:
      - REDIS_ADDR=archat_redis:6379
    depends_on:
      - redis
    networks:
      - monitoring
      - default  # 连接到原有网络访问Redis

  # ==================== RabbitMQ Exporter (可选) ====================
  rabbitmq-exporter:
    image: kbudde/rabbitmq-exporter:v1.0.0-RC19
    container_name: archat_rabbitmq_exporter
    restart: unless-stopped
    ports:
      - "9419:9419"
    environment:
      - RABBIT_URL=http://archat_rabbitmq:15672
      - RABBIT_USER=admin
      - RABBIT_PASSWORD=admin123
    depends_on:
      - rabbitmq
    networks:
      - monitoring
      - default  # 连接到原有网络访问RabbitMQ

networks:
  monitoring:
    driver: bridge
  default:
    external: true
    name: archat_default  # 连接到原有 docker-compose 的网络

volumes:
  prometheus-data:
  grafana-data:
```

```yaml
# docker-compose.monitoring.yml 中
networks:
  default:
    external: true
    name: archat_default  # 改为实际网络名
```

查看实际网络名:
```bash
docker network ls
# 找到类似 archat_default 或 archat_archat_default 的网络名
```

### 2. 数据持久化说明

**Prometheus 数据目录**:
- 容器内路径: `/prometheus`
- 宿主机映射: `prometheus-data` volume
- 数据保留: 15天(可在 `--storage.tsdb.retention.time` 调整)

**Grafana 数据目录**:
- 容器内路径: `/var/lib/grafana`
- 宿主机映射: `grafana-data` volume

**备份数据**:
```bash
# 查看 volume 位置
docker volume inspect archat_prometheus-data

# 备份(示例)
docker run --rm -v archat_prometheus-data:/data -v d:/backup:/backup alpine tar czf /backup/prometheus-backup.tar.gz /data
```

### 3. 资源限制(生产环境推荐)

在 `docker-compose.monitoring.yml` 中添加:
```yaml
services:
  prometheus:

        limits:
          cpus: '1.0'
          memory: 2G

          cpus: '0.5'

    # ... 其他配置
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M

          cpus: '0.25'
```

## 🐛 常见问题排查

### 问题1: Prometheus targets 显示 DOWN

**现象**:
```
http://localhost:9090/targets 中某些 target 状态为 DOWN (红色)
```

**排查步骤**:
```bash
# 1. 检查目标服务是否运行
docker ps | grep exporter

# 2. 检查网络连通性
docker exec archat_prometheus ping mysql-exporter

# 3. 手动测试指标接口
curl http://localhost:9104/metrics  # 替换为对应端口

# 4. 查看 Prometheus 日志
docker logs archat_prometheus
```

**常见原因**:
- ❌ 目标服务未启动 → `docker-compose up -d` 启动
- ❌ 网络不通 → 检查 `networks` 配置
- ❌ 端口映射错误 → 检查 `ports` 配置
- ❌ 数据库连接失败 → 检查 `DATA_SOURCE_NAME` 环境变量

### 问题2: Grafana 无法连接 Prometheus

**现象**:
```
Grafana 数据源测试失败: "HTTP Error Bad Gateway"
```

**排查步骤**:
```bash
# 1. 从 Grafana 容器内测试 Prometheus
docker exec archat_grafana curl http://prometheus:9090/-/healthy

# 2. 检查网络
docker network inspect monitoring
```

**解决方案**:
- 确保 Prometheus 和 Grafana 在同一网络
- 检查 `datasources/prometheus.yml` 中 URL 配置

### 问题3: Windows 下 Node Exporter 无法采集完整指标

**现象**:
```
部分系统指标缺失,如 filesystem、cpu 等
```

**原因**: Windows 下 Docker 运行在 WSL2,需要挂载 WSL 路径

**解决方案**(仅 Windows):
```yaml
node-exporter:
  # ... 其他配置
  volumes:
    - /proc:/host/proc:ro
    - /sys:/host/sys:ro
    - /:/rootfs:ro
  # Windows 下可能需要调整挂载路径
```

**备选方案**: 使用 [windows_exporter](https://github.com/prometheus-community/windows_exporter)(需要额外安装)

### 问题4: 磁盘空间不足

**现象**:
```
docker logs archat_prometheus
# 输出: "no space left on device"
```

**排查磁盘占用**:
```bash
# 查看 Docker 磁盘占用
docker system df

# 清理无用镜像/容器/卷
docker system prune -a --volumes
```

**调整数据保留时间**:
```yaml
prometheus:
  command:
    - '--storage.tsdb.retention.time=7d'  # 从15天改为7天
```

## 🔄 日常运维命令

### 启动/停止服务
```bash
# 启动监控服务
docker-compose -f docker-compose.monitoring.yml up -d

# 停止监控服务
docker-compose -f docker-compose.monitoring.yml down

# 重启某个服务
docker-compose -f docker-compose.monitoring.yml restart prometheus

# 查看日志
docker-compose -f docker-compose.monitoring.yml logs -f prometheus
docker-compose -f docker-compose.monitoring.yml logs -f grafana
```

### 配置热加载
```bash
# 修改 prometheus.yml 后,热加载配置(无需重启)
curl -X POST http://localhost:9090/-/reload

# 或者重启容器
docker restart archat_prometheus
```

```bash
# Prometheus 健康检查

# 输出: Prometheus is Healthy.

curl http://localhost:3000/api/health
# 输出: {"commit":"...","database":"ok","version":"10.2.2"}
```
