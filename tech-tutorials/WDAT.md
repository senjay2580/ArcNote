铜线本质上是一种导电材料，能够通过电流传输信号。电流可以携带不同类型的信息，包括：

1. **模拟信号**：在传统的电话通信中，信号以模拟的形式传输，这意味着信号的波形是连续的，可以代表声音、语音等自然现象的变化。
2. **数字信号**：计算机网络和拨号上网使用的是数字信号。这些信号通过对电流的开关（0 和 1）进行编码，表示信息。

### **模拟信号传输**

在模拟信号传输中，信号是通过 **电流的强弱** 或 **电压的变化** 来表示的。模拟信号通常是 **连续的**，它的变化可以精确地表示各种信息。具体来说：

- **电流（或电压）代表信号的幅度**：比如在音频信号传输中，电流的强弱或者电压的高低就代表声音的强弱。
- **波形表示信息**：信号的 **波形**（如正弦波、方波等）通过变化的电流或电压来传递不同的信息。例如，音频信号就是通过电流的波动（声波）传递声音的内容。

**例子：**

- 你在收听电台时，收音机通过电流的变化来获取无线电波（模拟信号）。不同的电流强度代表不同的声音频率，进而播放出声音。
- 在模拟电视信号中，电压波动表示画面和声音的信息。

### **2. 数字信号传输**

在数字信号传输中，信号通常通过 **电流的开关状态** 或 **电压的高低** 来表示信息。数字信号是由 **0** 和 **1** 组成的离散信号，通过二进制的方式进行传输。

- 开关信号

  ：通过电流的通与断（或电压的高与低）表示不同的数字信号。例如：

  - **0** 可能对应电压低或电流停止。
  - **1** 可能对应电压高或电流通过。

- **电流变化表示二进制数字**：例如，在 USB 数据传输中，电流的变化通过 **高电平** 和 **低电平** 的电压信号来表示二进制信息（0 或 1），进而传输数据。

**例子：**

- **以太网**：通过电流的变化来传递数字信号，数据通过电流的“跳变”表示每个比特（0 或 1）。
- **串行通信（比如 RS-232 或 USB）**：使用电流的 **电平变化** 来传递数据，通常是 **高电平**（代表 1）和 **低电平**（代表 0）。

### **3. 载波信号（Modulation）**

在很多情况下，我们会使用一种叫做 **调制（Modulation）** 的技术来将模拟信号或数字信号加载到电流上，这样可以使信号在通信过程中更稳定，或者在更大的距离上传输。

- **调幅（AM）**、**调频（FM）** 和 **调相（PM）** 是常见的模拟调制方法，它们通过改变载波信号（电流的幅度、频率或相位）来传递信息。
- **数字调制**（如 **ASK（幅度键控）**、**FSK（频率键控）**）则通过 **改变电流的幅度或频率** 来代表数字信号的 0 和 1。

**例子：**

- **无线广播**：广播站通过调制广播电波（即调制电流的频率或幅度），将音频信号传输到收音机。
- **Wi-Fi**：Wi-Fi 信号通过调制电波的 **频率** 或 **幅度** 来传递数字数据。

------

### **4. 电流携带信号的过程**

让我们通过一个简单的示例来更具体地理解：

**无线通信中的电流信号传输：**

1. **信息编码**：首先，发送端需要将 **要传输的信息**（例如语音、图像、数据等）转换成 **电信号**。如果是模拟信号，信息将直接用电流的幅度变化表示；如果是数字信号，信息将转化为一系列的 0 和 1，表示为高电压和低电压的变化。
2. **调制过程**：对于长距离传输，信号通常会通过调制过程加载到一个载波信号上。这样做的目的是为了让信号能在更远的距离上传播，而且不容易受到干扰。
3. **信号传输**：调制后的电流信号通过电缆、光纤或无线方式传输到接收端。
4. **信号解调**：接收端收到信号后，需要对其进行解调，即将传输的电流信号转换回原始的信息。对于模拟信号，就是恢复电流的幅度波动；对于数字信号，就是识别电流的高低电平。
5. **信息输出**：接收端将解调后的信息输出到相应的设备（例如电视、计算机、手机等），让用户能够听到声音、看到图像或读取数据。







## **1. PPPoE（Point-to-Point Protocol over Ethernet）是什么？**

PPPoE（以太网上的点对点协议）是一种用于 **宽带拨号上网** 的网络协议，常用于家庭宽带、校园网、ADSL（早期的电话线上网）等网络环境。

**PPPoE 的工作方式**：

- 你的设备（电脑、路由器）需要提供一个 **用户名和密码**（由网络服务提供商 ISP 提供）。
- 设备通过 **PPPoE 协议拨号**，建立一个基于以太网的点对点连接（类似于拨号上网）。
- 连接成功后，ISP 服务器会给设备分配一个 **IP 地址**，然后才能访问互联网。

### **PPPoE 主要应用场景**

- **校园网**：大多数校园网要求学生使用 PPPoE 账号（用户名+密码）拨号上网。
- **ADSL 宽带**：过去家庭用的 ADSL 宽带需要用 PPPoE 拨号，才能访问互联网。
- **部分光纤宽带**：一些老旧运营商的光纤网络仍然使用 PPPoE 拨号。

------

## **2. PPPoE 与路由器**

### **（1）路由器如何使用 PPPoE？**

如果你的网络是 **PPPoE 方式**，则需要 **在路由器上进行拨号**：

1. 进入路由器的管理界面（通常是 `192.168.1.1` 或 `192.168.0.1`）。
2. 在 **WAN 口设置** 选择 **PPPoE** 作为连接方式。
3. 输入宽带账号和密码（由 ISP 或校园网提供）。
4. 连接成功后，路由器就可以访问互联网，并通过 **Wi-Fi 或 LAN 口** 共享网络给其他设备（如电脑、电视等）。

### **（2）为什么需要路由器？**

- 单个设备拨号 vs. 路由器拨号

  ：

  - 如果 **直接用电脑拨号**（Windows/Mac 通过 PPPoE 连接），每次只有 **一台设备** 能上网。
  - **如果用路由器拨号**，路由器会作为“桥梁”，所有连接到路由器的设备（手机、电视、电脑等）都能同时上网。

------

## **3. PPPoE 与电视**

电视（尤其是 **智能电视** 或 **机顶盒**）连接互联网主要有以下几种方式：

- **Wi-Fi 连接路由器**：如果你的路由器使用 PPPoE 拨号，电视可以直接连接 Wi-Fi，无需额外配置。
- **有线网口连接路由器**：部分电视或 IPTV 机顶盒支持 **网线直连** 路由器（LAN 口）。
- **直接拨号（极少见）**：如果电视**没有连接路由器**，而是直接接入校园网/运营商网络（如光猫的 LAN 口），那么可能需要 **在电视上手动拨号**（但大多数电视系统不支持）。

### **（1）校园网 + 电视**

如果你的 **校园网** 采用 PPPoE 认证，而电视无法直接拨号，你可以：

1. **用电脑拨号后共享网络**（Windows 共享网络给电视，但不方便）。
2. **使用路由器拨号**（推荐）：路由器连接校园网并拨号，电视连接路由器即可上网。

### **（2）IPTV 电视**

- **部分宽带运营商提供 IPTV**（交互式电视），通常需要**专用 VLAN 或者特殊账号**。
- **如果是 IPTV 专线**，运营商可能要求将 **光猫的 LAN2 口直连机顶盒**。
- **如果是互联网电视**（比如小米电视、华为智慧屏），它跟普通设备一样连接路由器就可以上网，不需要特殊配置。

------

## **4. PPPoE 与校园网**

在校园网中，PPPoE 主要用于学生上网认证：

- 学生需要使用学校提供的 **PPPoE 账号**（用户名和密码）拨号上网。

- **问题 1：电脑能上网，但手机/电视无法使用？**

  - 直接拨号的方式只能让一台设备上网。

  - 解决方案

    ：

    - **用路由器拨号**，然后让所有设备连接路由器的 Wi-Fi。
    - **使用校园网 APP 认证**（部分学校允许在网页或 APP 登录，而非 PPPoE 拨号）。

- **问题 2：部分校园网限制路由器拨号？**

  - 一些学校限制了 **每个账号的设备数量**，或者检测到 **路由器拨号** 会封锁连接。
  - 解决方法：
    - 试试 **路由器 MAC 克隆**（模拟电脑的 MAC 地址）。
    - **咨询学校网络中心**，看是否有**多设备绑定方案**。

------

## **5. 总结**

| 设备       | 作用                                             |
| ---------- | ------------------------------------------------ |
| **PPPoE**  | 一种拨号上网方式，需要输入账号和密码连接互联网   |
| **路由器** | 负责拨号并共享网络给多个设备                     |
| **电视**   | 需要网络连接（Wi-Fi 或网线），可以通过路由器上网 |
| **校园网** | 可能需要 PPPoE 认证，通常限制设备数量            |

### **如何让校园网多个设备上网？**

1. 直接用 **电脑拨号**（但只能一台设备用）。
2. **用路由器拨号**，然后所有设备连接路由器的 Wi-Fi。
3. **检查校园网是否支持网页认证**，如果支持，可以试试用 APP 登录。

### **总结关系**

- **PPPoE 拨号 → 让设备连接互联网**。
- **路由器拨号 → 让多个设备共享网络**。
- **电视（尤其是智能电视）需要互联网 → 通过 Wi-Fi 或网线连接路由器**。
- **校园网通常用 PPPoE 认证 → 需要使用 PPPoE 账号连接，或者让路由器拨号后共享网络**。

### 前后端分离式项目部署

### **1. 部署后端服务**

假设后端是一个 Web 服务（例如 Spring Boot、Node.js 等），你需要将其部署到云服务器上，并确保它能够通过云服务器的公网 IP 访问。

#### **步骤：**

- **选择云服务商**：选择一个云服务提供商（如 AWS、阿里云、腾讯云等）并创建一个云服务器（虚拟机）。

- 部署后端应用

  ：

  - 将后端应用部署到云服务器上（例如将 JAR 文件上传到服务器，并通过 `java -jar` 启动 Spring Boot 应用，或者通过 Docker 部署应用）。
  - 确保后端服务监听在某个端口（例如 8080）并能够接受外部请求。
  - 配置防火墙规则，允许外部通过指定端口访问你的后端服务。

#### **公网访问：**

- 云服务商通常会为你分配一个公网 IP 地址，确保该 IP 可以访问你部署的后端服务。

- 开放端口

  ：你需要在云服务器的安全组或者防火墙中配置规则，允许外部请求访问后端应用的端口（如 8080）。

  - 例如，阿里云的安全组需要添加规则，允许公网 IP 访问 `8080` 端口。

------

### **2. 部署前端服务（Nginx）**

前端通常会通过构建好的静态文件（HTML、CSS、JS）进行服务，可以将前端部署在 Nginx 上。

#### **步骤：**

- **准备前端代码**：前端代码编译完成后（例如 React、Vue 项目构建），将其静态文件（如 `index.html` 和 `assets` 文件夹）上传到云服务器。

- **安装 Nginx**：在云服务器上安装 Nginx。

  - 使用以下命令在云服务器上安装 Nginx（以 Ubuntu 为例）：

    ```bash
    sudo apt update
    sudo apt install nginx
    ```

- **配置 Nginx**：

  - 将前端构建的静态文件放到 Nginx 配置的目录下（通常是 `/var/www/html`）。

  - 配置 Nginx，使其能够处理来自用户的 HTTP 请求，提供前端页面。

  - 编辑 `/etc/nginx/sites-available/default` 或者自定义配置文件，配置前端和后端的反向代理。

    例如，如果后端 API 服务运行在 `8080` 端口，而前端静态文件通过 Nginx 提供，可以设置反向代理：

    ```nginx
    server {
        listen 80;
    
        server_name your-domain.com;
    
        location / {
            root /var/www/html;
            index index.html;
        }
    
        location /api/ {
            proxy_pass http://127.0.0.1:8080/;
        }
    }
    ```

    这样，所有以 `/api/` 开头的请求会被转发到后端服务，其他请求则会返回前端静态页面。

- **启动 Nginx**：

  - 配置完成后，启动 Nginx：

    ```bash
    sudo systemctl restart nginx
    ```

  - 确保 Nginx 启动并且没有错误。

### **1. 网站常见的做法**

在现代 Web 应用中，通常的做法是将 **前端** 部署为用户访问的入口，而将 **后端服务** 通过 API 提供数据，且不会直接暴露给终端用户。前端和后端通过不同的端口或者路径进行区分。

#### **常见做法的流程**：

- **前端**：提供页面（HTML、CSS、JS）和用户界面。通常部署在 **Web 服务器（如 Nginx、Apache）** 上，用户通过 HTTP 或 HTTPS 访问。
- **后端**：提供 API（例如 RESTful API）来处理数据、业务逻辑等。后端服务不直接提供给用户，而是通过 **前端调用后端 API**（通常是通过 AJAX 或 Fetch 请求）来获取数据。

#### **访问流程**：

1. 用户访问网站时，实际上访问的是前端的静态文件（如 `index.html`），通过浏览器渲染前端页面。
2. 前端页面通过 API 请求向后端获取数据（例如 `GET /api/products`）。
3. 后端提供必要的数据，通过接口响应前端的请求。

### **2. 公网 IP 分配**

通常，**公网 IP 只需要分配给前端服务**，并通过 **反向代理** 或 **前端和后端分离** 的方式来实现前端和后端的交互。

#### **前端部署（Nginx）**：

- 在这种架构下，**前端** 会通过 Web 服务器（如 **Nginx**）暴露给用户。Nginx 会监听公网 IP 上的 80 或 443 端口，提供前端页面。
- 前端页面会包含调用后端 API 的逻辑，但是这些 API 请求会通过 **Nginx 反向代理** 转发到后端，而不是直接暴露给用户。

#### **后端 API 部署**：

- 后端服务通常会运行在云服务器的其他端口（如 `8080`），并不会直接对外开放给用户访问。
- 后端 API 只能通过 **前端发起的请求**来访问，通常前端会通过 `/api/` 路径与后端交互（例如，`http://your-public-ip/api/`）。
- 如果需要保护后端不被直接访问，可以在云服务中配置安全组规则，**仅允许前端服务器** 或 **内部服务** 访问后端 API。

### **3. Nginx 配置反向代理**

为了确保前端可以访问后端而不暴露后端给用户，你可以在 Nginx 中配置反向代理。以下是一个基本的示例：

#### **Nginx 配置示例**：

假设你希望用户访问 `http://your-public-ip/` 时，看到的是前端页面，而访问 `/api/` 时则会被代理到后端服务。

```
nginx复制代码server {
    listen 80;

    server_name your-public-ip;  # 你的公网 IP 或域名

    # 配置前端页面路径
    location / {
        root /var/www/html;  # 前端页面存放路径
        index index.html;
    }

    # 配置反向代理，将 /api 路径的请求转发到后端服务
    location /api/ {
        proxy_pass http://127.0.0.1:8080/;  # 后端服务地址
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### **如何工作**：

- 用户访问 `http://your-public-ip/` 时，Nginx 会提供前端页面。
- 用户访问 `http://your-public-ip/api/` 时，Nginx 会将请求代理到本地的后端服务（`127.0.0.1:8080`）。
- 后端 API 仍然通过 `/api/` 路径可访问，但是**后端的端口（8080）不会直接暴露给用户**，所有的访问都通过 Nginx 转发。

### **4. 安全性与隐藏后端**

通过前端和后端分离的方式，后端的真实 IP 和端口会隐藏在服务器内网中，只能通过 **前端的 API 调用**与后端交互。外部用户不能直接访问后端服务的端口，这样既能保证 **前端和后端的解耦**，也能增强 **安全性**。





**Vue 项目静态文件的访问原理**

1. 本地开发环境（npm run serve）
Vue CLI 默认启动一个开发服务器（如 webpack-dev-server），运行在 localhost:8080（端口可能不同）。

所有静态资源（包括实时编译的 JS/CSS）由该服务器托管。

通过内网穿透工具（如 cpolar）将 localhost:8080 暴露到公网，外部用户访问穿透地址（如 xxx.cpolar.cn）即可加载所有资源。

2. 生产环境构建（npm run build）
执行 npm run build 后，生成静态文件到 dist 目录。

需要将这些文件部署到 Web 服务器（如 Nginx、Apache、或静态服务器如 serve）。

通过内网穿透将 Web 服务器的端口（如 80）暴露到公网，外部用户访问穿透地址即可加载静态资源。
