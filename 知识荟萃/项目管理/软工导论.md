# 第一章

**过算机软件指计算机系统中的程序及其文档**

**软件分类**：系统软件、支撑软件、应用软件

**软件工程且标**：指生产具有正确性、可用性和开销合宜的产品。正确性是指软件产品达到预期功能的程度。

**软件过程** 是生产一个最终满足需求且达到工程目标的软件产品所需的步骤。

GB/T 8566 一 2007 标准把软件生存周期中可以开展的活动分为 **5 个基本过程、9 个支持过程和 7 个组织过程。**

# 第二章

**可行性分析：** 经济可行性、技术可行性（风险分析、资源分析、技术分析 ！哪些情况属于这些分析） 法律可行性、

**方案的选择和折衷** 一个基于计算机的系统可以有多个可行的实现方案，每个方案对成本、时间、人员、技术、设备都有不同的要求，不同方案开发出来的系统在功能、性能方面也会有所不同。因此要在多个可行的实现方案中作出选择。方案评估的依据是待开发系统的功能、性能、成本、开发时间、采用的技术、设备、风险以及对开发人员的要求等。

# 第四章

软件设计的任务：

**(1) 数据/类设计：** 数据/类设计将分析类模型变换成类的实现和软件实现所需要的数据结构。

**(2)体系结构设计：** 体系结构设计定义了软件的整体结构，由软件部件，外部可见的属性和它们之间的关系组成。

(3)**接口设计**：接口设计描述了软件内部、软件和协作系统之间以及软件同人之间的通信方式

(4) **部件级设计**：部件级设计将软件体系结构的结构性元素恋换为对软作部件的过程性描述。

# ==第十三章==：软件测试（大头）

白盒测试、黑盒测试

黑盒测试方法有：**等价类划分，边界值分析，比较测试，错误猜测和因果图方法。**

等价类可分为 **有效等价类和无效等价类。** （要知道意思都是什么）

基本路径测试 （计算复杂度什么的）**边界值分析要看**

**单元测试**（也要看）又称模块测试着重对软件设计的最小单元一软件构件或模块进行测试。

测试模块时必须为每个被测模块开发一个驱动(driver)程序和若干个桩(stub)模块。



集成测试又称组装测试，经单元测试后的模块需集成为软件系统，集成测试是对集成后的软件系统进行测试，主要用来揭露设计阶段产生的错误。

主要的调试方法有 3 种：蛮力法(brute force)、回溯法(backtracking）和原因排除法 cause elimination)。

由于软件项目的特殊性，将项目管理技术用于软件项目管理上，其有效的项目管理集中于4个P上：人员(people)、产品(product).、过程(process)和项目(project）

软件规模通常用程序的代码行(line of code,LOC)或千行代码KLOC(10 QLOC)来衡量。

Albrecht于1979年首次提出了面向功能的度量，它是一种针对软件的功能特性进行度量的方法，该方法主要考虑软件系统的“功能性”和“实用性”。**他建议一种称为“功能点”(function point,FP)的测量，功能点是基于软件信息域的特征（可直接测量）和软件复杂性进行计算的]**
