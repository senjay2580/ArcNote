# 附录 - 硬件专业术语表

> 涵盖课程术语 + 专业硬件知识库 + 业界专家常用名词，按分类排列，每类内部从最常用到最少用排序。
> 来源参考: CompTIA A+、Gamers Nexus、Buildzoid(Actually Hardcore Overclocking)、der8auer、Level1Techs、AnandTech、Tom's Hardware 等。

---

## 一、接口与连接

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **USB** | Universal Serial Bus | 通用接口，插鼠标键盘U盘充电线。USB 2.0(480Mbps)、3.0(5Gbps)、3.2 Gen2(10Gbps)、USB4(40Gbps) |
| **HDMI** | High-Definition Multimedia Interface | 视频+音频输出接口。HDMI 2.0(4K60Hz)、2.1(4K120Hz/8K60Hz) |
| **SATA** | Serial Advanced Technology Attachment | 连接硬盘的接口和协议。需要数据线+电源线，速度上限 6Gbps(约550MB/s)，较老较慢 |
| **PCIe** | Peripheral Component Interconnect Express | 高速数据通道，像高速公路。x1/x4/x16 = 车道数，3.0/4.0/5.0 = 每车道速度。显卡用x16，SSD用x4 |
| **M.2** | 无全称 | 一种小型插槽的外形标准，口香糖大小直接插主板，不需要线。M.2只是形状，不代表速度 |
| **NVMe** | Non-Volatile Memory Express | 专为SSD设计的高速传输协议，走PCIe通道。比SATA快几十倍 |
| **DP** | DisplayPort | 视频输出接口。DP 1.4(8K30Hz)、2.0(16K60Hz)，电竞显示器首选 |
| **Type-C** | USB Type-C | 正反都能插的椭圆形接口。注意Type-C只是外形，里面可以跑USB/雷电/视频等不同协议 |
| **Thunderbolt** | 雷电接口 | Intel开发的超高速接口，外形=Type-C。TB3(40Gbps)、TB4(40Gbps更严格)、TB5(80Gbps) |
| **RJ45** | Registered Jack 45 | 网线接口，方形口。支持1GbE/2.5GbE/10GbE等不同速率 |
| **mDP** | Mini DisplayPort | DP的缩小版，老款MacBook和部分显卡上有 |
| **VGA** | Video Graphics Array | 老式模拟视频接口(蓝色梯形口15针)，基本淘汰 |
| **DVI** | Digital Visual Interface | 比VGA新但也过时的视频接口。DVI-D(纯数字)、DVI-I(数字+模拟) |
| **DMI** | Direct Media Interface | CPU和芯片组之间的专用通道，带宽约8GB/s，是系统瓶颈 |
| **CNVi** | Connectivity Integration | Intel技术，WiFi卡拆成两半: 射频在M.2卡上，MAC/基带在芯片组里。不能换非CNVi网卡 |
| **u.FL** | 微型射频连接器 | WiFi网卡上的天线接头，靠按压卡住 |
| **SFF-8639 / U.2** | - | 企业级2.5寸NVMe SSD的接口，比M.2散热好，消费级很少见 |
| **OCuLink** | Optical-Copper Link | 高速PCIe外接接口，小外形机箱外接显卡用，比TB带宽大 |
| **SAS** | Serial Attached SCSI | 企业级硬盘接口，比SATA更可靠更快，服务器用 |
| **MCIO** | - | 新一代服务器内部PCIe连接器，替代SlimSAS |
| **CXL** | Compute Express Link | 基于PCIe的新互连协议，让CPU和加速器/内存共享数据，数据中心用 |

## 二、CPU 处理器

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **CPU** | Central Processing Unit | 中央处理器，执行所有指令和计算 |
| **核心(Core)** | - | CPU里独立的计算单元。6核=6个计算单元同时干活 |
| **线程(Thread)** | - | 逻辑计算单元。超线程让1核当2线程用。6核12线程=6物理核心12逻辑线程 |
| **主频(Clock Speed)** | - | 每秒时钟周期数，单位GHz。主频越高单任务越快 |
| **睿频(Turbo Boost)** | - | CPU自动提高到的最大频率。基础2.4GHz/睿频4.9GHz=忙时自动加速到4.9 |
| **IPC** | Instructions Per Clock | 每个时钟周期执行的指令数。IPC越高=同频率下性能越强。不同架构IPC不同，所以不能只看频率比性能 |
| **缓存(Cache)** | - | CPU内部超高速存储。L1(最快最小几十KB) - L2(中间几百KB) - L3(最慢最大几十MB) |
| **TDP** | Thermal Design Power | 热设计功耗(瓦特)，散热器要压住这个热量 |
| **PBP / MTP** | Processor Base Power / Max Turbo Power | Intel新命名，PBP=基础功耗(原TDP)，MTP=最大睿频功耗(实际更高) |
| **制程(nm)** | Process Node | 芯片制造工艺。数字越小越先进(晶体管更小更多更省电) |
| **超频(OC)** | Overclocking | 手动提高频率提升性能，但发热增大可能不稳定 |
| **核显(iGPU)** | Integrated GPU | 集成在CPU内部的显卡。Intel带F后缀=没核显，AMD带G后缀=有核显 |
| **超线程(HT/SMT)** | Hyper-Threading / SMT | 让一个物理核心当两个逻辑核心用。Intel叫HT，AMD叫SMT |
| **微架构(Microarchitecture)** | - | CPU的内部设计方案。如Intel的Raptor Lake、AMD的Zen 4。决定IPC和效率 |
| **流水线(Pipeline)** | - | CPU把一条指令拆成多个阶段(取指-解码-执行-写回)像工厂流水线一样并行处理 |
| **乱序执行(OoOE)** | Out-of-Order Execution | CPU不按程序顺序、而是按数据就绪情况调整执行顺序，提高效率 |
| **分支预测(Branch Prediction)** | - | CPU猜测if/else会走哪个分支提前执行，猜错就丢弃重来。现代CPU准确率>95% |
| **Die(裸片/晶圆切片)** | - | 从硅晶圆上切下来的一小块芯片。CPU拆掉顶盖看到的那块小方片就是die |
| **Chiplet(小芯片)** | - | 把一个大die拆成多个小die再封装到一起。AMD Ryzen用这个技术降低成本提高良率 |
| **IOD / CCD** | I/O Die / Core Complex Die | AMD Chiplet架构里，CCD是计算核心芯片，IOD是I/O和内存控制器芯片 |
| **Infinity Fabric(IF)** | - | AMD芯片内部/芯片间的互连总线，连接CCD和IOD |
| **Ring Bus / Mesh** | - | Intel CPU内部核心间的互连方式。Ring Bus=环形总线(少核心用)，Mesh=网格(多核心用) |
| **P-Core / E-Core** | Performance / Efficiency Core | Intel 12代起的混合架构。P核=性能核(强)，E核=能效核(省电)，调度器自动分配任务 |
| **Binning(体质分选)** | - | 工厂测试每个CPU，性能好的标为高端型号，差的标为低端。雷=体质差，雕=体质好 |
| **Stepping** | - | CPU的修订版本号。同型号不同stepping可能修了bug或略有改进。如B0、C0 |
| **Wafer(晶圆)** | - | 制造CPU的圆形硅片，直径300mm，一片切出几百个die |
| **良率(Yield)** | - | 晶圆上合格die的比例。良率越高成本越低，制程越先进良率越难提高 |
| **TIM** | Thermal Interface Material | CPU die和顶盖(IHS)之间的导热材料。高端CPU用焊锡，低端用硅脂 |
| **IHS** | Integrated Heat Spreader | CPU顶盖，那块金属盖子。把die的热量分散到更大面积再传给散热器 |
| **开盖(Delidding)** | - | 拆掉CPU的IHS顶盖，更换里面的TIM为液态金属，降低温度。有风险，发烧友玩法 |

## 三、CPU 封装与安装

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **Socket** | 插槽 | CPU安装到主板的接口类型。CPU和主板Socket必须一致 |
| **LGA** | Land Grid Array | 触点在CPU上，针脚在主板插槽里。Intel和AMD AM5用这个 |
| **PGA** | Pin Grid Array | 针脚在CPU上。AMD AM4及以前用，针脚弯了很难修 |
| **BGA** | Ball Grid Array | 锡球焊死在主板上，不可拆。笔记本CPU/GPU都是BGA |
| **LGA 1700** | - | Intel 12/13/14代酷睿的插槽 |
| **LGA 1851** | - | Intel Core Ultra 200系列的插槽 |
| **AM4** | - | AMD Ryzen 1000-5000系列的插槽(PGA) |
| **AM5** | - | AMD Ryzen 7000/9000系列的插槽(LGA) |
| **SP5 / TR5** | - | AMD EPYC服务器/线程撕裂者的插槽 |
| **Contact Frame(接触框架)** | - | 替换LGA 1700原装扣具的第三方框架，改善CPU与散热器的接触压力分布，降温几度 |

## 四、内存

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **RAM** | Random Access Memory | 随机存取存储器=内存。运行时临时存数据，断电就丢 |
| **DDR4 / DDR5** | Double Data Rate | 内存代数。DDR5更快更新，和DDR4插槽不兼容 |
| **双通道(Dual Channel)** | - | 两条内存同时工作带宽翻倍。4插槽时插2和4号位 |
| **频率(MHz)** | - | 内存工作速度。DDR4常见3200，DDR5常见5600-6400 |
| **时序(CL)** | CAS Latency | 内存响应延迟。CL16/CL36，数字越小越好 |
| **DIMM** | Dual In-line Memory Module | 台式机内存条标准，约13.3cm |
| **SO-DIMM** | Small Outline DIMM | 笔记本内存条标准，约6.7cm |
| **XMP** | Extreme Memory Profile | Intel平台内存超频方案，BIOS里开启才能跑标称频率 |
| **EXPO** | Extended Profiles for Overclocking | AMD平台内存超频方案，和XMP同一个意思 |
| **JEDEC** | Joint Electron Device Engineering Council | 内存标准制定组织。JEDEC频率=默认保守频率，XMP/EXPO=超频频率 |
| **Rank(列)** | - | 内存条上芯片的分组。单列(1R)、双列(2R)。双列性能略好但对主板兼容性要求高 |
| **Bank** | - | 内存芯片内部的存储区块。Bank越多并行度越高 |
| **tRCD** | RAS to CAS Delay | 行地址到列地址的延迟。和CL一起决定实际延迟 |
| **tRP** | Row Precharge Time | 行预充电时间 |
| **tRAS** | Row Active Time | 行激活时间 |
| **一时(First Word)延迟** | - | 用公式: 延迟(ns) = CL / 频率(MHz) x 2000。DDR4-3200 CL16 = 10ns，DDR5-6400 CL32 = 10ns，实际延迟差不多 |
| **UDIMM** | Unbuffered DIMM | 无缓冲内存，普通桌面电脑用的就是这个 |
| **RDIMM** | Registered DIMM | 带寄存器的内存，服务器用，支持更大容量 |
| **ECC** | Error-Correcting Code | 纠错内存，能自动检测修复1bit错误。服务器/工作站用，消费级主板大多不支持 |
| **LPDDR5** | Low Power DDR5 | 低功耗内存，手机和超薄笔记本用，焊死在主板上 |
| **CUDIMM / CSODIMM** | Clocked Unbuffered DIMM | DDR5新规格，内存条上加了时钟驱动芯片，支持更高频率更稳定 |
| **SPD** | Serial Presence Detect | 内存条上的小芯片，存储频率/时序/电压等参数信息。BIOS读SPD来识别内存 |
| **内存颗粒(IC)** | Integrated Circuit | 内存条上实际的存储芯片。三星B-die/海力士A-die等是超频玩家追捧的好颗粒 |

## 五、存储设备

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **SSD** | Solid State Drive | 固态硬盘，闪存芯片存数据，快、无噪音、抗震 |
| **HDD** | Hard Disk Drive | 机械硬盘，旋转磁盘+磁头，便宜大容量但慢、怕震 |
| **M.2 2280** | - | 22mm宽x80mm长，绝大多数NVMe SSD的尺寸 |
| **M.2 2230** | - | 22mm宽x30mm长，WiFi网卡和小型SSD用 |
| **M Key / B+M Key** | - | M.2金手指缺口位置。M Key=NVMe(快)，B+M=SATA(慢)，防呆设计 |
| **DRAM缓存** | - | SSD自带的小内存芯片，加速数据查找，有DRAM的SSD性能更稳 |
| **TBW** | Terabytes Written | SSD寿命指标，总共能写入多少TB。正常用5-10年没问题 |
| **NAND Flash(闪存)** | - | SSD里存数据的芯片。SLC(1bit/cell最耐用) > MLC(2bit) > TLC(3bit主流) > QLC(4bit最便宜) |
| **SLC Cache(模拟SLC缓存)** | - | TLC/QLC SSD用一部分空间模拟SLC模式加速写入。缓存用完后速度暴降(掉速) |
| **TRIM** | - | 操作系统告诉SSD哪些数据已删除可擦除的指令。不支持TRIM的SSD会越用越慢 |
| **GC(垃圾回收)** | Garbage Collection | SSD后台整理已删除数据、腾出干净空间的过程 |
| **Wear Leveling(磨损均衡)** | - | SSD控制器把写入均匀分散到所有闪存单元，避免某区域先坏 |
| **FTL** | Flash Translation Layer | 闪存转换层，把逻辑地址翻译成物理地址 |
| **Controller(主控)** | - | SSD上的处理器芯片，管闪存读写、纠错、磨损均衡。好主控=好性能+好寿命 |
| **HMB** | Host Memory Buffer | 无DRAM的SSD借用电脑内存作缓存，省成本方案 |
| **DWPD** | Drive Writes Per Day | 企业SSD寿命指标，每天能写满几次全盘 |
| **SMART** | Self-Monitoring Analysis and Reporting | 硬盘健康监测系统。CrystalDiskInfo读的就是SMART数据 |
| **RPM** | Revolutions Per Minute | 机械硬盘转速。5400(笔记本)、7200(台式)、10000/15000(企业) |
| **PMR / CMR** | Perpendicular/Conventional Magnetic Recording | 传统磁记录，性能正常。买HDD认准这个 |
| **SMR** | Shingled Magnetic Recording | 叠瓦式磁记录，随机写入差。便宜但不推荐做系统盘/NAS |
| **RAID** | Redundant Array of Independent Disks | 多硬盘组合。RAID 0(加速)、1(镜像)、5(校验)、10(镜像+加速) |
| **NAS** | Network Attached Storage | 网络存储，通过网线访问的私人云盘。群晖/威联通 |
| **3D NAND** | - | 闪存芯片垂直堆叠多层(如128层、232层)，同面积塞更多容量 |
| **OP(预留空间)** | Over-Provisioning | SSD预留一部分空间不给用户，用于GC和磨损均衡，提高寿命和性能 |

## 六、主板

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **主板** | Motherboard / Mainboard | 连接所有硬件的核心电路板 |
| **芯片组(PCH)** | Platform Controller Hub | 主板核心芯片，管USB/SATA/网卡等。如B760、Z790、B650、X870 |
| **VRM** | Voltage Regulator Module | 供电模块，CPU旁一排小方块，给CPU提供稳定电压 |
| **BIOS / UEFI** | Basic I/O System / Unified Extensible Firmware Interface | 主板固件，开机最先运行。设置启动顺序、调参数 |
| **PCB** | Printed Circuit Board | 印刷电路板，多层铜走线+焊接元件的板子 |
| **ATX** | Advanced Technology eXtended | 标准大板305x244mm，扩展性最好 |
| **mATX** | Micro-ATX | 244x244mm，主流性价比 |
| **Mini-ITX** | - | 170x170mm，迷你小板 |
| **E-ATX** | Extended ATX | 305x330mm，发烧/服务器大板 |
| **CMOS** | Complementary Metal-Oxide-Semiconductor | 由纽扣电池供电的存储区，保存BIOS设置和时间。拔电池=重置BIOS |
| **POST** | Power-On Self-Test | 开机自检，BIOS检测硬件是否正常 |
| **Debug灯 / Q-LED** | - | 主板上CPU/DRAM/VGA/BOOT四个灯，哪个亮=哪步出问题 |
| **北桥/南桥** | Northbridge/Southbridge | 老架构两芯片。现在北桥集成到CPU，只剩南桥(改叫PCH) |
| **I/O挡板** | I/O Shield | 机箱后方挡住主板接口的金属片 |
| **前面板接针** | Front Panel Header | 主板上连机箱开机键/USB/音频的小针脚 |
| **供电相数** | Phase Count | VRM有几组供电电路。12+2相=12相CPU+2相核显，越多越稳 |
| **PWM控制器** | Pulse Width Modulation Controller | VRM的大脑芯片，控制各相供电开关节奏 |
| **DrMOS / SPS** | Driver + MOSFET / Smart Power Stage | 驱动器+MOSFET集成一体的供电元件，效率高散热好 |
| **散热装甲** | Heatsink Armor | 覆盖VRM/M.2的金属散热片 |
| **BIOS Flashback** | - | 不装CPU不装内存也能刷BIOS，用U盘+按钮。新主板配老BIOS不认新CPU时救命 |
| **Clear CMOS** | - | 重置BIOS到出厂。跳线短接/按钮/拔电池 |
| **TPM** | Trusted Platform Module | 安全芯片存加密密钥。Win11要求TPM 2.0(现在多集成在CPU=fTPM) |
| **Q-Flash / M-Flash / EZ Flash** | - | 技嘉/微星/华硕在BIOS里刷固件的功能名 |
| **ACPI** | Advanced Configuration and Power Interface | 电源管理和硬件配置标准，控制睡眠/休眠/唤醒等 |
| **IPMI / BMC** | Intelligent Platform Management Interface | 服务器远程管理接口，可远程开关机/看BIOS/装系统 |

## 七、显卡

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **GPU** | Graphics Processing Unit | 图形处理器，显卡核心芯片 |
| **独显(dGPU)** | Discrete GPU | 独立显卡，有自己的显存和散热 |
| **显存(VRAM)** | Video RAM | 显卡专用内存。8/12/16/24GB |
| **GDDR6 / GDDR6X** | Graphics DDR6 | 显存类型，和普通DDR内存不同 |
| **CUDA Core** | Compute Unified Device Architecture | NVIDIA并行计算核心，数量越多越强 |
| **Stream Processor(流处理器)** | - | AMD显卡计算核心，和CUDA对应 |
| **RT Core** | Ray Tracing Core | NVIDIA光追硬件单元，RTX才有 |
| **Tensor Core** | - | NVIDIA AI/深度学习计算单元，DLSS用这个 |
| **光线追踪(Ray Tracing)** | - | 模拟真实光线的渲染技术，画面真实但吃性能 |
| **DLSS** | Deep Learning Super Sampling | NVIDIA AI超分辨率，低分辨率渲染再AI放大，帧数大增画质损失小 |
| **FSR** | FidelityFX Super Resolution | AMD超分辨率，和DLSS类似但不需专用硬件，A卡N卡都能用 |
| **帧生成(Frame Generation)** | - | DLSS 3/FSR 3功能，AI生成中间帧提高帧率 |
| **RTX / GTX** | - | NVIDIA系列。RTX=有光追，GTX=无光追(老系列) |
| **TGP / TBP** | Total Graphics Power / Total Board Power | 显卡整卡功耗，决定需要多大电源 |
| **位宽(Bus Width)** | - | 显存和GPU间通道宽度。128/192/256/384bit |
| **公版(FE/Reference)** | Founders Edition | NVIDIA/AMD自己设计的显卡。第三方(华硕/微星等)的叫非公/AIB版 |
| **AIB** | Add-In Board Partner | 显卡合作伙伴(华硕、微星、技嘉、七彩虹等) |
| **核心频率(Core Clock)** | - | GPU芯片工作频率，有基础和加速频率 |
| **显存频率/等效频率** | - | GDDR6实际频率x4=等效频率(如2000MHz等效8000MHz) |
| **Shader(着色器)** | - | GPU上执行渲染的可编程单元。顶点/像素/计算着色器 |
| **ROP** | Render Output Unit | 光栅化输出单元，影响高分辨率性能 |

## 十七、单位与度量

| 术语 | 含义 | 换算/说明 |
|------|------|-----------|
| **GB / TB** | 容量 | 1TB = 1024GB。内存/硬盘容量 |
| **MB/s、GB/s** | 传输速度 | 1GB/s = 1024MB/s。SSD速度、接口带宽 |
| **Mbps / Gbps** | 比特率 | 注意大小写! MB/s是字节，Mbps是比特。1GB/s = 8Gbps。网速常用Mbps |
| **GHz / MHz** | 频率 | 1GHz = 1000MHz。CPU主频/内存频率 |
| **GT/s** | 传输率 | 每秒几十亿次传输。PCIe 4.0 = 16GT/s |
| **IOPS** | I/O Operations Per Second | 每秒读写操作次数。SSD随机性能的核心指标(比MB/s更重要) |
| **nm(纳米)** | 长度 | CPU制程。越小越先进 |
| **W(瓦特)** | 功率 | TDP/TGP/电源功率 |
| **V(伏特)** | 电压 | DDR5 1.1V，主板12V/5V/3.3V |
| **A(安培)** | 电流 | 功率=电压x电流(P=V*I)。12V供电50A=600W |
| **mV(毫伏)** | 电压 | 1V=1000mV。纹波、Vdroop都用mV |
| **bit(位)** | 数据最小单位 | 一个0或1。显存位宽用bit |
| **Byte(字节)** | 数据单位 | 1Byte=8bit。文件大小用Byte |
| **pin** | 针脚/触点 | 接口触点数。24pin/8pin |
| **dB / dBA** | 分贝 | 噪音单位。25dBA(安静)、35dBA(正常)、45dBA+(吵) |
