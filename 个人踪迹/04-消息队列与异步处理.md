### 1.2 队列定义

```java
@Configuration
public class RabbitMQConfig {
    
    public static final String EXCHANGE_NAME = "reitti-exchange";
    
    // 位置数据队列 - 处理原始GPS数据
    public static final String LOCATION_DATA_QUEUE = "reitti.location.data.v2";
    public static final String LOCATION_DATA_ROUTING_KEY = "reitti.location.data.v2";
    
    // 重要地点队列 - 处理新创建的地点
    public static final String SIGNIFICANT_PLACE_QUEUE = "reitti.place.created.v2";
    public static final String SIGNIFICANT_PLACE_ROUTING_KEY = "reitti.place.created.v2";
    
    // 行程重算队列 - 触发行程重新计算
    public static final String RECALCULATE_TRIP_QUEUE = "reitti.trip.recalculate.v2";
    public static final String DETECT_TRIP_RECALCULATION_ROUTING_KEY = "reitti.trip.recalculate.v2";
    
    // 处理触发队列 - 触发处理管道
    public static final String TRIGGER_PROCESSING_PIPELINE_QUEUE = "reitti.processing.v2";
    public static final String TRIGGER_PROCESSING_PIPELINE_ROUTING_KEY = "reitti.processing.start.v2";
    
    // 用户事件队列 - 用户相关事件
    public static final String USER_EVENT_QUEUE = "reitti.user.events.v2";
    public static final String USER_EVENT_ROUTING_KEY = "reitti.user.events.updated.v2";
    
    // 死信队列
    public static final String DLX_NAME = "reitti.dlx.exchange";
    public static final String DLQ_NAME = "reitti.dql.v2";
}
```

### 1.3 队列配置详解

```java
@Bean
public Queue locationDataQueue() {
    return QueueBuilder.durable(LOCATION_DATA_QUEUE)
            // 配置死信交换机
            .withArgument("x-dead-letter-exchange", DLX_NAME)
            // 配置死信路由键
            .withArgument("x-dead-letter-routing-key", DLQ_NAME)
            .build();
}

@Bean
public Queue triggerProcessingQueue() {
    // 非持久化队列 - 重启后不保留
    return QueueBuilder.nonDurable(TRIGGER_PROCESSING_PIPELINE_QUEUE)
            .withArgument("x-dead-letter-exchange", DLX_NAME)
            .withArgument("x-dead-letter-routing-key", DLQ_NAME)
            .build();
}
```

**设计要点**:
- `durable`: 持久化队列，RabbitMQ 重启后保留
- `nonDurable`: 非持久化队列，适用于临时任务
- 所有队列都配置死信交换机，失败消息不会丢失

## 二、消息重试机制

### 2.1 重试配置

```properties
# application.properties

# 启用重试
spring.rabbitmq.listener.simple.retry.enabled=true

# 初始重试间隔 1秒
spring.rabbitmq.listener.simple.retry.initial-interval=1000ms

# 最大重试次数 4次
spring.rabbitmq.listener.simple.retry.max-attempts=4

# 重试间隔倍数 2倍
spring.rabbitmq.listener.simple.retry.multiplier=2

# 最大重试间隔 10秒
spring.rabbitmq.listener.simple.retry.max-interval=10000ms

# 预取数量
spring.rabbitmq.listener.simple.prefetch=10

# 关键配置: 重试耗尽后不重新入队，触发死信
spring.rabbitmq.listener.simple.default-requeue-rejected=false
```

### 2.2 重试时间线

```
第1次尝试 → 失败 → 等待 1s
第2次尝试 → 失败 → 等待 2s (1s × 2)
第3次尝试 → 失败 → 等待 4s (2s × 2)
第4次尝试 → 失败 → 等待 8s (4s × 2, 但不超过10s)
第5次尝试 → 失败 → 发送到死信队列
```

```java
public class ProcessingPipelineTrigger {
    
    private final UnifiedLocationProcessingService processingService;

    @RabbitListener(
        queues = RabbitMQConfig.TRIGGER_PROCESSING_PIPELINE_QUEUE,
        concurrency = "${reitti.events.concurrency:1-16}"

    public void onTriggerProcessing(TriggerProcessingEvent event) {
        logger.info("Received processing trigger for user [{}]", event.getUsername());

            event.getUsername(),

    }
}
```

### 4.3 并发配置

```properties

reitti.events.concurrency=1-16
```

```java
@RabbitListener(
    queues = RabbitMQConfig.TRIGGER_PROCESSING_PIPELINE_QUEUE,
    concurrency = "${reitti.events.concurrency:1-16}"  // 1-16个消费者
```

**并发策略**:
- `1-16`: 最少1个消费者，最多16个消费者

- 低负载时减少资源消耗，高负载时自动扩展

## 五、消息发送模式

### 5.1 消息分发服务

```java
@Service
public class MessageDispatcherService {
    
    private final RabbitTemplate rabbitTemplate;
    
    public void dispatchLocationData(String username, List<LocationPoint> points) {
        LocationDataEvent event = new LocationDataEvent(username, points);
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.EXCHANGE_NAME,
            RabbitMQConfig.LOCATION_DATA_ROUTING_KEY,
            event
        );
    }
    
    public void triggerProcessing(String username, Instant earliest, Instant latest) {
        TriggerProcessingEvent event = new TriggerProcessingEvent(
            username, null, earliest, latest
        );
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.EXCHANGE_NAME,
            RabbitMQConfig.TRIGGER_PROCESSING_PIPELINE_ROUTING_KEY,
            event
        );
    }
    
    public void notifyPlaceCreated(Long placeId, String username) {
        SignificantPlaceCreatedEvent event = new SignificantPlaceCreatedEvent(
            placeId, username
        );
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.EXCHANGE_NAME,
            RabbitMQConfig.SIGNIFICANT_PLACE_ROUTING_KEY,
            event
        );
    }
}
```

## 六、延迟处理机制

### 6.1 处理等待时间

```properties
# 最后一次数据输入后等待15秒再开始处理
reitti.import.processing-idle-start-time=15
```

**设计目的**:
- 避免频繁触发处理
- 等待批量数据完整到达
- 减少重复计算

### 6.2 实现方式

```java
@Service
public class DefaultImportProcessor implements ImportProcessor {
    
    private final Map<String, ScheduledFuture<?>> pendingProcessing = new ConcurrentHashMap<>();
    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
    
    @Value("${reitti.import.processing-idle-start-time:15}")
    private int processingIdleStartTime;
    
    public void processBatch(User user, List<LocationPoint> points) {
        // 保存原始数据
        saveRawLocationPoints(user, points);
        
        // 取消之前的延迟任务
        ScheduledFuture<?> existing = pendingProcessing.get(user.getUsername());
        if (existing != null) {
            existing.cancel(false);
        }
        
        // 创建新的延迟任务
        ScheduledFuture<?> future = scheduler.schedule(
            () -> triggerProcessing(user),
            processingIdleStartTime,
            TimeUnit.SECONDS
        );
        pendingProcessing.put(user.getUsername(), future);
    }
}
```

## 七、队列监控

### 7.1 队列状态服务

```java
@Service
public class QueueStatsService {
    
    private final RabbitAdmin rabbitAdmin;
    
    public QueueStats getQueueStats(String queueName) {
        Properties props = rabbitAdmin.getQueueProperties(queueName);
        if (props != null) {
            int messageCount = (Integer) props.get(RabbitAdmin.QUEUE_MESSAGE_COUNT);
            int consumerCount = (Integer) props.get(RabbitAdmin.QUEUE_CONSUMER_COUNT);
            return new QueueStats(queueName, messageCount, consumerCount);
        }
        return null;
    }
    
    public List<QueueStats> getAllQueueStats() {
        return List.of(
            getQueueStats(RabbitMQConfig.LOCATION_DATA_QUEUE),
            getQueueStats(RabbitMQConfig.SIGNIFICANT_PLACE_QUEUE),
            getQueueStats(RabbitMQConfig.RECALCULATE_TRIP_QUEUE),
            getQueueStats(RabbitMQConfig.TRIGGER_PROCESSING_PIPELINE_QUEUE),
            getQueueStats(RabbitMQConfig.DLQ_NAME)
        );
    }
}
```

## 八、学习要点

### 8.1 消息队列设计原则

1. **队列版本化**: 使用 `.v2` 后缀，便于升级迁移
2. **死信处理**: 所有队列配置 DLX，失败消息不丢失
3. **持久化策略**: 重要数据用持久化队列，临时任务用非持久化
4. **并发控制**: 使用动态并发配置适应负载变化

### 8.2 可靠性保证

1. **消息确认**: 处理完成后才确认消息
2. **重试机制**: 指数退避重试，避免雪崩
3. **幂等处理**: 消息可能重复，处理逻辑需幂等
4. **死信监控**: 监控死信队列，及时发现问题

### 8.3 性能优化

1. **批量处理**: 合并多个位置点一起处理
2. **延迟触发**: 等待数据完整后再处理
3. **预取配置**: 合理设置 prefetch 提高吞吐
4. **消费者扩展**: 根据负载自动调整消费者数量

### 8.4 监控与运维

1. **队列深度**: 监控消息堆积情况
2. **消费者数量**: 监控活跃消费者
3. **死信数量**: 监控处理失败的消息
4. **处理延迟**: 监控消息处理时间
