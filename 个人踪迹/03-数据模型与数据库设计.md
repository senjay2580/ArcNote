## 一、核心领域模型

### 1.1 地理位置模型

#### RawLocationPoint (原始位置点)
```java
public class RawLocationPoint {
    private Long id;
```

**设计亮点**:
- 不可变对象设计，通过 `withXxx()` 方法创建新实例
- 支持标记为合成点 (用于数据密度归一化)
- 支持标记为忽略 (用于异常数据过滤)

## 二、数据库表结构

### 2.1 初始表结构 (V1)

```sql
-- 原始位置点表
CREATE TABLE raw_location_points (
    id                BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    accuracy_meters   FLOAT(53) NOT NULL,
    latitude          FLOAT(53) NOT NULL,
    longitude         FLOAT(53) NOT NULL,
    timestamp         TIMESTAMP(6) WITH TIME ZONE NOT NULL,
    user_id           BIGINT NOT NULL REFERENCES users(id)
);

-- 重要地点表
CREATE TABLE significant_places (
    id                 BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    address            VARCHAR(255),
    name               VARCHAR(255),
    latitude_centroid  FLOAT(53) NOT NULL,
    longitude_centroid FLOAT(53) NOT NULL,
    user_id            BIGINT NOT NULL REFERENCES users(id)
);

-- 行程表
CREATE TABLE trips (
    id                        BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    duration_seconds          BIGINT NOT NULL,
    start_time                TIMESTAMP(6) WITH TIME ZONE NOT NULL,
    end_time                  TIMESTAMP(6) WITH TIME ZONE NOT NULL,
    estimated_distance_meters FLOAT(53),
    transport_mode_inferred   VARCHAR(255),
    start_place_id            BIGINT REFERENCES significant_places(id),
    end_place_id              BIGINT REFERENCES significant_places(id),
    user_id                   BIGINT NOT NULL REFERENCES users(id)
);

-- 访问表
CREATE TABLE visits (
    id               BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    duration_seconds BIGINT NOT NULL,
    start_time       TIMESTAMP(6) WITH TIME ZONE NOT NULL,
    end_time         TIMESTAMP(6) WITH TIME ZONE NOT NULL,
    place_id         BIGINT NOT NULL REFERENCES significant_places(id),
    user_id          BIGINT NOT NULL REFERENCES users(id)
);

-- 用户表
CREATE TABLE users (
    id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    display_name VARCHAR(255) NOT NULL,
    username     VARCHAR(255) NOT NULL UNIQUE
);
```

## 三、数据访问层设计

### 3.1 JDBC Service 模式

项目使用纯 JDBC 而非 JPA/Hibernate，提供更好的性能控制:

```java
@Service
public class RawLocationPointJdbcService {
    private final JdbcTemplate jdbcTemplate;
    
    // 批量插入优化
    public List<RawLocationPoint> bulkInsert(User user, List<RawLocationPoint> points) {
        String sql = """
            INSERT INTO raw_location_points 
            (user_id, timestamp, geom, accuracy_meters, elevation_meters, processed, synthetic, ignored, version)
            VALUES (?, ?, ST_SetSRID(ST_MakePoint(?, ?), 4326), ?, ?, ?, ?, ?, 1)
            ON CONFLICT (user_id, timestamp, geom) DO NOTHING
            RETURNING id
            """;
        // 使用 BatchPreparedStatementSetter 批量执行
    }
    
    // 空间聚类查询
    public List<ClusteredPoint> findClusteredPointsInTimeRangeForUser(
            User user, Instant start, Instant end, int minPoints, double distance) {
        String sql = """
            SELECT id, timestamp, geom, accuracy_meters, elevation_meters, 
                   processed, synthetic, ignored, version,
                   ST_ClusterDBSCAN(geom, eps := ?, minpoints := ?) 
                       OVER (ORDER BY timestamp) as cluster_id
            FROM raw_location_points
            WHERE user_id = ? AND timestamp BETWEEN ? AND ? AND NOT ignored
            ORDER BY timestamp
            """;
        return jdbcTemplate.query(sql, rowMapper, distance, minPoints, 
                                  user.getId(), start, end);
    }
}
```

### 3.2 PointReaderWriter (空间数据读写)

```java
public class PointReaderWriter {
    private final GeometryFactory geometryFactory;
    
    // 从 ResultSet 读取 PostGIS Point
    public GeoPoint readPoint(ResultSet rs, String columnName) throws SQLException {
        PGgeometry geom = (PGgeometry) rs.getObject(columnName);
        if (geom != null) {
            Point point = (Point) geom.getGeometry();
            return new GeoPoint(point.getY(), point.getX()); // lat, lng
        }
        return null;
    }
    
    // 写入 PostGIS Point
    public void setPoint(PreparedStatement ps, int index, GeoPoint point) 
            throws SQLException {
        Point geom = geometryFactory.createPoint(
            new Coordinate(point.longitude(), point.latitude())
        );
        geom.setSRID(4326);
        ps.setObject(index, new PGgeometry(geom));
    }
}
```

### 4.2 乐观锁并发控制

```java
public class OptimisticLockException extends RuntimeException {
    public OptimisticLockException(String message) {
        super(message);
    }
}

// 更新时检查版本
public void update(SignificantPlace place) {
    int updated = jdbcTemplate.update("""
        UPDATE significant_places 
        SET name = ?, address = ?, version = version + 1
        WHERE id = ? AND version = ?
        """, place.getName(), place.getAddress(), 
             place.getId(), place.getVersion());
    
    if (updated == 0) {
        throw new OptimisticLockException(
            "SignificantPlace was modified by another transaction");
    }
}
```

## 五、学习要点

### 5.1 数据库设计原则

1. **渐进式迁移**: 70+ 个小步迁移，每次只做一件事
2. **空间数据优化**: 使用 PostGIS 原生类型和索引
3. **时间数据处理**: 使用 `TIMESTAMP WITH TIME ZONE`
4. **并发控制**: 乐观锁避免悲观锁的性能问题

### 5.2 性能优化技巧

1. **批量操作**: 使用 `ON CONFLICT DO NOTHING` 去重
2. **空间索引**: GIST 索引加速空间查询
3. **复合索引**: `(user_id, timestamp)` 加速时间范围查询
4. **JDBC 直接访问**: 避免 ORM 的额外开销

### 5.3 可维护性设计

1. **清晰的表命名**: 表名反映业务含义
2. **版本控制**: Flyway 管理所有数据库变更
3. **预览隔离**: 预览数据不影响生产数据
4. **软删除/标记**: 使用标记而非物理删除
