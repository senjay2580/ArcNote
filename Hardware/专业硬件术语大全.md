## 一、CPU 架构深度

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **微架构** | Microarchitecture | CPU 内部电路设计方案，决定 IPC 高低。同一 ISA 可有不同微架构，如 Intel 的 Alder Lake、AMD 的 Zen 4 |
| **IPC** | Instructions Per Clock | 每时钟周期执行的指令数，是衡量架构效率的核心指标，与频率共同决定实际性能 |
| **流水线** | Pipeline | 将指令拆分为多个阶段（取指→译码→执行→写回）并行处理，像工厂流水线提升吞吐量 |
| **流水线深度** | Pipeline Depth / Stages | 流水线的阶段数。深管线可以跑更高频率，但分支预测失败惩罚更大 |
| **前端** | Frontend | CPU 中负责取指和译码的部分，将指令翻译成微操作（µops）送给执行单元 |
| **后端** | Backend | CPU 中负责调度和执行微操作的部分，包含执行单元、ROB、RS 等 |
| **分支预测** | Branch Prediction | CPU 提前猜测条件跳转方向并预先执行，预测错误需冲刷流水线，影响性能 |
| **分支预测失败惩罚** | Branch Misprediction Penalty | 分支预测错误时清空错误路径并重新执行的代价，以时钟周期计 |
| **乱序执行** | Out-of-Order Execution (OoOE) | CPU 不按指令顺序执行，而是把没有依赖关系的后续指令提前执行，提高执行单元利用率 |
| **投机执行** | Speculative Execution | CPU 预先执行可能需要的代码（如分支两侧），确认后再提交结果，Spectre 漏洞与此相关 |
| **超标量** | Superscalar | CPU 每个时钟周期可以同时发射多条指令到不同执行单元，现代高性能 CPU 都是超标量 |
| **重排序缓冲区** | ROB (Reorder Buffer) | 保证乱序执行后按原始顺序提交结果的硬件缓冲，大 ROB = 更深的乱序窗口 |
| **保留站** | RS (Reservation Station) | 存放等待操作数就绪的微操作，操作数齐全后立即发射到执行单元 |
| **寄存器重命名** | Register Renaming | 把逻辑寄存器映射到更多物理寄存器，消除写后写（WAW）和写后读（WAR）假依赖 |
| **执行端口** | Execution Port | CPU 后端的执行通道，不同端口负责不同类型运算（整数/浮点/加载/存储） |
| **µop** | Micro-operation | CISC 指令被解码器拆分成的更小内部操作单元，CPU 实际调度和执行的粒度 |
| **µop 缓存** | µop Cache / Decoded ICache | 缓存已译码的微操作，避免重复译码热点代码，节省前端带宽 |
| **指令集架构** | ISA (Instruction Set Architecture) | CPU 能理解的指令集合，x86-64 是 PC 主流，ARM 用于移动端 |
| **x86 / x86-64** | - | Intel 发明的 CISC 指令集，x86-64 是 64 位扩展版本，PC 行业标准 |
| **CISC** | Complex Instruction Set Computer | 复杂指令集，单条指令可完成复杂操作，代表：x86 |
| **RISC** | Reduced Instruction Set Computer | 精简指令集，指令简单统一，更易流水线化，代表：ARM、RISC-V |
| **SSE** | Streaming SIMD Extensions | Intel 的 SIMD 扩展，每次处理 128 位数据，SSE2/3/4.1/4.2 逐代增强 |
| **AVX** | Advanced Vector Extensions | 扩展 SIMD 宽度至 256 位（AVX2）和 512 位（AVX-512），用于科学计算和 AI |
| **AVX-512** | Advanced Vector Extensions 512-bit | 每次处理 512 位数据的 SIMD 扩展，性能强但功耗高、可能降频 |
| **SIMD** | Single Instruction Multiple Data | 一条指令同时处理多个数据，音视频、AI、加密运算的基础 |
| **FMA** | Fused Multiply-Add | 把乘法和加法合并为一条指令（a×b+c），提高浮点吞吐量，AI 训练大量使用 |
| **Die（裸片）** | Silicon Die | 从晶圆上切下的单个 CPU 芯片本体，封装前的状态 |
| **晶圆** | Wafer | 制造芯片的硅圆盘，直径通常 300mm，一张晶圆可切出数百个 Die |
| **良率** | Yield | 一张晶圆上可用裸片的比例，越高越好，制程越先进初期良率越低 |
| **Chiplet（小芯片）** | Chiplet | 把 CPU 拆成多个小裸片分别制造再封装在一起，可混用不同制程，降低成本 |
| **Monolithic Die（单片式）** | Monolithic | 整个 CPU 用一个裸片制造，集成度高但面积大、良率低 |
| **CCD** | Core Complex Die | AMD Zen 架构的计算小芯片，包含核心和 L3 缓存，Ryzen 7000 每个 CCD 有 8 核 |
| **CCX** | Core Complex | AMD 早期架构中 CPU 核心的分组单元，同 CCX 内的核心共享 L3 缓存 |
| **IOD / IOX** | I/O Die / I/O Chiplet | AMD 芯片中负责内存控制器、PCIe、USB 等 I/O 功能的专用裸片 |
| **Infinity Fabric** | - | AMD 芯片内 CCD 与 IOD 之间的高速互联总线，频率与内存频率相关联 |
| **Ring Bus（环形总线）** | Ring Bus | Intel Skylake 之前和 Alder Lake 中用于连接核心与 LLC 的环形互联网络 |
| **Mesh 互联** | Mesh Interconnect | Intel 高核心数 CPU 中用网格状拓扑连接核心与 L3 的互联方案，可扩展性优于 Ring |
| **IMC** | Integrated Memory Controller | 集成在 CPU 内的内存控制器，直接控制内存读写，消除了老式北桥的延迟 |
| **LLC** | Last Level Cache | 最后一级缓存（通常是 L3），所有核心共享，未命中才访问内存 |
| **L1 / L2 / L3 缓存** | Level 1/2/3 Cache | CPU 三级缓存，L1 最快最小（几十KB），L3 最慢最大（数十MB） |
| **缓存命中率** | Cache Hit Rate | 数据在缓存中找到的比例，越高性能越好，是算法优化的重要考量 |
| **缓存一致性** | Cache Coherency | 多核 CPU 中各核心缓存数据保持一致的协议机制（MESI 等） |
| **P-core** | Performance Core | Intel 12 代起 Hybrid 架构中的大核，追求高单核性能，有超线程 |
| **E-core** | Efficiency Core | Intel Hybrid 架构中的小核，追求能效，无超线程，适合后台任务 |
| **Thread Director** | Intel Thread Director | Intel 调度硬件提示，告知操作系统将线程分配到合适的大/小核 |
| **硅彩票** | Silicon Lottery | 同批次 CPU 制造误差导致个体差异，部分个体可超更高频率，是随机的 |
| **Binning（分档）** | Binning | 对晶圆上裸片按性能/功耗/良率测试后分级的过程，高频型号来自质量更好的 Die |
| **Stepping（步进）** | Stepping | CPU 的硬件修订版本，同代 CPU 可能有多个 Stepping，用于修复 Bug 或改善特性 |
| **Vcore** | Core Voltage | CPU 核心工作电压，超频时提升 Vcore 提高稳定性，但增加功耗和发热 |
| **VID** | Voltage IDentification | CPU 向主板 VRM 请求的目标电压编码，CPU 内部根据负载动态调整 |
| **Vdroop / 电压骤降** | Voltage Droop | CPU 突然高负载瞬间 Vcore 下跌的现象，由线路阻抗和 VRM 响应速度决定 |
| **LLC** | Load Line Calibration | 主板 BIOS 中补偿 Vdroop 的设置，LLC 等级越高补偿越强，过高会导致 Vovershoot |
| **Vovershoot** | Voltage Overshoot | LLC 设置过高时，轻载瞬间电压超过设定值的现象，长期过高会加速 CPU 退化 |
| **DVFS** | Dynamic Voltage and Frequency Scaling | 动态调整 CPU 电压和频率的机制，负载低时降压降频省电，满载时升压升频 |
| **C-states** | CPU Power States | CPU 的睡眠/省电状态，C0=运行，C1=停止主频但保持供电，C6=深度睡眠，数字越大越省电 |
| **P-states** | Performance States | CPU 在不同性能需求下工作的频率/电压组合，ACPI 电源管理的一部分 |
| **TurboBoost / Precision Boost** | - | Intel/AMD 的自动超频技术，在功耗和温度允许范围内自动拉升频率 |
| **PL1 / PL2** | Power Limit 1 / 2 | Intel 的功耗限制。PL1 是长期功耗上限，PL2 是短期（几十秒）爆发功耗上限 |
| **Tau（时间窗口）** | Power Limit Time Window | PL2 持续的时间上限，超过后降回 PL1。主板可以解除这个限制（Power Limit Unlock） |
| **TDP** | Thermal Design Power | 散热器需要能带走的热量，现代 CPU 实际满载功耗往往远超标称 TDP |
| **PPT / TDC / EDC** | Package Power Tracking / etc. | AMD 平台的三维功耗限制：整体功耗、VRM 电流（稳态/瞬态）限制 |
| **电迁移** | Electromigration | 过高电流密度导致芯片内部金属互连线原子迁移，长期过压超频的劣化原因 |
| **热节流** | Thermal Throttling | CPU/GPU 过热时自动降频保护自身，温度降低后恢复 |
| **核心频率** | Core Clock / Base Clock | CPU 日常运行的基础频率（单位 GHz），是保证稳定运行的最低频率 |
| **制程节点** | Process Node | 晶圆厂的制造工艺代号，如 TSMC N3E、Samsung 4nm，数字不再是真实物理尺寸 |
| **FinFET** | Fin Field-Effect Transistor | 三维鳍式晶体管结构，7nm 以下主流工艺，控制漏电优于平面晶体管 |
| **GAA** | Gate-All-Around | 环绕栅极晶体管，下一代工艺（TSMC 2nm、Samsung 3nm）用于取代 FinFET |
| **封装** | Packaging | 将裸片保护并提供电气连接的工艺，如 LGA、BGA、Flip-Chip |
| **Flip-Chip** | - | 将裸片倒扣通过锡球连接基板的封装方式，现代 CPU/GPU 通用封装方法 |
| **基板** | Substrate / Interposer | 芯片和 PCB 之间的中间层，提供电气连接和机械支撑 |
| **CoWoS** | Chip on Wafer on Substrate | 台积电的 2.5D 封装技术，将 HBM 和 GPU 并排放在同一硅中介层上，用于 AI 加速卡 |

## 二、内存深度

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **内存颗粒** | DRAM Die | 内存条上实际存储数据的芯片，品牌和代号（如三星 B-die）直接影响超频潜力 |
| **三星 B-die** | Samsung B-die | 三星生产的高品质 DDR4 颗粒，超频潜力最好，可稳定跑 DDR4-4000+，溢价明显 |
| **三星 E-die** | Samsung E-die | 三星较新 DDR4/DDR5 颗粒，超频性能略逊 B-die，但更普及 |
| **海力士 A-die** | SK Hynix A-die | 海力士高端颗粒，DDR5 时代表现突出，是 DDR5-6000+ 超频主力之一 |
| **美光 B-die** | Micron B-die | 美光的高品质颗粒，部分型号超频潜力不亚于三星 B-die |
| **SPD** | Serial Presence Detect | 内存条上的小 EEPROM 芯片，存储容量/频率/时序等参数供主板读取 |
| **JEDEC** | Joint Electron Device Engineering Council | 制定内存标准的行业组织，JEDEC 频率是内存不开 XMP 时的保守默认频率 |
| **XMP** | Extreme Memory Profile | 存储在 SPD 中的 Intel 平台超频配置文件，开启后自动应用厂商测试的高频时序 |
| **EXPO** | Extended Profiles for Overclocking | AMD 平台对应 XMP 的超频配置规范，DDR5 时代 AMD/Intel 都开始支持双标 |
| **内存通道** | Memory Channel | CPU 到内存控制器的独立数据通道，双通道（2×64bit = 128bit）带宽是单通道两倍 |
| **Rank（排）** | Memory Rank | 内存条上共同组成一个 64bit 数据宽度的颗粒组合，单面单 Rank，双面通常双 Rank |
| **Bank（存储组）** | Bank | DRAM 内部独立可并行操作的存储矩阵单元，更多 Bank 可更好隐藏延迟 |
| **Bank Group** | Bank Group | DDR4/DDR5 中将 Bank 分组的结构，不同 Bank Group 间切换更快，提升带宽 |
| **Row（行）/ Column（列）** | Row / Column | DRAM 内部地址寻址的两个维度，读写前先激活行，再选列 |
| **行激活** | Row Activation / RAS | 打开 DRAM 存储矩阵某一行的操作，是访问数据的第一步 |
| **tCL / CL** | CAS Latency | 发出列地址到数据出现的延迟，是最常被引用的内存时序参数 |
| **tRCD** | RAS to CAS Delay | 行地址到列地址操作之间的最小等待时间，越小越好 |
| **tRP** | Row Precharge Time | 关闭当前行（预充电）所需的时间，之后才能打开新行 |
| **tRAS** | Row Active Time | 行保持激活状态的最短时间，必须 ≥ tRCD + tCL |
| **tRC** | Row Cycle Time | tRAS + tRP，代表连续访问同一 Bank 不同行的最短周期 |
| **tRFC** | Refresh Cycle Time | DRAM 刷新操作的时间，DDR5 颗粒容量更大所以 tRFC 更长 |
| **tREFI** | Refresh Interval | 相邻两次刷新操作的间隔，适当拉长 tREFI 可以减少刷新打断，提升带宽 |
| **tWR** | Write Recovery Time | 写入完成后到发起预充电的等待时间 |
| **tRTP** | Read to Precharge Time | 读取完成后到发起预充电的等待时间 |
| **tFAW** | Four Activate Window | 在连续4次 Bank 激活操作中的时间窗口限制，防止供电不足 |
| **Command Rate (CR)** | Command Rate | 地址/命令总线上连续命令之间的间隔，1T 性能最好，2T 兼容性更好 |
| **主时序** | Primary Timings | CL-tRCD-tRP-tRAS 四个最核心的时序参数 |
| **次级时序** | Secondary Timings | tRC、tRFC、tFAW 等第二层时序，深度调优时需要关注 |
| **三级时序** | Tertiary Timings | tRDRD、tWRWR 等更细粒度的时序，极限超频才需要手动设置 |
| **Gear 1** | Gear 1 Mode | 内存控制器与内存以 1:1 频率同步运行，延迟最低，Intel 12/13/14 代限于 DDR4-3200 或 DDR5-4800 |
| **Gear 2** | Gear 2 Mode | 内存控制器频率为内存频率的一半（2:1），延迟增加但可支持更高内存频率 |
| **Gear 4** | Gear 4 Mode | 4:1 频率比，仅 DDR5 高频时使用，延迟最高，带宽优先场景才用 |
| **内存控制器** | Memory Controller / IMC | CPU 内部管理内存读写的硬件单元，其质量决定内存超频上限和延迟 |
| **内存电压** | DRAM Voltage (VDIMM) | 内存工作电压，DDR4 标准 1.2V，超频可至 1.35-1.5V；DDR5 标准 1.1V |
| **SA 电压** | System Agent Voltage (VCCSA) | Intel 平台系统代理电压，影响内存控制器稳定性，内存高频超频时需微调 |
| **IO 电压** | I/O Voltage (VCCIO) | Intel 平台内存 I/O 电压，直接影响信号完整性，过高损伤 IMC |
| **VDDQ** | Data Line Voltage | DDR5 中数据线工作电压，独立于 VDD，通常 1.1V |
| **VPP** | Activation Power Supply | DRAM 行激活使用的高压，DDR4=2.5V，DDR5=1.8V |
| **ECC** | Error-Correcting Code | 错误校正码内存，可检测并纠正单比特错误，服务器/工作站必备 |
| **on-die ECC** | On-Die ECC | DDR5 颗粒内部自带的 ECC，对外透明，用于应对小制程下的比特翻转 |
| **UDIMM** | Unbuffered DIMM | 无缓冲内存，消费级标准内存，信号直达 IMC |
| **RDIMM** | Registered DIMM | 注册缓冲内存，地址/命令信号经寄存器中继，支持更多 Rank，服务器专用 |
| **LRDIMM** | Load-Reduced DIMM | 负载减少内存，在 RDIMM 基础上数据线也经缓冲，支持更大容量，企业级使用 |
| **内存交织** | Memory Interleaving | 将连续地址分布到不同内存控制器/通道/Rank 以提高并行度和带宽的技术 |
| **刷新** | Refresh | DRAM 电容中的电荷会泄漏，必须定期刷新重写，刷新期间内存不可访问 |
| **内存时序自动化** | DRAM Timing Training | 主板 BIOS 开机时自动测试并选择最优时序的过程，失败时降频重试 |
| **Memory Training** | - | 每次开机 POST 阶段主板对内存进行电气校准的过程，DDR5 首次训练特别慢 |
| **HBM** | High Bandwidth Memory | 将 DRAM Die 堆叠并通过硅中介层连接 GPU 的高带宽内存，带宽极高，AI 加速卡专用 |
| **LPDDR** | Low Power DDR | 低功耗 DDR，焊接在笔记本/手机主板上，无法升级但功耗和延迟更低 |
| **内存带宽** | Memory Bandwidth | 单位时间内 CPU 与内存间传输的数据量，影响大数据量运算场景（视频编辑、游戏大地图） |

## 三、存储深度

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **NAND Flash** | - | SSD 中实际存储数据的非易失性闪存芯片，断电不丢数据 |
| **SLC** | Single-Level Cell | 每个存储单元存 1 bit，速度最快、耐久性最高（10 万次 P/E），成本最高 |
| **MLC** | Multi-Level Cell | 每个存储单元存 2 bit，速度和耐久性平衡（3000 次 P/E），企业级 SSD 常用 |
| **TLC** | Triple-Level Cell | 每个存储单元存 3 bit，消费级 SSD 主流（1000 次 P/E），速度和成本折中 |
| **QLC** | Quad-Level Cell | 每个存储单元存 4 bit，密度最高最便宜但耐久性最差（300-500 次 P/E）|
| **PLC** | Penta-Level Cell | 每单元 5 bit，实验室阶段，耐久性极差，目前不用于消费级产品 |
| **3D NAND** | 3D NAND / V-NAND | 将 NAND 单元垂直堆叠多层的技术，提升密度的同时改善了 2D NAND 的制程缩减问题 |
| **层数** | Layer Count | 3D NAND 的垂直层数，如 176 层、232 层，层数越多密度越大 |
| **P/E 周期** | Program/Erase Cycle | 闪存单元的擦写寿命，每次 P/E 循环会轻微损耗绝缘层 |
| **主控（控制器）** | SSD Controller | SSD 的"大脑"，负责 FTL 映射、纠错、磨损均衡、缓存管理，品牌如慧荣 SM2267、群联 E18 |
| **固件** | Firmware | 烧录在 SSD 主控上的软件，影响性能、稳定性和寿命，可更新 |
| **FTL** | Flash Translation Layer | 闪存转换层，将文件系统的逻辑地址映射到 NAND 物理地址，隐藏 NAND 的块擦写特性 |
| **磨损均衡** | Wear Leveling | 将写入操作均匀分散到所有 NAND 块，避免某些块提前磨损报废 |
| **垃圾回收** | Garbage Collection (GC) | SSD 后台整理已删除但未清除的 NAND 块，腾出空间，可能引起写入放大 |
| **TRIM** | - | 操作系统通知 SSD 哪些逻辑块已删除的命令，帮助 GC 工作，保持性能 |
| **写入放大** | Write Amplification (WA) | 实际写入 NAND 的数据量与主机写入量之比，WA=1 最理想，垃圾回收会增大 WA |
| **过量配置** | Over-Provisioning (OP) | 留出不对外开放的额外 NAND 容量用于 GC 和坏块管理，提升耐久性和性能稳定性 |
| **SLC Cache** | SLC Cache / Write Cache | 主控将部分 TLC/QLC 空间模拟为 SLC 使用，大幅提升突发写入速度，写满后速度骤降 |
| **HMB** | Host Memory Buffer | 无 DRAM 缓存的 SSD 借用主机内存存放映射表的技术，改善随机读性能 |
| **DRAM 缓存** | DRAM Cache | SSD 板载的独立 DRAM 芯片，存放频繁访问的 FTL 映射表，无 DRAM SSD 读写映射更慢 |
| **IOPS** | Input/Output Operations Per Second | 每秒读写操作次数，随机性能指标，影响系统响应速度（比顺序读写更重要）|
| **顺序读写** | Sequential Read/Write | 连续地址的读写速度，大文件传输指标，NVMe 4.0 可达 7000MB/s |
| **随机读写** | Random Read/Write | 分散地址的读写速度，系统盘体验指标，以 4K IOPS 衡量 |
| **队列深度** | Queue Depth (QD) | 同时发送给 SSD 的未完成 I/O 请求数，QD1 模拟单任务，QD32 模拟多任务并发 |
| **AHCI** | Advanced Host Controller Interface | SATA 设备的传输协议，最大队列深度 32，单队列，为机械硬盘设计 |
| **NVMe** | Non-Volatile Memory Express | 专为 NAND SSD 设计的高速协议，支持 65535 队列 x 65536 深度，延迟极低 |
| **NVMe 命名空间** | NVMe Namespace | NVMe 协议中将一块 SSD 划分为多个独立逻辑存储空间的功能，企业级应用 |
| **PCIe 5.0 SSD** | - | 走 PCIe 5.0 x4 通道的顶级 SSD，顺序读写可达 14GB/s，但发热大 |
| **RAID 0** | Striping | 数据条带化写入两块以上硬盘，速度翻倍但无冗余，任何一块损坏数据全丢 |
| **RAID 1** | Mirroring | 数据镜像写入两块硬盘，一块损坏另一块完整，容量减半，读取可加速 |
| **RAID 5** | Distributed Parity | 数据+奇偶校验分布在3块以上硬盘，允许1块损坏，容量为 (n-1)×单盘 |
| **RAID 6** | Dual Parity | 双奇偶校验，允许2块同时损坏，需4块以上 |
| **RAID 10** | Stripe of Mirrors | 先镜像后条带，性能和冗余兼顾，需偶数块硬盘 |
| **坏块** | Bad Block | NAND 中无法可靠读写的擦除块，主控会标记并绕过 |
| **S.M.A.R.T.** | Self-Monitoring, Analysis, Reporting Technology | 硬盘内置的自我监控系统，记录健康状态、已用寿命、坏扇区等，用 CrystalDiskInfo 查看 |
| **CrystalDiskMark** | - | 常用 SSD/HDD 性能测试软件，测顺序和随机读写 |
| **AS SSD** | - | 专为 SSD 设计的基准测试工具，测无压缩数据性能更客观 |
| **测速缓存穿透** | Cache Bypass Test | 测试 SSD 写入超过 SLC Cache 后的真实 NAND 写入速度 |
| **SAS** | Serial Attached SCSI | 企业级存储接口，比 SATA 更可靠，支持全双工，常见于服务器硬盘 |
| **HDD 转速** | RPM (Revolutions Per Minute) | 机械硬盘磁盘转速，5400 RPM 省电、7200 RPM 性能更好，10000/15000 RPM 用于服务器 |
| **PMR / CMR** | Perpendicular/Conventional Magnetic Recording | 传统垂直磁记录方式，每道独立，随机写入性能好于 SMR |
| **SMR** | Shingled Magnetic Recording | 叠瓦磁记录，磁道重叠提升密度但随机写入慢，需大量重写，廉价大容量 HDD 常用 |

## 四、主板与 PCB

| 术语 | 全称 | 一句话解释 |
|------|------|-----------|
| **PCB** | Printed Circuit Board | 印刷电路板，通过蚀刻铜箔形成导线，承载所有焊接元件的基板 |
| **层数** | Layer Count | PCB 中铜层的数量，消费级主板通常 4-8 层，高端服务器/显卡 PCB 可达 12+ 层 |
| **铜厚** | Copper Weight | PCB 铜箔厚度，以盎司/平方英尺（oz）计，1 oz ≈ 35µm，VRM 区域常用 2 oz 加强载流 |
| **铜箔** | Copper Trace | PCB 上传导信号和电流的铜质走线 |
| **走线（Trace）** | Trace | PCB 上两点间的铜线路，宽度影响载流量，长度影响信号延迟和阻抗 |
| **过孔（Via）** | Via | 穿透 PCB 层间的金属化小孔，用于不同层铜线的互联 |
| **盲孔** | Blind Via | 只从表层延伸到内层（不穿透全板）的过孔，高密度 PCB 节省空间 |
| **埋孔** | Buried Via | 完全在内层之间的过孔，表面看不到，BGA 封装密集板常用 |
| **阻焊层** | Solder Mask | 覆盖在铜线上的绿色/黑色保护涂层，防止焊锡桥接和短路，只露出焊盘 |
| **丝印层** | Silkscreen | PCB 表面印刷的白色字符和标识，标注元件编号、极性、版本等信息 |
| **焊盘（Pad）** | Pad | PCB 上用于焊接元件引脚的裸露铜面区域 |
| **封装（Footprint）** | Footprint | PCB 上某个元件的焊盘排列图案，必须与实物元件引脚完全对应 |
| **阻抗控制** | Impedance Control | 控制高速信号走线的特征阻抗（通常 50Ω 单端、100Ω 差分），保证信号完整性 |
| **差分对** | Differential Pair | 用两根等长、等间距的走线传输信号，一正一负，抗噪声能力强，USB/PCIe/HDMI 都用 |
| **等长走线** | Length Matching | 将差分对或同组信号走线调成等长，消除时序偏斜（Skew） |
| **参考平面** | Reference Plane | 高速走线下方/上方的完整铜层（电源层/地层），为信号提供稳定返回路径和阻抗参考 |
| **地平面** | Ground Plane | 覆铜接地的完整铜层，为信号提供低阻抗返回路径，减少 EMI |
| **电源平面** | Power Plane | 覆铜接电源的铜层，为所有元件提供低阻抗供电 |
| **SMD** | Surface Mount Device | 表面贴装元件，直接焊在 PCB 表面的焊盘上，现代 PCB 上绝大多数元件是 SMD |
| **SMT** | Surface Mount Technology | 表面贴装技术，将 SMD 元件用焊膏贴装并回流焊接的生产工艺 |
| **THT** | Through-Hole Technology | 通孔插装技术，元件引脚穿过 PCB 孔后在背面焊接，机械强度更好 |
| **回流焊** | Reflow Soldering | SMT 生产工艺：涂焊膏→贴片→过回流炉加热熔锡→冷却，一次焊接所有 SMD |
| **波峰焊** | Wave Soldering | 将装好 THT 元件的 PCB 划过液态锡波，一次焊接所有插装元件 |
| **焊膏** | Solder Paste | 由锡粉+助焊剂混合的膏状物，印刷在焊盘上后加热融化完成 SMT 焊接 |
| **BGA** | Ball Grid Array | 底部用锡球阵列连接的封装，如 CPU/GPU/芯片组。焊接需精确回流焊，维修需专用设备 |
| **BGA 重焊（Reball）** | BGA Reballing | 拆下 BGA 芯片后重新植锡球再焊回的工艺，笔记本显卡维修常用 |
| **植锡** | Ball Placement | 为 BGA 芯片底部重新放置锡球的过程，需要钢网和热风台 |
| **焊锡** | Solder | 锡铅（Sn63Pb37 等）或无铅（SAC305：Sn96.5Ag3Cu0.5）焊料，熔化后连接元件和焊盘 |
| **助焊剂** | Flux | 焊接时去除氧化膜、提高焊锡流动性的化学品，分松香型和水溶型 |
| **冷焊** | Cold Joint | 焊点温度不足或焊后移动导致的不良焊接，外观呈灰暗粗糙，接触电阻高 |
| **连焊/桥接** | Solder Bridge | 相邻焊点之间意外连通的焊锡，导致短路 |
| **立碑** | Tombstoning | SMD 小元件一端先熔化上拉，另一端翘起的焊接缺陷，多见于贴片电阻/电容 |
| **去耦电容** | Decoupling Capacitor | 紧靠芯片放置的小电容，滤除供电噪声，补充芯片高速开关时的瞬态电流需求 |
| **旁路电容** | Bypass Capacitor | 与去耦电容类似，将电源噪声旁路到地，两者常互换使用 |
| **MLCC** | Multi-Layer Ceramic Capacitor | 多层陶瓷电容，高频特性好，尺寸小，VRM 输出端大量使用 |
| **铁氧体磁珠** | Ferrite Bead | 串在信号线或电源线上抑制高频噪声的元件，通交流阻高频 |
| **共模扼流圈** | Common Mode Choke | 抑制共模 EMI 的元件，以太网口旁边常见，过滤差分信号中的共模噪声 |
| **保险丝** | Fuse | 过流时熔断保护电路的元件，部分主板用可恢复的 PTC 热敏电阻替代 |
| **TVS 二极管** | Transient Voltage Suppressor | 钳制瞬态过压的保护二极管，接口旁常见，吸收静电或浪涌 |
| **测试点** | Test Point | PCB 上专门用于探针测量的裸露焊盘，维修时用来确认电压 |
| **金手指镀金** | Gold Finger Plating | PCB 边缘连接器触点上的镀金层，防氧化，提高接触可靠性 |
| **电路图** | Schematic | 电路逻辑关系图，用符号表示元件和连接关系，维修时用来追踪信号 |
| **板图（Boardview）** | Boardview / PCB Layout | PCB 的实际元件位置和走线视图，对照原理图用于物理定位故障点 |
| **飞线（Bodge Wire）** | Bodge Wire / Jumper Wire | 在 PCB 表面用细导线手工连接两点，绕过损坏走线的维修手段 |
