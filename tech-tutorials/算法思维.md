```ad-info
核心算法思维可以总结成 **「分布式数据的「无冲突契约式协同」」**，拆解成 3 个可落地的算法逻辑：

1. **先本地自治，再全局收敛**
    
    算法的核心前提是「信任节点的本地操作」：不管网络状态如何，每个节点都可以先在本地完成数据修改，不用等待其他节点的同意或同步，这是解决分布式场景下 “卡顿、离线不可用” 的基础；
    
    而全局的一致性，交给算法提前定义的规则自动完成，不需要人工干预。
2. **用「唯一标识 + 优先级规则」消解冲突**
    
    算法不会让冲突发生，而是提前给所有数据操作约定好 “解决冲突的标准答案”：
    - 给每个操作 / 数据片段分配**唯一的身份标识**（比如时间戳、客户端 ID + 逻辑时钟、全局唯一字符 ID），用来区分不同节点的操作；
    - 提前定义**优先级规则**（比如时间戳晚的优先、ID 字典序小的优先），所有节点都遵守同一套规则，不管操作顺序如何，合并结果都是确定的。
3. **幂等的合并逻辑，保证结果唯一**
    
    算法的合并逻辑是「幂等、交换、结合」的：
    - 幂等：重复合并多次，结果不会变；
    - 交换：不管先合并 A 和 B，还是先合并 B 和 A，结果一样；
    - 结合：分多次合并（比如先合并 A+B，再合并 C），和一次性合并（A+B+C）的结果一样；
        
        这就保证了不管多少个节点、多少次修改，最终一定会收敛到同一个结果。
```
